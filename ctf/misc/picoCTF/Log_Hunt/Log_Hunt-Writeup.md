# Log Hunt Writeup

## 1. 問題概要

本問は、サーバログ中に断片的に漏洩したフラグの一部を収集し、元のフラグを復元するフォレンジック系問題である。ログには `FLAGPART` として複数の行が記録されており、散在・重複している可能性がある。これらを正しく連結し、完全なフラグを構築することが目的である。

## 2. 解析対象ログ

与えられたログには以下の断片が含まれていた。

```
[1990-08-09 10:00:10] INFO FLAGPART: picoCTF{us3_
[1990-08-09 10:02:55] INFO FLAGPART: y0urlinux_
[1990-08-09 10:05:54] INFO FLAGPART: sk1lls_
[1990-08-09 12:28:52] INFO FLAGPART: cedfa5fb}
```

## 3. 解法設計

* `FLAGPART:` に続く部分をすべて抽出する。
* 断片を時刻順に並べる。
* フラグ形式 `picoCTF{...}` に従い、単純連結によって完全なフラグを復元する。
* 重複断片が存在する可能性があるが、本問のデータでは不要であった。

## 4. 復元結果

断片を結合すると、以下の完全なフラグとなる。

```
picoCTF{us3_y0urlinux_sk1lls_cedfa5fb}
```

## 5. 最終確認

* 開始 `{` と終了 `}` が適切に閉じている。
* 各断片が不自然なく連結されている。
* picoCTF の形式に整合する。
