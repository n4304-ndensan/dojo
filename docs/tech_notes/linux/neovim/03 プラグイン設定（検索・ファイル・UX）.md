# プラグイン設定（検索・ファイル・UX）

## この文書の責務

Neovim の検索、ファイル管理、UX 基盤となるプラグイン群を設定します。ここで扱うのは **fzf-lua**（検索）、**oil.nvim**（ファイル操作）、**snacks.nvim**（UX 統合）、**gitsigns.nvim**（Git 差分表示）、**which-key.nvim**（キー一覧）です。これらはすべて `lua/plugins/core.lua` に1ファイルで定義します。

## 前提

- [[02 Neovim コア設定]] が完了していること
- lazy.nvim が動作すること

## 設定ファイル

`~/.config/nvim/lua/plugins/core.lua`:

```lua
return {
  {
    "folke/which-key.nvim",
    event = "VeryLazy",
    opts = {},
  },

  {
    "folke/snacks.nvim",
    priority = 1000,
    lazy = false,
    opts = {
      bigfile = { enabled = true },
      notifier = { enabled = true },
      quickfile = { enabled = true },
      terminal = { enabled = true },
      picker = { enabled = true },
    },
  },

  {
    "ibhagwan/fzf-lua",
    dependencies = { "nvim-tree/nvim-web-devicons" },
    opts = {},
  },

  {
    "stevearc/oil.nvim",
    dependencies = { "nvim-tree/nvim-web-devicons" },
    opts = {
      default_file_explorer = true,
      view_options = {
        show_hidden = true,
      },
    },
  },

  {
    "lewis6991/gitsigns.nvim",
    opts = {},
  },
}
```

## 各プラグインの役割と設計判断

### which-key.nvim

キーマッピングの一覧をポップアップ表示するプラグインです。`<leader>` を押して少し待つと、利用可能なキーバインドが表示されます。

- `event = "VeryLazy"`: 起動後に遅延読み込みする。起動速度に影響しない
- `opts = {}`: デフォルト設定で十分。キーマッピング側に `desc` を付けていれば自動で認識する

### snacks.nvim

folke 氏が提供する「小さなユーティリティの詰め合わせ」です。個別にプラグインを入れるよりも部品点数が減り、管理が楽になります。

有効化するモジュール:

| モジュール | 効果 |
| --- | --- |
| `bigfile` | 大きなファイルを開いたときに自動で treesitter や LSP を無効化し、フリーズを防ぐ |
| `notifier` | `vim.notify` の出力を右上にポップアップ表示する |
| `quickfile` | ファイルを開くときにバッファの初期化を高速化する |
| `terminal` | 統合ターミナルを提供する |
| `picker` | 汎用ピッカー。snacks 内部で使う。fzf-lua と競合はしない |

> **Telescope ではなく fzf-lua を選んだ理由**: Telescope は最も人気のある検索プラグインですが、Lua 実装のファジーマッチャーを使うため、大規模リポジトリでは fzf-lua よりレスポンスが遅くなることがあります。fzf-lua はネイティブの fzf バイナリを使うため、体感速度で優位です。検索を主に使う構成では fzf-lua の方が合います。

### fzf-lua

ファジーファインダーです。ファイル検索、テキスト検索（grep）、バッファ一覧、Git 操作など、ほぼすべての「一覧から選ぶ」操作を担います。

- `dependencies`: アイコン表示に nvim-web-devicons を使う
- `opts = {}`: デフォルト設定で十分に実用的。カスタマイズが必要になったら後から追加する

keymaps.lua で定義したキーマッピングとの対応:

| キー | 関数 | 操作 |
| --- | --- | --- |
| `<leader>ff` | `files()` | ファイル名で検索（ripgrep + fd） |
| `<leader>fg` | `live_grep()` | ファイル内テキストを検索 |
| `<leader>fb` | `buffers()` | 開いているバッファの一覧 |
| `<leader>fr` | `oldfiles()` | 最近開いたファイルの一覧 |
| `<leader>gs` | `git_status()` | Git の変更ファイル一覧 |

### oil.nvim

ファイルマネージャーです。ディレクトリの内容をバッファとして開き、通常のテキスト編集操作（削除、リネーム、コピー）でファイル操作ができます。

- `default_file_explorer = true`: Neovim 標準の netrw を置き換える
- `show_hidden = true`: ドットファイルを表示する

keymaps.lua で `-` にマッピングしているため、ノーマルモードで `-` を押すと現在のファイルの親ディレクトリが oil バッファで開きます。

> **なぜ NvimTree や neo-tree ではないか**: ツリー型ファイラーはサイドバーに常駐して画面を占有します。oil.nvim はバッファとしてディレクトリを開くため、ファイル操作が終わったら消えます。「ファイルを探す → 開く → 消える」のサイクルが速く、画面を無駄にしません。

### gitsigns.nvim

バッファの左端に Git diff のマーカー（追加・変更・削除）をリアルタイム表示します。

- `opts = {}`: デフォルト設定で十分。サイン列は options.lua で `signcolumn = "yes"` にしているため、常に表示される

## 動作確認

ファイルを配置してから Neovim を起動し、`:Lazy sync` を実行します。

```bash
nvim
```

Lazy の管理画面が開いた場合は `S`（Sync）を押してプラグインをインストールします。

確認手順:

1. `<leader>ff` でファイル検索が開くこと
2. `<leader>fg` でテキスト検索（live grep）が動くこと
3. `-` で oil.nvim のファイラーが開くこと
4. Git リポジトリ内のファイルを開き、左端に差分マーカーが出ること
5. `<leader>` を押して which-key のポップアップが出ること
