# zsh ドキュメントガイド

## この文書の責務

Ubuntu 24.04 上に zsh を中心とした CLI 操作環境を構築する一連の手順書群への入口です。zsh 単体の設定ではなく、「CLI 全体の設計」として検索・移動・並列作業・編集までを一貫して扱います。

## 前提

- OS: Ubuntu 24.04 LTS
- シェル: zsh（bash から移行）
- 設計方針: 起動速度・操作効率・拡張性・再現性をすべて満たす

## CLI 全体の設計思想

zsh は「シェル単体」ではなく、CLI 操作環境全体の入力インターフェースとして位置づけます。

```text
zsh ─────── 入力インターフェース（コマンド入力・補完・履歴）
starship ── 状態可視化（ブランチ・言語バージョン・コマンド実行時間）
fzf ─────── 検索（履歴・ファイル・ディレクトリ）
zoxide ──── 移動（学習型ディレクトリジャンプ）
tmux ────── 並列作業（ウィンドウ・ペイン・セッション）
neovim ──── 編集（→ [[00 Neovim ドキュメントガイド]] を参照）
```

この構成により、GUI を使わなくても開発作業のほぼすべてを CLI で完結できます。

## Linux ツール一覧

この構成で使うツールを、役割ごとに整理すると次のようになります。

### 1. シェル基盤

| ツール | カテゴリ | 主用途 | 代表コマンド | 主な代替対象 | 導入箇所 |
| --- | --- | --- | --- | --- | --- |
| zsh | シェル | 補完、履歴、対話操作の中心 | `zsh` | bash | [[02 zsh 基盤セットアップ]] |
| zinit | プラグイン管理 | zsh プラグインの宣言的管理、遅延ロード | `zinit light ...` | oh-my-zsh | [[02 zsh 基盤セットアップ]] |
| starship | プロンプト | Git 状態、言語バージョン、実行時間の可視化 | `starship init zsh` | 手書きの `PS1` / powerlevel10k | [[02 zsh 基盤セットアップ]] |

### 2. 検索・移動

| ツール | カテゴリ | 主用途 | 代表コマンド | 主な代替対象 | 導入箇所 |

| fzf | ファジー検索 | 履歴、ファイル、ディレクトリ、候補一覧の絞り込み | `history \| fzf` / `fd \| fzf` | `history \| grep` / 手作業スクロール | [[03 zshrc 設定とプラグイン]], [[04 CLI 操作環境の構築]] |

### 4. 最小構成で先に入れるもの

最初に入れるべき優先度は次の通りです。

| 優先度 | ツール | 理由 |
| --- | --- | --- |
| 必須 | zsh | すべての土台になる |
| 必須 | zinit | zsh の設定を軽量かつ再現可能に保つ |
| 必須 | starship | プロンプト情報を即座に見えるようにする |
| 必須 | fzf | 履歴検索とファイル選択の体験を大きく改善する |
| 必須 | ripgrep (`rg`) | コード検索の速度が段違いに上がる |
| 必須 | fd | `find` の代わりとして日常的に使う |
| 推奨 | zoxide | ディレクトリ移動の回数が多いほど効く |
| 推奨 | eza | 一覧表示が見やすくなる |
| 推奨 | tmux | 長時間作業や並列作業で効く |

## Neovim との関係

この zsh 文書群と [[00 Neovim ドキュメントガイド]] は相互補完の関係にあります。Neovim の OS 基盤セットアップで導入する ripgrep、fd、fzf などのツールは zsh 側でも使います。セットアップ順序としては、**zsh 環境を先に整えてから Neovim に進む**のが自然です。
