# AKS 自動スケーリング設計（Cluster Autoscaler + HPA + Spot VM）

## 1. 背景（シナリオ）

ある組織では、Azure Kubernetes Service（AKS）を利用して多様なコンテナ化アプリケーションをホストする予定です。これらのアプリケーションは、負荷変動が大きい環境で運用されるため、ピーク時でもパフォーマンスを維持できる柔軟なスケーリングが必要になります。

同時に、クラウドコストを最適化することも重要な要件です。そのため、スポットインスタンスを活用してコスト削減を行いながら、可用性とレジリエンスを確保する必要があります。

さらに、アプリケーションのパフォーマンスを維持するためには、スケーリングイベントを適切に監視し、問題が発生した際には迅速に対応できる監視・アラート機能も必要です。

---

## 2. 要件整理

この問題では AKS クラスターのスケーリング戦略を設計する必要があります。

シナリオから読み取れる要件は次の通りです。

まず、可用性とスケーリングに関する要件です。

- ピーク負荷時でも高可用性を維持
- ノード数の自動スケーリング
- ポッド数の自動スケーリング

次に、コスト最適化の要件です。

- スポットインスタンスを利用
- コストパフォーマンスの向上

最後に、運用管理の要件です。

- スケーリングイベントの監視
- パフォーマンス監視
- アラート機能

これらを満たすためには、Kubernetes の複数のスケーリング機能を組み合わせる必要があります。

---

## 3. 技術の基本概念

Kubernetes では、スケーリングを実現するために複数の仕組みが存在します。

まず、**Horizontal Pod Autoscaler（HPA）** です。

HPA は CPU 使用率やメモリ使用率などのメトリクスに基づいて、ポッドのレプリカ数を自動的に増減します。

この仕組みにより、アプリケーションの処理能力を柔軟に拡張できます。

次に、**Cluster Autoscaler** です。

Cluster Autoscaler はクラスターのノード数を自動的に増減させる機能です。

具体的には次のような動作をします。

- スケジュールできないポッドがある場合にノードを追加
- 使用率が低い場合はノードを削除

この機能により、インフラリソースの効率的な利用が可能になります。

さらに、コスト最適化のために **Spot VM（スポットインスタンス）** を利用できます。

スポットインスタンスは Azure の余剰リソースを低価格で利用できる VM です。

ただし、Azure の判断により停止される可能性があるため、可用性の高いワークロード設計が必要になります。

---

## 4. アーキテクチャまたは設計のポイント

AKS のスケーリング戦略では、次の3つの仕組みを組み合わせることが一般的です。

まず、アプリケーションレベルのスケーリングです。

- **Horizontal Pod Autoscaler**
- ポッド数の自動調整

次に、インフラレベルのスケーリングです。

- **Cluster Autoscaler**
- ノード数の自動調整

最後に、コスト最適化です。

- **Spot VM ノードプール**
- 低コストでのスケールアウト

この構成では、負荷が増加すると次のような流れでスケールします。

1. CPU 使用率増加  
2. HPA がポッド数を増加  
3. ノード不足が発生  
4. Cluster Autoscaler がノードを追加  

このように、ポッドとノードの両方が自動的にスケールします。

---

## 5. 設計判断（なぜこの構成になるか）

この問題では **ノードスケーリングとポッドスケーリングの両方**が必要です。

Cluster Autoscaler と HPA を組み合わせることで次のメリットがあります。

まず、可用性です。

- 負荷に応じたスケーリング
- 高可用性の維持

次に、コスト最適化です。

- スポットインスタンス活用
- 未使用リソース削減

さらに、運用効率です。

- 自動スケーリング
- 手動管理の削減

そのため、この組み合わせが AKS のベストプラクティスとなります。

---

## 6. 他の選択肢が誤りな理由

まず **Virtual Kubelet + ACI** です。

Virtual Kubelet はバーストワークロードに有効ですが、スポットインスタンスによるコスト最適化には直接対応していません。

次に **HPA + Azure Monitor** です。

HPA はポッドのスケーリングのみを提供します。ノードの自動スケーリングには **Cluster Autoscaler** が必要です。

最後に **手動スケーリング**です。

手動スケーリングでは、ピーク負荷への迅速な対応ができず、可用性やパフォーマンス要件を満たせません。

---

## 7. 最終回答

**A. Cluster Autoscaler + Horizontal Pod Autoscaler + Spot Instances**

---

## 8. まとめ

AKS のスケーリング設計では、複数のスケーリング機能を組み合わせることが重要です。

主なスケーリング機能は次の通りです。

まず、アプリケーションスケーリングです。

- **Horizontal Pod Autoscaler (HPA)**
  → ポッド数をスケール

次に、インフラスケーリングです。

- **Cluster Autoscaler**
  → ノード数をスケール

最後に、コスト最適化です。

- **Spot VM**
  → 低コストコンピュート

そのため、**AKS クラスターのスケーリング戦略では HPA と Cluster Autoscaler を組み合わせ、スポットインスタンスでコスト最適化する構成が最適**となります。
