# AKSマイクロサービス環境におけるAzure Container Registryの設計

## 1. 背景（シナリオ）

ある組織では、マイクロサービスアーキテクチャをAzure Kubernetes Service（AKS）上に構築しています。このアプリケーションは複数のAzureリージョンに展開されており、グローバルにサービスを提供しています。

この環境では、各マイクロサービスがコンテナイメージとして管理されており、AKSクラスターがこれらのコンテナイメージを取得してデプロイします。そのため、コンテナイメージを効率的かつ安全に管理する仕組みが必要になります。

さらに、アプリケーションは複数リージョンに展開されているため、各リージョンからコンテナイメージを高速に取得できるようにする必要があります。また、セキュリティやアクセス管理、CI/CDパイプラインとの統合も重要です。

---

## 2. 要件整理

この問題では、AKS環境でコンテナイメージを管理するためのAzureサービスを選択する必要があります。

シナリオから読み取れる要件は次の通りです。

まず、コンテナイメージを複数リージョンに自動複製する必要があります。これにより、各リージョンのAKSクラスターがローカルに近いレジストリからイメージを取得できるようになります。

次に、コンテナイメージへのアクセスを細かく制御する必要があります。つまり、ユーザーやサービスごとに権限を設定できる仕組みが必要です。

さらに、コンテナイメージのセキュリティを確保する必要があります。脆弱性スキャンやセキュリティ統合が求められます。

また、CI/CDパイプラインで利用するために、コンテナイメージのバージョン管理やタグ管理が必要になります。

要件をまとめると次の通りです。

- コンテナイメージの管理  
- リージョン間の自動レプリケーション  
- Azure ADベースのアクセス制御  
- コンテナセキュリティ  
- イメージのバージョン管理  

---

## 3. 技術の基本概念

Azure Container Registry（ACR）は、Azureで提供されるマネージドコンテナレジストリサービスです。

ACRはDockerやOCI形式のコンテナイメージを保存・管理するために設計されています。AKSとの統合が非常に強く、コンテナベースのマイクロサービス環境で広く利用されています。

ACRの主な機能には次のようなものがあります。

- コンテナイメージの保存と管理  
- イメージタグによるバージョン管理  
- Azure Entra ID（旧Azure AD）による認証  
- ロールベースアクセス制御（RBAC）  
- セキュリティスキャン  

これらの機能により、安全かつ効率的にコンテナイメージを管理できます。

---

## 4. アーキテクチャまたは設計のポイント

複数リージョンでAKSを運用する場合、コンテナイメージの配布方法が重要になります。

Azure Container Registryでは「ジオレプリケーション」という機能が提供されています。この機能を利用すると、レジストリのコンテナイメージを複数のAzureリージョンに自動的に複製できます。

この構成では次のような仕組みになります。

1. コンテナイメージをACRへプッシュ  
2. ACRが複数リージョンへ自動複製  
3. 各リージョンのAKSがローカルレジストリから取得  

このアーキテクチャには次のメリットがあります。

- イメージ取得の高速化  
- ネットワーク遅延の削減  
- 可用性の向上  
- グローバル展開の簡素化  

また、Azure Entra IDと統合することで、AKSや開発チームのアクセス権限を細かく制御できます。

---

## 5. 設計判断（なぜこの構成になるか）

この問題では、コンテナイメージ管理のために複数の要件を満たす必要があります。

Azure Container Registryは、コンテナ管理専用サービスとして次の機能を提供します。

- コンテナイメージの保存  
- 自動ジオレプリケーション  
- Azure AD統合  
- セキュリティ機能  
- CI/CD統合  

特に、AKSとACRはネイティブ統合されているため、Kubernetes環境でのコンテナ管理が非常に簡単になります。

そのため、AKSを使用したマイクロサービスアーキテクチャではACRを利用するのがベストプラクティスです。

---

## 6. 他の選択肢が誤りな理由

Azure Blob Storageはデータ保存サービスであり、コンテナイメージ管理専用ではありません。ジオレプリケーションやタグ管理などのコンテナ専用機能が不足しています。

Azure App Serviceはアプリケーションホスティングサービスであり、コンテナイメージレジストリとして使用するものではありません。

Azure DevOps Artifactsはパッケージ管理サービスであり、NuGetやnpmなどのパッケージ管理が主目的です。コンテナレジストリとしての機能はACRほど充実していません。

---

## 7. 最終回答

A.  
Azure Active Directory統合およびジオレプリケーション対応の **Azure Container Registry**

---

## 8. まとめ

AKSベースのマイクロサービス環境では、コンテナイメージを効率的に管理するためのレジストリが必要になります。

Azure Container Registryは次の機能を提供します。

- コンテナイメージ管理  
- ジオレプリケーション  
- Azure Entra ID統合  
- セキュリティ機能  
- バージョン管理  

そのため、複数リージョンに展開されたAKS環境でコンテナイメージを管理する場合は **Azure Container Registry（ACR）** を使用するのが最適です。
