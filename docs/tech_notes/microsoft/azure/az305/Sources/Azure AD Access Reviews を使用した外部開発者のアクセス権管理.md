# Azure AD Access Reviews を使用した外部開発者のアクセス権管理

## 1. 背景（シナリオ）

企業は、外部企業が開発したカスタムアプリケーション「App1」を運用しています。このアプリケーションの開発者は外部企業に所属しており、アプリケーションコンポーネントに対してロールベースアクセス制御（RBAC）による権限が付与されています。

しかし、外部開発者のアクセス権は必要以上に長期間残るとセキュリティリスクになる可能性があります。そのため、開発者が現在も本当にアクセス権を必要としているのかを定期的に確認する仕組みが必要です。

さらに、この確認プロセスでは開発者の管理者に対して定期的なレポートを送信し、不要と判断された権限を自動的に削除する仕組みを導入する必要があります。

## 2. 要件整理

このシナリオでは、アクセス管理とガバナンスの観点から次のような要件が存在します。

まず、外部開発者のアクセス権を定期的に確認できる仕組みが必要です。これは組織のセキュリティポリシーやコンプライアンス要件を満たすために重要です。

さらに、アクセスの必要性を確認するレビュー結果を管理者が確認できるよう、定期的なレポートが必要になります。また、レビューの結果として不要と判断されたアクセス権は自動的に削除される必要があります。

この問題の要件は次のように整理できます。

・RBAC 権限の定期レビュー  
・外部ユーザーのアクセス管理  
・管理者への定期レポート送信  
・不要なアクセス権の自動削除  

## 3. 技術の基本概念

Azure ではアイデンティティとアクセス管理を提供するサービスとして Azure Active Directory（現在は Microsoft Entra ID）が利用されます。このサービスにはアクセスガバナンスを実現する機能が含まれています。

その中でも重要な機能の一つが Access Reviews です。この機能は、ユーザーが特定のアプリケーションやグループ、ロールにアクセスする必要があるかどうかを定期的に確認するための仕組みです。

Access Reviews では次のような機能を提供します。

まず、特定のユーザーやグループに対して定期的なアクセスレビューを設定できます。レビューは週次・月次などのスケジュールで自動的に実行されます。

次に、レビュー担当者（マネージャーやリソース所有者）に対してレビュー依頼が送信されます。担当者はそのユーザーが引き続きアクセス権を必要とするかどうかを判断できます。

さらに、レビュー結果に基づいて不要と判断されたアクセス権を自動的に削除する設定も可能です。これによりアクセス権管理の運用負荷を大幅に削減できます。

## 4. アーキテクチャまたは設計のポイント

このシナリオでは、外部開発者のアクセス権を定期的に確認する仕組みが必要です。そのため、アクセス権のライフサイクル管理を自動化できるサービスを利用することが重要になります。

Azure AD Access Reviews を利用することで、特定のアプリケーションやロールに対するアクセスを定期的にレビューする仕組みを構築できます。レビューの対象として外部ユーザーを含めることも可能です。

また、レビューのスケジュールを月次に設定することで、開発者の管理者へ定期的にレビュー依頼を送信できます。レビュー担当者が不要と判断したアクセス権は自動的に削除するよう設定できるため、管理作業を自動化できます。

この仕組みにより、アクセス権の監査と削除を自動化しながらセキュリティを維持することができます。

## 5. 設計判断（なぜこの構成になるか）

Azure AD Access Reviews は、アクセス権の定期確認と自動削除を目的としたアクセスガバナンス機能です。この機能はまさに今回の要件である「アクセスの必要性の確認」と「不要な権限の削除」に対応しています。

さらに Access Reviews ではレビューの結果を管理者に通知することができ、定期的なレビューサイクルを設定できます。これにより、外部開発者のアクセス権を継続的に管理できます。

また、レビューで承認されなかったユーザーのアクセス権を自動的に削除する設定が可能であるため、運用の自動化も実現できます。

このように、Access Reviews はアクセス管理・コンプライアンス・自動化のすべての要件を満たすサービスです。

## 6. 他の選択肢が誤りな理由

この問題には複数の Azure サービスが提示されていますが、それぞれ目的が異なります。

まず Azure Automation はスクリプトを自動実行するサービスですが、アクセスレビューや権限管理の仕組みをネイティブで提供するものではありません。そのため、このシナリオでは運用負荷が大きくなります。

次に Azure AD 条件付きアクセスは、ログイン時の条件（場所、デバイス、リスクなど）に基づいてアクセスを制御する仕組みです。これはアクセス権の定期レビューとは目的が異なります。

また Azure AD Privileged Identity Management（PIM）は特権ロール（管理者権限など）の管理を目的としたサービスです。今回のシナリオでは一般的な RBAC アクセスのレビューであり、PIM よりも Access Reviews の方が適しています。

## 7. 最終回答

A. Azure AD でアクセスレビューを作成する

## 8. まとめ

組織では外部ユーザーや開発者に対してアクセス権を付与することがありますが、不要な権限が残り続けるとセキュリティリスクになります。そのため、定期的にアクセス権を見直す仕組みが重要です。

Azure AD の Access Reviews を利用することで、ユーザーのアクセス権を定期的に確認し、不要な権限を自動的に削除することができます。さらに、レビュー結果を管理者へ通知することで、アクセスガバナンスを強化することができます。

この機能は外部ユーザーのアクセス管理やコンプライアンス要件の対応にも有効であり、Azure 環境におけるアクセス管理のベストプラクティスの一つです。
