# Azure AD Application Proxy と条件付きアクセスによるセキュアなアクセス制御

## 1. 背景（シナリオ）

組織がWebアプリケーション（App3）をAzureでデプロイする際には、セキュリティ要件とアクセス制御が重要です。アプリケーションはAzure AD認証を必要とし、アクセスは特定の信頼できるIP範囲からのみ可能で、企業ネットワーク内のユーザーは認証プロンプトなしでシームレスにアクセスできる必要があります。

また、企業ネットワーク外のユーザーに対しては、カスタムログインページにリダイレクトし、アクセスを制限する必要があります。

これらの要件を満たすために最適なソリューションを導入する必要があります。

## 2. 要件整理

シナリオからの要件を整理すると、以下の点が求められています。

1. **Azure AD 認証**: アプリケーションはAzure Active Directory（Azure AD）を通じて認証される必要がある。
2. **IP範囲によるアクセス制御**: アプリケーションは特定の信頼できるIP範囲からのみアクセス可能でなければならない。
3. **企業ネットワーク内ユーザーのシームレスアクセス**: 企業ネットワーク内からアプリケーションにアクセスする際に、認証の依頼を受けるべきではない。
4. **企業ネットワーク外ユーザーのカスタムログインページリダイレクト**: 企業ネットワーク外のユーザーは、カスタムログインページにリダイレクトされる必要がある。

## 3. 技術の基本概念

### Azure AD Application Proxy:
Azure AD Application Proxyは、企業ネットワーク内のアプリケーションを安全に公開するためのサービスです。これにより、オンプレミスのアプリケーションをAzure AD認証を使って管理し、リモートからでもセキュアにアクセスできるようにします。

- **条件付きアクセス**: Azure AD 条件付きアクセスは、ユーザーの位置やデバイス状態、その他の条件に基づいてアクセスを制御するポリシーを設定できます。これにより、企業ネットワーク内のユーザーにはシームレスにアクセスを許可し、外部ユーザーにはカスタムログインページにリダイレクトするなどのポリシーを適用できます。

### 必要な機能:
- **IP範囲の制限**: 条件付きアクセスポリシーを使用して、特定のIP範囲からのみアクセスを許可する設定が可能です。
- **シームレスなアクセス**: 企業ネットワーク内のユーザーが認証を求められることなくアクセスできるように、条件付きアクセスポリシーで企業IP範囲を定義し、企業内のユーザーには認証プロンプトなしでアクセスできるように設定できます。
- **カスタムログインページ**: 条件付きアクセスのカスタムポリシーを使用して、企業ネットワーク外からアクセスするユーザーを特定のカスタムログインページにリダイレクトできます。

## 4. アーキテクチャまたは設計のポイント

Azure AD Application Proxyと条件付きアクセスを組み合わせることで、次の要件を効率的に満たせます。

- **Azure AD 認証**: Azure AD Application Proxyは、Azure AD認証とシームレスに統合され、ユーザーはすべてAzure ADを通じて認証されます。
- **IP範囲の制限**: 条件付きアクセスポリシーを設定することで、特定のIP範囲（企業ネットワーク内など）からのみアクセスを許可できます。
- **シームレスアクセス**: 企業ネットワークからのアクセス時に、条件付きアクセスで企業内IP範囲を指定することにより、認証を求めずシームレスにアクセスできます。
- **外部ユーザー向けカスタムログインページ**: 条件付きアクセスにより、企業外のユーザーに対してカスタムログインページへのリダイレクトを行い、セキュアにログインを促すことができます。

## 5. 他の選択肢が誤りな理由

- **B. サインインリスクポリシー付きAzure AD ID保護**: ID保護はユーザーのサインインリスクを評価して対応しますが、特定のIP範囲に基づくアクセス制御や企業ネットワーク内のシームレスアクセス、外部ユーザーへのリダイレクト機能には対応していません。
  
- **C. IPフィルタリングによるAzure API管理**: Azure API ManagementはAPIの管理に使用されますが、Webアプリケーションのユーザー認証やIP範囲によるアクセス制限には直接対応していません。また、条件付きアクセスやカスタムログインページの機能は提供しません。
  
- **D. 認証ポリシー付きAzureフロントドア**: Azure Front Doorは、グローバルなアプリケーションアクセラレーションとロードバランシングを提供しますが、Azure AD認証との統合や、条件付きアクセス、カスタムログインページへのリダイレクトには対応していません。

## 6. 結論

**A. 条件付きアクセス付きAzure ADアプリケーションプロキシ**は、指定された要件を最も効率的に満たすソリューションです。これにより、特定のIP範囲からのアクセス制御、企業ネットワーク内のシームレスな認証、企業外ユーザーへのカスタムログインページへのリダイレクトが実現できます。

このアプローチは、セキュリティ要件を満たし、ユーザーエクスペリエンスを最適化しながら、Azure AD認証を安全かつ効率的に適用するための最適な選択肢です。
