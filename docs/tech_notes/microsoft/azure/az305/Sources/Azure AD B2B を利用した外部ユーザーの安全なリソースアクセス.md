# Azure AD B2B を利用した外部ユーザーの安全なリソースアクセス

## 1. 背景（シナリオ）

ある企業では Azure Active Directory（現在の Microsoft Entra ID）を使用してユーザーとアクセス権限を管理しています。このテナントには、自社の社員だけでなく、パートナー企業や外部組織のユーザーがゲストとして追加されています。

企業は外部組織のユーザーと安全にコラボレーションする必要がありますが、外部ユーザーに新しいアカウントを作成させたり、追加の認証システムを導入させたりすることは避けたいと考えています。つまり、外部ユーザーは自分の組織で使用している既存の資格情報をそのまま利用して企業のリソースへアクセスできる必要があります。

さらに、この仕組みは既存の Azure AD 環境に大きな変更を加えることなく導入できる必要があります。そのため、外部組織との安全な認証連携を実現する Azure のアイデンティティ管理機能を適切に選択することが重要になります。

## 2. 要件整理

このシナリオでは、外部ユーザーとのコラボレーション環境を安全に構築する必要があります。問題文から読み取れる要件を整理すると、次のようになります。

まず、外部組織のユーザーが企業のリソースにアクセスできる必要があります。これはパートナー企業や協力会社との共同作業などを想定しています。

次に、外部ユーザーは自分の組織の資格情報を使用してログインできる必要があります。つまり、新しいユーザーアカウントやパスワードを作成する必要がない仕組みが求められています。

さらに、既存の Azure AD 環境に追加の構成変更や複雑なインフラ構築を行わずに実装できる必要があります。

これらの要件をまとめると次の通りです。

- 外部組織のユーザーが企業リソースへアクセスできる  
- 外部ユーザーは自分の組織の資格情報を使用する  
- Azure AD と統合されたアクセス管理が必要  
- 環境に大きな変更を加えずに実装できる  

## 3. 技術の基本概念

Azure Active Directory（Microsoft Entra ID）には、外部ユーザーと安全にコラボレーションするための機能として **Azure AD B2B（Business-to-Business）** があります。

Azure AD B2B は、外部組織のユーザーを「ゲストユーザー」として Azure AD テナントに招待する仕組みです。このユーザーは企業のディレクトリに登録されますが、認証は自分の所属組織の ID プロバイダーで行われます。

つまり、外部ユーザーは自分の組織のアカウントをそのまま利用してログインできるため、新しいアカウントを作成する必要がありません。これにより、ユーザー体験を損なうことなく安全なコラボレーションが可能になります。

また、Azure AD B2B では次のような機能も利用できます。

- ゲストユーザー管理  
- 条件付きアクセスとの統合  
- 多要素認証（MFA）の適用  
- アクセスログおよび監査  

## 4. アーキテクチャまたは設計のポイント

このシナリオでは、外部ユーザーを Azure AD テナントにゲストとして招待し、Azure AD B2B を使用してアクセス管理を行う設計が適しています。

このアーキテクチャでは、まず外部組織のユーザーを Azure AD に招待します。ユーザーは招待メールを受け取り、既存の組織アカウントでサインインします。

次に、Azure AD 内でそのユーザーをゲストユーザーとして管理します。これにより、どのリソースにアクセスできるかを細かく制御できます。

さらに、必要に応じて条件付きアクセスや MFA を適用することで、セキュリティポリシーを統一できます。

この構成により、外部ユーザーは自分の組織の資格情報を使用しながら、安全に企業リソースへアクセスできます。

## 5. 設計判断（なぜこの構成になるか）

この問題の正解は **A. Azure AD B2B（Business-to-Business）** です。

Azure AD B2B は、外部ユーザーが自分の組織の資格情報を使用してリソースへアクセスできるように設計された機能です。外部ユーザーをゲストとして招待することで、企業の Azure AD テナントに安全にアクセスできます。

また、この機能は既存の Azure AD 環境に追加のインフラ構築を必要としません。そのため、環境への変更を最小限に抑えながら外部ユーザーとのコラボレーションを実現できます。

さらに、Azure AD の認証・認可機能と統合されているため、アクセス制御やセキュリティポリシーを一元管理できます。

## 6. 他の選択肢が誤りな理由

Azure AD Identity Protection はユーザーのログインリスクを検出し、リスクベースのアクセス制御を行うセキュリティサービスです。しかし、外部ユーザーが自分の組織の資格情報を使用してアクセスする機能を提供するものではありません。

Azure AD 条件付きアクセスは、ユーザーの場所やデバイス状態などの条件に基づいてアクセスを制御する仕組みです。セキュリティポリシーを適用するための機能ですが、外部ユーザーを招待する機能ではありません。

Azure AD Join はデバイスを Azure AD に参加させるための機能です。これはユーザーアクセス管理ではなく、デバイス管理を目的とした機能です。

## 7. 最終回答

A. Azure AD B2B（Business-to-Business）

## 8. まとめ

この問題は、Azure Active Directory における外部ユーザーコラボレーション機能を理解しているかを確認する問題です。企業がパートナー企業や外部ユーザーと安全に共同作業を行うためには、外部アイデンティティを適切に管理する仕組みが必要になります。

Azure AD B2B は、外部ユーザーをゲストとして招待し、自分の組織の資格情報を使用してリソースへアクセスできるようにする機能です。これにより、ユーザー体験を損なうことなく安全なコラボレーションを実現できます。

したがって、外部組織のユーザーが自分の資格情報を使用して企業リソースへアクセスできるようにする最適なソリューションは Azure AD B2B です。
