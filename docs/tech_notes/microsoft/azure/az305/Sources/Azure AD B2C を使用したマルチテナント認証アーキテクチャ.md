# Azure AD B2C を使用したマルチテナント認証アーキテクチャ

## 1. 背景（シナリオ）

ある企業では、複数の組織のユーザーが利用する **マルチテナントアプリケーション** を開発しています。このアプリケーションは、従業員が休暇申請（Vacation Request）を提出するためのサービスとして設計されています。

現在はカスタム認証システムを使用していますが、セキュリティやユーザー体験を改善するために **Azure Active Directory（Azure AD）を利用したシングルサインオン（SSO）** を導入する計画があります。

しかし、このアプリケーションは複数の企業が利用するため、次の課題があります。

- 異なる組織のユーザー認証
- 組織ごとのデータ隔離
- シームレスなログイン体験

つまり、**複数テナント環境で安全に認証を管理できる仕組み**が必要になります。

---

## 2. 要件整理

この問題の要件を整理すると、次の3つが重要です。

まず、ユーザー認証です。

- シングルサインオン（SSO）
- Azure ADベース認証

次に、マルチテナント対応です。

- 複数の組織ユーザー
- 組織ごとのテナント管理

さらに、セキュリティ要件です。

- テナント隔離
- ユーザーアクセス制御

このような要件を満たすためには、**外部ユーザー管理に特化したID管理サービス**が必要になります。

---

## 3. 技術の基本概念

Azure では外部ユーザー向けの認証サービスとして **Azure AD B2C（Azure Active Directory Business to Consumer）** が提供されています。

Azure AD B2C は、顧客向けアプリケーションの認証管理を目的としたサービスです。

主な特徴は次の通りです。

まず、外部ユーザー認証です。

- 顧客ユーザー管理
- マルチテナント対応

次に、複数のIDプロバイダー対応です。

- Azure AD
- Google
- Facebook
- Microsoftアカウント

さらに、カスタマイズ可能な認証フローです。

- サインイン
- サインアップ
- パスワードリセット
- プロファイル編集

これらは **User Flows（ユーザーフロー）** によって構成できます。

---

## 4. アーキテクチャまたは設計のポイント

Azure AD B2C では **ユーザーフロー（User Flow）** を使用して認証プロセスを定義します。

ユーザーフローには次の種類があります。

まず、ユーザー登録です。

- サインアップフロー

次に、ユーザー認証です。

- サインインフロー

さらに、アカウント管理です。

- パスワードリセット
- プロファイル更新

この仕組みにより、認証プロセスを簡単に管理できます。

また、Azure AD B2C では **クレームベース認証**を利用します。

クレームには次の情報を含めることができます。

- ユーザーID
- 組織ID
- ロール
- テナント識別子

これにより、アプリケーション側で **テナントごとのアクセス制御** を実装できます。

---

## 5. 設計判断（なぜこの構成になるか）

この問題では **マルチテナント環境でのユーザー認証**が重要です。

Azure AD B2C を使用することで次のメリットがあります。

まず、マルチテナント対応です。

- 異なる組織のユーザー管理
- テナント分離

次に、ユーザー体験です。

- SSO対応
- シンプルなログイン

さらに、セキュリティです。

- MFA対応
- 条件付きアクセス

このため、マルチテナント SaaS アプリケーションの認証基盤として非常に適しています。

---

## 6. 他の選択肢が誤りな理由

まず **OAuth 2.0 Authorization Code Flow（PKCE）** です。

これは認証プロトコルであり、マルチテナント管理機能を提供するものではありません。

次に **SAML 2.0 マルチテナント構成**です。

SAMLはSSOを提供できますが、モダンアプリケーションではOpenID ConnectベースのB2Cがより管理しやすいです。

最後に **OpenID Connect + カスタムIDプロバイダー**です。

これは実装可能ですが、ID管理を自分で構築する必要があり運用負荷が高くなります。

Azure AD B2Cはこれらを **マネージドサービスとして提供**します。

## 8. まとめ

Azureの認証サービスは用途によって使い分けます。

代表的なサービスは次の通りです。

まず、社内ユーザー認証です。

- **Azure AD**
  → 企業ユーザー管理

次に、外部ユーザー認証です。

- **Azure AD B2C**
  → 顧客ユーザー管理

さらに、B2Bコラボレーションです。

- **Azure AD B2B**
  → パートナー共有

そのため、**複数組織のユーザーを対象としたマルチテナントアプリケーションでは Azure AD B2C が最適な認証ソリューション**になります。
