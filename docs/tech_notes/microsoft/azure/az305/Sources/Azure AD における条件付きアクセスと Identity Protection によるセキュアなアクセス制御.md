# Azure AD における条件付きアクセスと Identity Protection によるセキュアなアクセス制御

## 1. 背景（シナリオ）

ある組織では、新しいクラウドベースのアプリケーションを導入する予定です。このアプリケーションは Azure Active Directory（現在は Microsoft Entra ID）と統合され、ユーザーはSAMLベースのシングルサインオン（SSO）を使用してアクセスします。

この環境では、セキュリティを強化するために次の要件が定義されています。

まず、すべてのユーザーがアプリケーションへアクセスする際に **多要素認証（MFA）** を必須にする必要があります。

さらに、アクセスは **ユーザーの役割** に基づいて制御する必要があります。

また、サインインの試行に伴う **リスクレベル** を評価し、リスクが高い場合には追加のセキュリティ対策を適用する必要があります。

これらの要件を満たすためには、Azure ADのセキュリティ機能を組み合わせて利用する必要があります。

## 2. 要件整理

この問題の重要な要件を整理すると次の通りです。

- SAMLベースのSSO
- すべてのユーザーにMFAを適用
- ユーザーの役割に基づくアクセス制御
- サインインリスクに基づくアクセス制御
- セキュリティポリシーの自動適用

これらの要件を満たすには **アクセス制御ポリシー** と **リスク検出機能** の両方が必要になります。

## 3. 技術の基本概念

Azure ADでは、アクセス制御とセキュリティ強化のために複数の機能が提供されています。

まず **Azure AD Conditional Access（条件付きアクセス）** があります。

条件付きアクセスは、ユーザーやデバイスの条件に基づいてアクセスを制御する仕組みです。

例えば次の条件を設定できます。

- ユーザーまたはグループ
- アプリケーション
- デバイス状態
- サインインリスク
- ロケーション

そして、条件が満たされた場合に次のようなアクションを適用できます。

- MFAを要求
- アクセスを許可
- アクセスをブロック

次に **Azure AD Identity Protection** があります。

Identity Protectionは、ユーザーのサインイン行動を分析し、機械学習を利用してリスクを検出するサービスです。

例えば次のようなリスクを検出できます。

- 異常な場所からのログイン
- 不審なIPアドレス
- 不審なサインインパターン

これらのリスク情報を条件付きアクセスポリシーと組み合わせることで、リスクベースのアクセス制御が可能になります。

## 4. アーキテクチャまたは設計のポイント

このシナリオでは、次の2つのセキュリティ要素が重要になります。

まず、すべてのユーザーにMFAを強制することです。これは条件付きアクセスを使用して実装できます。

条件付きアクセスポリシーを作成し、特定のアプリケーションへアクセスするユーザーに対してMFAを必須にすることができます。

次に、サインインリスクに基づくアクセス制御です。

Identity Protectionはサインインのリスクレベルを分析し、その結果を条件付きアクセスに提供します。

その結果、次のような制御が可能になります。

例えば、リスクの高いサインインの場合にMFAを強制したり、アクセスを完全にブロックしたりすることができます。

## 5. 設計判断（なぜこの構成になるか）

この問題では次の2つの要件を満たす必要があります。

- MFAを強制する
- サインインリスクに基づいてアクセス制御する

MFAの強制やロールベースのアクセス制御は **Conditional Access** によって実装できます。

一方で、サインインリスクの検出は **Identity Protection** が担当します。

Identity Protectionはリスクを検出し、その結果を条件付きアクセスポリシーに渡すことで、リスクベースのアクセス制御を実現します。

この2つを組み合わせることで、セキュアなアクセス管理が可能になります。

## 6. 他の選択肢が誤りな理由

### B. Azure AD Identity Governance

Identity Governanceは、アクセスレビューやライフサイクル管理などを提供するサービスです。

ユーザーアクセス管理には有効ですが、この問題の要件であるMFA強制やリスクベースアクセス制御には直接関係しません。

### C. Azure AD Application Proxy

Application ProxyはオンプレミスアプリケーションをAzure AD経由で公開するためのサービスです。

SSOを提供することはできますが、MFAやリスクベースアクセス制御を実装する機能ではありません。

## 7. 最終回答

この要件を満たすために設定する必要がある機能は次の2つです。

**Azure AD Conditional Access**  
**Azure AD Identity Protection**

したがって **正解は A と D** です。

## 8. まとめ

Azure ADでは、セキュアなアクセス制御を実現するために複数の機能を組み合わせて利用します。

条件付きアクセスは、ユーザーやデバイスの条件に基づいてアクセスを制御する仕組みです。

Identity Protectionは、ユーザーのサインイン行動を分析し、リスクを検出します。

この2つを組み合わせることで次のようなセキュリティが実現できます。

- MFAの強制
- リスクベース認証
- 不審なサインインの検出
- セキュアなSSO環境

そのため、Azure ADの高度なセキュリティ構成では **Conditional Access + Identity Protection の組み合わせがベストプラクティス**となります。
