# Azure AD を用いた SAML ベース LOB アプリケーションへの安全なアクセス設計

## 1. 背景（シナリオ）
組織は、SAML ベースのシングルサインオン(SSO)をサポートするカスタムビジネスライン(LOB)アプリケーションを開発しました。ユーザーは Azure Active Directory (Azure AD) の認証情報を使ってアクセスします。さらに、信頼できない場所からのログインには多要素認証(MFA)を適用する必要があります。

## 2. 要件整理
- Azure AD 認証情報で SSO を提供
- 信頼できない場所からのアクセスに対して MFA を適用
- ユーザー管理とアクセス制御を集中管理
- 条件付きアクセスポリシーで動的制御

## 3. 技術の基本概念
- **Azure AD エンタープライズアプリケーション**
  - カスタム LOB アプリケーションを登録可能
  - SAML ベース SSO 構成が可能
  - ユーザー割り当て、グループ管理、アクセス制御を集中管理
  - 条件付きアクセスポリシーと連携可能
- **条件付きアクセスポリシー**
  - ユーザーの場所、デバイス、アプリケーションに応じてアクセス制御を適用
  - 信頼できない場所からのアクセスには MFA を強制
  - アプリケーションやユーザー単位で柔軟にルールを設定可能

## 4. アーキテクチャまたは設計のポイント
- LOB アプリケーションを Azure AD にエンタープライズアプリケーションとして登録
- SAML SSO を設定し、Azure AD 認証を経由
- 条件付きアクセスポリシーを作成し、場所に基づく MFA 要件を適用
- これにより、ユーザーはシームレスなサインオンを体験しつつ、セキュリティは保証される

## 5. 設計判断（なぜこの構成になるか）
- Azure AD エンタープライズアプリケーションで SAML SSO を簡単に提供
- 条件付きアクセスポリシーで MFA を動的に強制
- 両機能の組み合わせにより、安全かつ効率的なアクセス管理が可能

## 6. 他の選択肢が誤りな理由
- **PIM (特権型アイデンティティ管理)**: 管理者権限やジャストインタイムアクセス向けで、一般ユーザーの SSO や MFA 強制には不適
- **Azure AD Identity Protection**: リスクベース条件付きアクセスは補完的機能であり、場所に基づく MFA 強制を直接実装するものではない

## 7. 最終回答
**A) Azure AD エンタープライズアプリケーション**  
**B) 条件付きアクセスポリシー**

## 8. まとめ
カスタム LOB アプリケーションで SAML SSO を実現し、信頼できない場所からのアクセスに対して MFA を適用するには、Azure AD エンタープライズアプリケーションと条件付きアクセスポリシーを組み合わせて使用することが最適です。この組み合わせにより、セキュアかつユーザーに優しいアクセス体験を提供できます。
