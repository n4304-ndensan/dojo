# Azure AD パススルー認証によるハイブリッド ID 認証

## 1. 背景（シナリオ）

ある企業では、オンプレミス環境とクラウド環境の両方を利用するハイブリッド IT 環境を運用しています。社内のユーザーの多くは既存のオンプレミス Active Directory を使用して認証を行っていますが、Azure 上のリソースにもアクセスできるようにする必要があります。

この環境では、ユーザーが既存のオンプレミス認証情報をそのまま利用して Azure のサービスにサインインできることが重要です。さらに、企業のセキュリティポリシーを維持するため、可能な限り安全な認証方法を採用する必要があります。

そのため、オンプレミスの認証情報を Azure AD（Microsoft Entra ID）で利用できるようにする認証方式を選択する必要があります。

## 2. 要件整理

このシナリオでは、ハイブリッド ID 管理の観点から重要な要件があります。

まず、ユーザーは既存のオンプレミス Active Directory の資格情報を使用して Azure リソースにサインインできる必要があります。これにより、ユーザーは新しいアカウントやパスワードを覚える必要がありません。

次に、認証プロセスは可能な限り安全である必要があります。特に、ユーザーパスワードをクラウドに保存しない方法が望ましい場合があります。

さらに、ユーザー体験を向上させるため、シングルサインオン（SSO）によるシームレスな認証も重要です。

この問題の要件は次のように整理できます。

・オンプレミス認証情報を使用した Azure サインイン  
・高いセキュリティ  
・ハイブリッド ID 管理  
・シングルサインオンの実現  

## 3. 技術の基本概念

Azure AD（Microsoft Entra ID）では、オンプレミス Active Directory と連携するための複数の認証方式が提供されています。

代表的な方法として、以下のような認証方式があります。

まず、**パスワードハッシュ同期（Password Hash Synchronization）** は、オンプレミス AD のパスワードハッシュを Azure AD に同期する方式です。

次に、**パススルー認証（Pass-through Authentication）** は、ユーザーが Azure にサインインするときに、認証をオンプレミス AD に転送して検証する方式です。

また、**Active Directory Federation Services（AD FS）** を使用するフェデレーション認証もありますが、追加のインフラ管理が必要になります。

## 4. アーキテクチャまたは設計のポイント

パススルー認証では、ユーザーが Azure AD にサインインしようとすると、その認証要求はオンプレミスの Active Directory に送信されます。オンプレミスの AD が認証を行い、その結果が Azure AD に返されます。

この仕組みにより、ユーザーのパスワードは Azure AD に保存されることなく認証が行われます。つまり、企業の認証ポリシーをオンプレミス環境で維持したままクラウドサービスを利用できます。

さらに、シームレス SSO と組み合わせることで、企業ネットワーク内のユーザーは資格情報を再入力することなく Azure サービスにアクセスできるようになります。

## 5. 設計判断（なぜこの構成になるか）

Azure AD パススルー認証は、オンプレミス認証情報を使用しながらクラウドサービスへ安全にアクセスするための方法です。

この方式では、ユーザーのパスワードが Azure に保存されないため、セキュリティポリシーを維持しやすいという利点があります。

また、シームレス SSO を組み合わせることでユーザーのログイン体験も改善されます。

そのため、オンプレミスとクラウドが混在する環境では非常に一般的なハイブリッド認証方式となっています。

## 6. 他の選択肢が誤りな理由

Azure AD Join とシームレス SSO はデバイス管理やログイン体験の改善に役立ちますが、オンプレミス認証情報の検証方法としては十分ではありません。

Azure AD Identity Protection はリスクベースのアクセス制御を提供するセキュリティ機能であり、認証方式そのものではありません。

Windows Hello for Business はパスワードレス認証を提供する技術ですが、オンプレミス認証情報を Azure で直接使用する要件を満たすものではありません。

## 7. 最終回答

B. Azure AD パススルー認証を設定する

## 8. まとめ

ハイブリッド環境では、オンプレミス Active Directory と Azure AD を統合する認証方式が必要になります。

Azure AD パススルー認証は、オンプレミスの認証情報を使用しながら Azure サービスへ安全にアクセスできる方法です。

この方式は、パスワードをクラウドに保存することなく認証を行えるため、高いセキュリティと優れたユーザー体験を同時に実現できます。
