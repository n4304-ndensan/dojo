# Azure API Management を利用した Logic Apps の外部公開

## 1. 背景（シナリオ）

ある組織では、複数の Azure Logic Apps を使用したソリューションを構築しています。これらの Logic Apps には HTTP トリガーが含まれており、オンプレミスの Web サービスへアクセスするためのインテグレーション処理を提供しています。

このシステムでは、外部の開発者がこれらの Logic Apps を利用できるようにする必要があります。ただし、既存の Logic Apps の構成やコードを変更することなく公開することが求められています。

さらに、外部開発者は自分たちの既存の ID プロバイダーを使用して認証する必要があります。また、外部ユーザーからのリクエストは内部ユーザーとは異なり、リクエスト数を制限するレート制限を適用する必要があります。

これらの要件を満たすためには、Logic Apps を API として公開しつつ、認証管理、アクセス制御、レート制限などを提供できるサービスを利用する必要があります。

## 2. 要件整理

このシナリオの要件を整理すると、次のようになります。

このアーキテクチャでは、Logic Apps を直接変更せずに外部開発者に公開する必要があります。そのため、API 管理レイヤーを追加する必要があります。

- Logic Apps を外部開発者に公開する  
- Logic Apps 自体は変更しない  
- 外部開発者は既存の ID プロバイダーを利用する  
- 外部ユーザーのリクエストにレート制限を適用する  
- API のセキュリティとアクセス管理を実施する  

このような要件から、API ゲートウェイ機能を提供するサービスが必要であることが分かります。

## 3. 技術の基本概念

Azure API Management（APIM）は、API の公開、管理、セキュリティ制御を提供する API ゲートウェイサービスです。バックエンドのサービスを変更することなく API を公開できる点が特徴です。

APIM には次のような主要機能があります。

まず、API ゲートウェイ機能です。APIM はバックエンドサービス（Logic Apps など）の前に配置され、外部ユーザーからのリクエストを受け取ります。

次に、認証および認可の管理です。OAuth 2.0 や OpenID Connect などの標準認証方式を利用できるため、既存の ID プロバイダーと統合できます。

さらに、レート制限（Rate Limiting）やスロットリング機能を提供しています。これにより、特定のユーザーやアプリケーションに対して API 呼び出し数を制限することができます。

また、APIM には開発者ポータルがあり、外部開発者が API ドキュメントを確認したり、テストを実行することも可能です。

## 4. アーキテクチャまたは設計のポイント

このシナリオでは、Azure API Management を Logic Apps の前に配置するアーキテクチャが最適です。

まず、Logic Apps の HTTP エンドポイントをバックエンド API として APIM に登録します。これにより、Logic Apps を変更することなく API として公開できます。

次に、APIM のポリシー機能を使用して、外部開発者のリクエストに対するレート制限を設定します。例えば、一定時間内に許可される API 呼び出し回数を制限することができます。

さらに、APIM の認証機能を利用して外部の ID プロバイダーと統合することで、開発者は既存の認証基盤を使用して API にアクセスできるようになります。

このように APIM を利用することで、セキュリティ、アクセス制御、トラフィック管理を一元的に管理できます。

## 5. 設計判断（なぜこの構成になるか）

この問題では「Logic Apps を変更せずに外部開発者へ公開する」という要件が重要です。

Azure API Management は API ゲートウェイとして機能するため、バックエンドサービスを変更することなく API を公開できます。

さらに、APIM では次の機能を提供します。

- 外部 ID プロバイダーとの認証統合  
- レート制限およびスロットリング  
- API セキュリティポリシー  
- API 使用状況の監視  

そのため、Logic Apps を直接公開するのではなく、API Management を介して公開することで、セキュリティと管理性を大幅に向上させることができます。

## 6. 他の選択肢が誤りな理由

B. Azure AD B2B  

Azure AD B2B は外部ユーザーを組織のディレクトリへ招待するためのサービスです。API 管理やレート制限機能は提供されません。

C. Azure AD Application Proxy  

Application Proxy はオンプレミスアプリケーションを安全に公開するためのサービスですが、API ゲートウェイ機能やレート制限機能は提供しません。

D. Azure Front Door  

Azure Front Door はグローバルロードバランサーおよび CDN 機能を提供するサービスです。API 管理や開発者ポータル、レート制限などの API 管理機能は提供されません。

## 7. 最終回答

A. Azure API Management

## 8. まとめ

Azure Logic Apps を外部開発者に公開する場合、API 管理レイヤーを導入することでセキュリティと管理性を向上させることができます。

Azure API Management は API ゲートウェイとして機能し、バックエンドサービスを変更することなく API を公開できます。また、認証統合、レート制限、アクセス制御などの機能を提供します。

そのため、外部開発者に Logic Apps を安全に公開する最適なソリューションは Azure API Management となります。
