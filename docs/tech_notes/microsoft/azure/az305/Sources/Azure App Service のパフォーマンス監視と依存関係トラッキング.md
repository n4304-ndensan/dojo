# Azure App Service のパフォーマンス監視と依存関係トラッキング

## 1. 背景（シナリオ）

ある企業では、Azure App Service 上で Web アプリケーションを運用しています。このアプリケーションは複数のコンポーネントで構成されており、外部 API やデータベースなどの依存関係を持っています。

運用チームは、アプリケーションのパフォーマンスを継続的に監視し、問題が発生した際には迅速に原因を特定できるようにしたいと考えています。特に、アプリケーションのレスポンスが遅くなった場合に、どのコンポーネントがボトルネックになっているのかを分析できる仕組みが必要です。

さらに、アプリケーションは既に本番環境で稼働しているため、監視機能を導入する際には **アプリケーションコードの変更を必要としない方法**が求められています。コード変更が必要な場合は、再ビルドや再デプロイが必要になるため、運用負荷が増加してしまいます。

このような条件を満たしながら、アプリケーションのパフォーマンス分析と依存関係の追跡を行える監視ソリューションを選択する必要があります。

---

## 2. 要件整理

この問題では、アプリケーション監視に関するいくつかの重要な要件があります。それぞれの条件を整理することで、適切な Azure サービスを選択することができます。

まず、監視対象に関する要件があります。対象は Azure 仮想マシンではなく **Azure App Service 上のアプリケーション**です。そのため、インフラレベルではなくアプリケーションレベルの監視が必要になります。

次に、分析内容に関する要件があります。単純なメトリック監視ではなく、アプリケーションの内部動作を分析する必要があります。

この分析には、次のような情報が必要になります。

- アプリケーションの応答時間
- 各コンポーネント間の依存関係
- パフォーマンスのボトルネック

さらに、導入方法に関する要件があります。

- **アプリケーションコードの変更を必要としない**

これらの条件を満たすサービスとして、Azure のアプリケーションパフォーマンス監視（APM）サービスを使用する必要があります。

---

## 3. 技術の基本概念

### Application Performance Monitoring（APM）

APM（Application Performance Monitoring）は、アプリケーションのパフォーマンスや依存関係を監視し、問題の原因を特定するための仕組みです。APM ツールを使用すると、アプリケーションの内部処理や外部サービスとの通信を可視化することができます。

APM の主な目的は、アプリケーションのパフォーマンス問題を迅速に特定することです。そのためには、単純な CPU 使用率やメモリ使用率ではなく、アプリケーションレベルのテレメトリを収集する必要があります。

---

[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Application Insights]]
### Azure Application Insights

[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Azure Monitor]]
Azure Application Insights は Azure Monitor の一部であり、アプリケーションのパフォーマンスと使用状況を監視するためのサービスです。このサービスは Web アプリケーションや API、バックエンドサービスなどの監視に広く使用されています。

Application Insights の重要な機能を理解するためには、どのようなデータを収集できるのかを整理する必要があります。

Application Insights は次のようなテレメトリを収集できます。

- リクエストの応答時間
- 失敗したリクエスト
- 依存関係の呼び出し（データベース、HTTP API など）
- 例外情報
- パフォーマンスメトリクス

これにより、アプリケーションのパフォーマンスを詳細に分析することができます。

---

## 4. アーキテクチャまたは設計のポイント

Azure App Service を監視する場合、Application Insights を App Service に統合することでアプリケーションのテレメトリを収集できます。Azure ポータルから Application Insights を有効化することで、アプリケーションの監視を簡単に開始できます。

この統合では、アプリケーションのリクエスト処理、外部サービスへの依存関係、データベース呼び出しなどが自動的に記録されます。これにより、アプリケーションの処理フローを可視化することができます。

例えば、あるユーザーリクエストが Web API を呼び出し、その後データベースにアクセスする場合、それぞれの処理時間が記録されます。このデータを分析することで、どの処理がパフォーマンスのボトルネックになっているかを特定できます。

さらに、Application Insights では **Application Map** と呼ばれる機能があり、アプリケーションコンポーネント間の依存関係を視覚的に表示することができます。

---

## 5. 設計判断（なぜこの構成になるか）

この問題の重要なポイントは **コード変更なしでアプリケーション監視を行うこと**です。

Application Insights は Azure App Service とネイティブに統合されており、ポータルから有効化するだけで監視機能を追加することができます。これにより、既存のアプリケーションコードを変更することなく、パフォーマンス分析を実施できます。

また、Application Insights はアプリケーションレベルの監視を提供するため、依存関係トラッキングやリクエスト分析などの機能を利用できます。

これらの機能は、アプリケーションのパフォーマンス問題を特定するために非常に重要です。

---

## 6. 他の選択肢が誤りな理由

### Azure Monitor for VMs

Azure Monitor for VMs は仮想マシンの監視を目的としたサービスです。CPU 使用率やメモリ使用率などのインフラメトリクスは取得できますが、アプリケーションの依存関係やパフォーマンス分析は行えません。

### Azure Network Watcher

Network Watcher はネットワークレベルの監視やトラブルシューティングを行うサービスです。接続性の確認やパケットキャプチャなどは可能ですが、アプリケーションのパフォーマンス分析には使用できません。

### Azure Service Health

Service Health は Azure プラットフォームの障害やメンテナンス情報を通知するサービスです。アプリケーションのパフォーマンス監視や依存関係分析には使用できません。

---

## 7. 最終回答

B. Azure Application Insights
