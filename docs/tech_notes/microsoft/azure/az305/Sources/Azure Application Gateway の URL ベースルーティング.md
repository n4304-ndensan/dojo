# Azure Application Gateway の URL ベースルーティング

## 1. 背景（シナリオ）

AzureでWebアプリケーションを運用する場合、複数のバックエンドサーバー（仮想マシン）にトラフィックを分散する必要があります。特に、マイクロサービスアーキテクチャや複数機能を持つWebアプリケーションでは、URLパスによって処理するサービスが異なることがあります。

例えば、次のような構成が考えられます。

・`/api` へのアクセスはAPIサーバーに送る  
・`/home` へのアクセスはWebフロントエンドサーバーに送る  

このような要件では、単純なロードバランサーではなく **HTTPレベルでルーティングできる仕組み**が必要になります。

この問題では、Azure Application Gateway を使用して **URLパスに基づいてトラフィックを異なる仮想マシンへルーティングする方法**が問われています。

## 2. 要件整理

問題文のシナリオを整理すると、以下の条件があります。

まず、仮想マシン上でWebアプリケーションが実行されています。そして、異なるURLパスに基づいてトラフィックを処理する必要があります。

このシナリオでは次の要件が存在します。

・Azure Application Gateway を使用する  
・複数の仮想マシンがバックエンドとして存在する  
・URLパスによってトラフィックを振り分ける必要がある  
・HTTPレベルでトラフィックをルーティングする  

つまり、単純なロードバランシングではなく **Layer7（HTTPレベル）でのルーティング**が必要になります。

## 3. 技術の基本概念

Azure Application Gateway は、Azureの **Layer7ロードバランサー（HTTP/HTTPSロードバランサー）**です。

通常のAzure Load BalancerはLayer4（TCP/UDP）で動作しますが、Application GatewayはHTTPリクエストの内容を理解してルーティングできます。

Application Gateway の主な機能には次のようなものがあります。

・URLベースのルーティング  
・SSLオフロード  
・Cookieベースセッションアフィニティ  
・Web Application Firewall（WAF）  
・ヘルスプローブ  

これらの機能により、Webアプリケーションのトラフィックを高度に制御することができます。

## 4. アーキテクチャまたは設計のポイント

この問題で重要になるのは **URLベースのルーティング（Path-based routing）**です。これは、HTTPリクエストのURLパスに基づいてバックエンドを選択する機能です。

URLベースルーティングを使用すると、Application Gateway は受信リクエストを解析し、指定されたパスルールに従ってトラフィックを適切なバックエンドプールに送信します。

典型的な設定は次のようになります。

・`/api/*` → APIサーバーのバックエンドプール  
・`/home/*` → Webサーバーのバックエンドプール  
・`/images/*` → 静的コンテンツサーバー  

この仕組みにより、同じドメインでもURLによって異なるサービスへトラフィックを分散できます。

## 5. 設計判断（なぜこの構成になるか）

この問題の要件は「URLパスに基づいてトラフィックをルーティングすること」です。

そのため、次の機能が必要になります。

・HTTPリクエストを解析できる  
・URLパスを確認できる  
・パスごとにバックエンドを指定できる  
・複数のバックエンドプールを使用できる  

これらの条件を満たす Azure Application Gateway の機能が **URLベースのルーティング（Path-based routing）**です。

この機能により、URLのパスごとに異なるバックエンド仮想マシンへトラフィックを送ることができます。

## 6. 他の選択肢が誤りな理由

この問題では、Application Gatewayの他の機能も選択肢として提示されていますが、それぞれ役割が異なります。

まず SSLオフロードです。SSLオフロードは HTTPS通信の復号処理をApplication Gatewayで実行する機能です。これによりバックエンドサーバーの負荷を軽減できますが、URLパスによるルーティングには関係ありません。

次にヘルスプローブです。ヘルスプローブはバックエンドサーバーの状態を監視し、正常なサーバーのみにトラフィックを送るための機能です。これは可用性のための機能であり、URLルーティングとは別の役割です。

最後にWeb Application Firewall（WAF）です。WAFはSQLインジェクションやクロスサイトスクリプティングなどのWeb攻撃からアプリケーションを保護するセキュリティ機能です。トラフィックの振り分け機能ではありません。

## 7. 最終回答

A. URLベースのルーティング

## 8. まとめ

Azure Application Gateway は Layer7 ロードバランサーであり、HTTPリクエストの内容に基づいたトラフィック制御が可能です。

その中でも **URLベースのルーティング（Path-based routing）**は、URLパスに応じて異なるバックエンドサービスへトラフィックを送るための機能です。これにより、同じドメインでも `/api`、`/home`、`/images` などのパスごとに異なる仮想マシンへリクエストを振り分けることができます。

そのため、URLパスに基づいてトラフィックをルーティングする必要がある場合には **URLベースのルーティング**を設定するのが最適な方法です。
