# Azure Automation を利用した VM 自動起動・停止と監視

## 1. 背景（シナリオ）

ある組織では、Azure 仮想マシン（VM）の使用コストを削減するために、より高度な自動化戦略を導入することを検討しています。現在、仮想マシンは常時稼働しているため、営業時間外でもコンピュートコストが発生してしまっています。

そのため、組織では次のような自動化を実装する必要があります。まず、営業時間外には仮想マシンを自動的にシャットダウンし、営業開始前には自動的に再起動する仕組みを構築する必要があります。さらに、スケジュールは簡単に変更できる必要があります。

加えて、この自動化ソリューションは単なるスケジュール実行だけではなく、仮想マシンの状態を監視し、ログを取得し、問題が発生した場合にアラートを送信できる仕組みも提供する必要があります。

## 2. 要件整理

このシナリオの要件を整理すると、次のようなポイントが重要になります。

- 仮想マシンを営業時間外に自動停止する  
- 営業時間前に自動起動する  
- スケジュールを簡単に変更できる  
- 仮想マシンの状態を監視する  
- ログ収集機能を提供する  
- 異常時のアラート通知  

これらの要件から、「自動化」と「監視」の2つの機能を組み合わせたソリューションが必要であることが分かります。

## 3. 技術の基本概念

Azure では、リソース管理や運用タスクを自動化するために Azure Automation というサービスが提供されています。このサービスを使用すると、Azure 環境のさまざまな運用タスクを自動化できます。

Azure Automation の中心となる機能が Runbook です。Runbook は PowerShell や Python などのスクリプトとして作成され、特定のタスクを自動実行するために使用されます。

Runbook を使用すると、例えば次のような操作を自動化できます。

- 仮想マシンの起動  
- 仮想マシンの停止  
- リソースの構成変更  
- メンテナンス処理の実行  

また、Runbook はスケジュール機能と連携することで、指定した時間に自動実行することが可能です。

一方、Azure Monitor は Azure リソースの監視サービスです。このサービスはログ収集、メトリック分析、アラート通知などの機能を提供します。

## 4. アーキテクチャまたは設計のポイント

このシナリオでは、自動化と監視を組み合わせた設計が必要になります。

まず、Azure Automation を使用して Runbook を作成します。この Runbook では、Azure 仮想マシンを停止するスクリプトと起動するスクリプトを実装します。

次に、Automation のスケジュール機能を使用して、以下のようなスケジュールを設定します。

- 夜間に VM を自動停止  
- 朝の営業開始前に VM を自動起動  

このスケジュールは Azure Portal から簡単に変更できるため、運用の柔軟性も確保できます。

さらに、Azure Monitor を使用して仮想マシンの状態や Runbook の実行ログを収集します。これにより、Runbook が正常に実行されたかどうかや、仮想マシンの状態を監視できます。

## 5. 設計判断（なぜこの構成になるか）

この問題では、仮想マシンの自動起動・停止と監視機能の両方が必要です。

Azure Automation は Azure リソースの運用タスクを自動化するために設計されたサービスであり、Runbook を利用することで仮想マシンの起動や停止を簡単に自動化できます。

さらに、Azure Monitor を組み合わせることで次の機能を実現できます。

- 仮想マシンの状態監視  
- Runbook 実行ログの記録  
- 異常発生時のアラート通知  

このように Automation と Monitor を組み合わせることで、スケジュールベースの自動化と監視を同時に実現できます。

## 6. 他の選択肢が誤りな理由

この問題では他の Azure サービスも選択肢として提示されていますが、要件を完全には満たしていません。

B. Azure Logic Apps と Azure Functions  

Logic Apps や Functions はワークフロー自動化やサーバーレス処理に適していますが、仮想マシンの運用管理のための標準的な自動化ツールではありません。Azure Automation の方が VM 管理には適しています。

C. Azure DevOps パイプライン  

Azure DevOps は CI/CD やアプリケーションデプロイメントのためのサービスであり、仮想マシンの運用スケジュール管理には適していません。

D. Azure Scheduler と Azure Cost Management  

Azure Scheduler はすでに非推奨となっているサービスです。また、Azure Cost Management はコスト分析ツールであり、自動化機能は提供していません。

## 7. 最終回答

A. Azure Automation（Runbooks）と Azure Monitor

## 8. まとめ

Azure 仮想マシンの運用コストを削減するためには、営業時間外の自動停止と営業時間前の自動起動を実装することが効果的です。

Azure Automation の Runbook を使用することで、仮想マシンの起動・停止をスケジュールに基づいて自動化できます。さらに Azure Monitor を組み合わせることで、ログ収集、状態監視、アラート通知を実現できます。

そのため、Azure Automation と Azure Monitor の組み合わせが、このシナリオの要件を満たす最適なソリューションとなります。
