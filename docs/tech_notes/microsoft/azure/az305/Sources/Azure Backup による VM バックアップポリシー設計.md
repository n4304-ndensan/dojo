# Azure Backup による VM バックアップポリシー設計

## 1. 背景（シナリオ）

ある企業では、Azure 仮想マシン（VM）上で重要なアプリケーションを運用しています。これらの仮想マシンは業務にとって重要な役割を果たしているため、データ保護と迅速な復旧が可能なバックアップ戦略が必要です。

システム管理者は、VM のバックアップポリシーを設計する任務を担当しています。バックアップは定期的に取得される必要があり、さらにバックアップデータを一定期間保持する必要があります。

また、システム障害やデータ損失が発生した場合には、できるだけ短時間で復旧できることが求められています。これは **RTO（Recovery Time Objective）** を最小化することを意味します。

そのため、Azure 上で仮想マシンのバックアップを自動化し、柔軟な保持ポリシーを設定できるサービスを選択する必要があります。

## 2. 要件整理

問題文から読み取れるバックアップ要件を整理すると次のようになります。

まず、仮想マシンのバックアップは毎日実行され、そのバックアップを 30 日間保持する必要があります。

次に、週単位のバックアップを取得し、それを 12 週間保持する必要があります。

さらに、障害発生時には迅速に復旧できる必要があるため、復旧時間目標（RTO）を最小化できる仕組みが必要になります。

これらをまとめると次の要件になります。

- Azure VM の定期バックアップ  
- 日次バックアップを 30 日間保持  
- 週次バックアップを 12 週間保持  
- 迅速な復旧（低 RTO）  

## 3. 技術の基本概念

Azure にはデータ保護とバックアップを実現するためのサービスとして **Azure Backup** が提供されています。

Azure Backup は Azure リソース（VM、SQL Database、File Share など）のバックアップを自動的に取得し、安全に保存するサービスです。

Azure Backup は **Recovery Services Vault** を利用してバックアップデータを保存します。この Vault はバックアップデータを管理するためのコンテナであり、バックアップポリシーや復旧ポイントの管理を行います。

バックアップポリシーでは、次のような設定を柔軟に行うことができます。

- バックアップスケジュール  
- 日次・週次・月次・年次の保持期間  
- 復旧ポイントの管理  

## 4. アーキテクチャまたは設計のポイント

このシナリオでは Azure Backup を使用して VM のバックアップポリシーを構成します。

まず、Recovery Services Vault を作成し、その中でバックアップポリシーを定義します。

次に、バックアップポリシーで日次バックアップを設定し、その保持期間を 30 日に設定します。

さらに、週次バックアップを設定し、その保持期間を 12 週間に設定します。

Azure Backup はスナップショットベースのバックアップとストレージベースのバックアップを組み合わせており、迅速な復元を可能にします。これにより RTO を最小限に抑えることができます。

## 5. 設計判断（なぜこの構成になるか）

この問題の正解は **A. Azure Backup** です。

Azure Backup は Azure VM のバックアップを自動化するサービスであり、日次・週次の保持ポリシーを簡単に設定できます。

また、Azure Backup は VM の復元を迅速に行えるよう設計されており、復旧時間目標（RTO）を最小化することができます。

さらに、Azure Portal からバックアップポリシーの管理、復旧ポイントの確認、復元操作を一元管理できます。

## 6. 他の選択肢が誤りな理由

Azure Site Recovery は災害復旧（Disaster Recovery）を目的としたサービスであり、仮想マシンを別リージョンにレプリケーションする機能を提供します。バックアップ保持ポリシーの管理には適していません。

Azure Storage の Blob Snapshot を利用してバックアップを作成することも可能ですが、バックアップスケジュールや保持ポリシーを手動で管理する必要があります。そのため運用管理が複雑になります。

Azure File Sync はオンプレミスと Azure File Share のファイル同期を目的としたサービスであり、VM バックアップのためのサービスではありません。

## 7. 最終回答

A. Azure Backup

## 8. まとめ

この問題は Azure におけるバックアップサービスの役割を理解しているかを確認する問題です。

Azure Backup は Azure VM のバックアップを自動化し、日次や週次などの柔軟な保持ポリシーを設定できるサービスです。また、迅速な復元機能により復旧時間目標（RTO）を最小化できます。

そのため、Azure VM のバックアップポリシーを設計する場合に最も適したサービスは Azure Backup です。
