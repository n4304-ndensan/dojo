# Azure Batch による GPU バッチワークロードのコスト最適化

## 1. 背景（シナリオ）

機械学習（Machine Learning）のワークロードでは、GPU を使用した高性能な計算が必要になる場合があります。特にディープラーニングのトレーニングや推論処理では、GPU による並列処理が大きなパフォーマンス向上をもたらします。

しかし GPU 仮想マシンは Azure の中でも非常に高価なコンピューティングリソースです。そのため、GPU を必要とするワークロードでは **必要な時間だけ GPU を使用するアーキテクチャ**が重要になります。

今回のシナリオでは、機械学習ワークロードが **毎晩数時間だけ実行されるバッチ処理**として動作します。つまり、この処理は 24 時間連続で動くものではありません。

例えば、次のようなケースが考えられます。

日中はデータ収集を行い、夜間にモデル学習やデータ処理を実行するケースです。バッチ処理が完了した後は GPU コンピューティングを使用しません。

このようなワークロードでは、GPU 仮想マシンを常に稼働させるとコストが大きく無駄になります。

そのため、このシナリオでは **コストを最小化しながら GPU バッチ処理を実行できるソリューション**が求められています。

---

## 2. 要件整理

問題の要件を整理すると、次のポイントが重要になります。

まず、このワークロードは **GPU コンピューティングを必要とする機械学習処理**です。

次に、処理は **毎晩数時間のみ実行されるバッチジョブ**です。

また、ワークロードは常時稼働するものではありません。

さらに、このソリューションでは **コストを最小化すること**が重要です。

このような条件では、GPU リソースを **必要なときだけ割り当て、不要なときは解放する仕組み**が必要になります。

この要件に最も適した Azure サービスが **Azure Batch**です。

---

## 3. Azure Batch の概要

Azure Batch は、大規模なバッチ処理ワークロードを実行するために設計された Azure のコンピューティングサービスです。

Azure Batch を使用すると、次のような処理を自動化できます。

- 大量のバッチジョブの実行  
- ジョブキュー管理  
- タスクの並列実行  
- 失敗タスクの再試行  
- コンピューティングリソースの自動スケーリング  

Batch は特に **科学計算、金融計算、レンダリング、機械学習トレーニング**などのワークロードで使用されます。

---

## 4. GPU 対応 Batch プール

Azure Batch では、仮想マシンを **プール（pool）**として管理します。

[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Azure Virtual Machines]]
このプールには GPU 対応の VM サイズを指定することができます。

例えば次のような GPU VM を使用できます。

- NC シリーズ  
- ND シリーズ  
- NV シリーズ  

Batch はこの GPU VM プールを使用して、並列処理タスクを実行します。

---

## 5. 自動スケーリングとコスト最適化

Azure Batch の最大のメリットは **自動スケーリングと VM の割り当て解除（deallocation）**です。

Batch では、ジョブが実行されると必要な数の VM が自動的に割り当てられます。

そしてジョブが完了すると、VM を **非割り当て（deallocate）**できます。

VM が非割り当て状態になると、コンピューティング料金は発生しません。

つまり、GPU を使用している時間だけ課金されます。

この仕組みにより、夜間バッチ処理のようなワークロードで大幅なコスト削減が可能になります。

---

## 6. バッチワークロードに適した機能

Azure Batch はバッチ処理に特化した機能を多数提供しています。

例えば、ジョブは複数のタスクに分割して並列処理できます。

また、タスクが失敗した場合には自動的に再試行されます。

さらに、タスクの依存関係を定義することも可能です。

このような機能により、複雑なバッチワークフローを簡単に管理できます。

---

## 7. 他の選択肢が適さない理由

### 常時稼働 GPU VM

GPU 仮想マシンを常時稼働させると、ジョブが実行されていない時間でも料金が発生します。

夜間のみ実行されるワークロードでは非常に非効率です。

---

### VM Scale Sets

VM Scale Sets でもスケール制御は可能ですが、バッチジョブ管理やタスクスケジューリングの機能はありません。

---

[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Azure Kubernetes Service (AKS)]]
### AKS

AKS はコンテナオーケストレーションに適したサービスですが、単純なバッチ処理にはオーバーヘッドが大きくなります。

---

## 8. 最終回答

D. Azure Batch（使用していないときに非割り当てされる GPU 対応 VM プール）
