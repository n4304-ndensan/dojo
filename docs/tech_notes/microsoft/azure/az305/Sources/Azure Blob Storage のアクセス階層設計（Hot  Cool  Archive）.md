# Azure Blob Storage のアクセス階層設計（Hot / Cool / Archive）

## 1. 背景（シナリオ）

ある企業では、オンプレミスのファイルサーバーに保存されている約500GBのファイルをAzureへ移行する計画があります。これらのファイルは大規模なリポジトリに保存されており、日常的にアクセスされるわけではありません。

しかし、完全にアクセスできなくなるわけではなく、ユーザーが要求した場合には一定時間以内に取得できる必要があります。今回の要件では、ファイルは **リクエスト後12時間以内にアクセス可能である必要**があります。

さらに、企業はストレージコストをできる限り削減することを重視しています。そのため、頻繁にアクセスされないデータに対して最適なストレージ層を選択することが重要になります。

## 2. 要件整理

この問題のポイントは、Azure Blob Storageのアクセス階層（Access Tier）を適切に選択することです。

シナリオから読み取れる要件は以下の通りです。

- 約500GBのデータをAzureへ移行
- ファイルはめったにアクセスされない
- アクセス要求後 **12時間以内に取得可能**
- ストレージコストを最小化
- Azure Blob Storageを使用

Azureでは、データアクセス頻度に応じて複数のストレージ階層が提供されています。

## 3. 技術の基本概念

Azure Blob Storageには、データアクセス頻度に応じて次の3つのアクセス階層があります。

まず **Hot Tier（ホット層）** です。これは頻繁にアクセスされるデータ向けのストレージです。読み取り性能が高く、即時アクセスが可能ですが、ストレージコストは最も高くなります。

次に **Cool Tier（クール層）** です。これはアクセス頻度が低いデータ向けのストレージです。データは即時アクセス可能ですが、ホット層よりストレージコストが低くなります。

最後に **Archive Tier（アーカイブ層）** です。これはほとんどアクセスされない長期保存データ向けのストレージです。ストレージコストは最も低いですが、データを読み出す際には **再水和（rehydration）** が必要になり、数時間かかる場合があります。

## 4. アーキテクチャまたは設計のポイント

この問題では、ストレージコストを最小化しながら、12時間以内にデータを取得できる必要があります。

Archive Tierは、データを取り出す際に再水和処理が必要になります。この処理は通常数時間で完了しますが、**最大で12時間程度かかる場合があります。**

そのため、Archive Tierはこの要件に適合する可能性があります。

また、Cool Tierも候補になります。Cool Tierは低頻度アクセス用のストレージであり、データは即時アクセス可能です。さらに、Hot Tierよりストレージコストが低いため、コスト削減にも適しています。

この問題では、Archive TierとCool Tierの両方が条件を満たす可能性があります。

## 5. 設計判断（なぜこの構成になるか）

このシナリオでは、データアクセス頻度が非常に低く、コスト削減が重要な要件です。

Archive TierはAzureで最も低コストのストレージオプションです。アクセス頻度が非常に低いデータには最適な選択です。

また、Cool Tierは低頻度アクセスデータ向けに設計されており、即時アクセスが可能です。

さらに、Azureでは次の2種類のストレージアカウントでArchive Tierを使用できます。

- 汎用v2ストレージアカウント（General Purpose v2）
- Blob Storageアカウント

そのため、次の2つの選択肢が正解になります。

- アーカイブアクセス層を持つ汎用v2ストレージアカウント
- アーカイブアクセス層を持つBlobストレージアカウント

## 6. 他の選択肢が誤りな理由

### B. クールアクセス層付き汎用v2ストレージアカウント

Cool Tierは低頻度アクセスには適していますが、Archive Tierよりもストレージコストが高くなります。

この問題では「ストレージコストを最小化する」という要件があるため、Archive Tierの方が適しています。

### D. ホットアクセス層付きBlobストレージアカウント

Hot Tierは頻繁にアクセスされるデータ向けです。

そのため、次の問題があります。

- ストレージコストが高い
- このシナリオのアクセス頻度に適していない

## 7. 最終回答

この要件を満たす最適なAzureストレージソリューションは次の2つです。

- **アーカイブアクセス層を持つ汎用v2ストレージアカウント**
- **アーカイブアクセス層を持つBlobストレージアカウント**

したがって **正解は A と C** です。

## 8. まとめ

Azure Blob Storageでは、データアクセス頻度に応じてアクセス階層を選択できます。

主な階層は次の3つです。

- Hot Tier（頻繁アクセス）
- Cool Tier（低頻度アクセス）
- Archive Tier（ほぼアクセスなし）

Archive Tierは最も低コストのストレージであり、長期保存データに適しています。

今回のシナリオでは、データはめったにアクセスされず、コスト削減が重要な要件であるため、**Archive Tierが最も適したストレージ階層**になります。
