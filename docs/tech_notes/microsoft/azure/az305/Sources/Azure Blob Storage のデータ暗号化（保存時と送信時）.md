# Azure Blob Storage のデータ暗号化（保存時と送信時）

## 1. 背景（シナリオ）

ある組織では、Azure Blob Storage に機密データを保存しています。このデータには個人情報やビジネス上重要な情報が含まれているため、セキュリティ対策を強化する必要があります。

特に重要なのは、データがストレージに保存されているときだけでなく、ネットワークを通じて転送される際にも保護されることです。もし暗号化されていない状態で通信されると、通信途中でデータが盗聴されたり改ざんされたりする可能性があります。

そのため、このシナリオでは **保存時（At Rest）と送信時（In Transit）の両方でデータを暗号化する方法**を選択する必要があります。

## 2. 要件整理

問題の要件を整理すると、次の2つの暗号化が必要です。

まず **保存時の暗号化（Encryption at Rest）**です。これはデータが Azure Blob Storage に保存されている状態で暗号化されていることを意味します。

次に **送信時の暗号化（Encryption in Transit）**です。これはクライアントと Azure Storage 間でデータが転送される際に暗号化されることを意味します。

つまり、この問題では次の2つを実現する必要があります。

- ストレージ内のデータを暗号化する  
- ネットワーク通信中のデータを暗号化する  

## 3. 技術の基本概念

Azure Storage では、データ保護のために複数の暗号化機能が提供されています。

保存時の暗号化には **Azure Storage Service Encryption（SSE）** が使用されます。SSE は Azure Storage の組み込み機能であり、ストレージアカウントに保存されるデータを自動的に暗号化します。

SSE は AES-256 暗号化アルゴリズムを使用しており、データがディスクに保存される前に暗号化されます。データが読み取られる際には自動的に復号化されるため、アプリケーション側で特別な処理は必要ありません。

一方、送信時の暗号化には **TLS/SSL（Transport Layer Security / Secure Sockets Layer）** が使用されます。TLS は HTTPS 通信で使用される暗号化プロトコルであり、ネットワーク通信中のデータを保護します。

## 4. アーキテクチャまたは設計のポイント

クラウド環境のデータセキュリティでは、データがどの状態にあるかを理解することが重要です。データには主に次の2つの状態があります。

まず、ストレージに保存されている状態です。この状態では、ディスク上のデータを暗号化することで不正アクセスを防ぎます。

次に、ネットワークを通じて転送されている状態です。この状態では、通信を暗号化することで盗聴や改ざんを防ぎます。

Azure Blob Storage では、この2つの状態に対してそれぞれ次の対策を実装します。

- Azure Storage Service Encryption（SSE）による保存時暗号化  
- TLS/SSL による通信暗号化  

このように、保存時と送信時の両方で暗号化を実装することで、データの機密性を確保できます。

## 5. 設計判断（なぜこの構成になるか）

Azure Storage Service Encryption は Azure Storage に組み込まれた機能であり、ストレージアカウントに保存されるすべてのデータを自動的に暗号化します。これにより、保存されたデータが不正にアクセスされた場合でも、暗号化されているため安全性が確保されます。

また、TLS/SSL を使用することで、クライアントと Azure Blob Storage の間で送信されるデータが暗号化されます。これにより、ネットワーク上での盗聴や中間者攻撃を防ぐことができます。

この2つの方法を組み合わせることで、データのライフサイクル全体を通じてセキュリティを確保できます。

## 6. 他の選択肢が誤りな理由

Azure Key Vault は暗号化キーの管理には非常に有効ですが、データそのものの暗号化を直接行う機能ではありません。Key Vault は SSE で使用されるカスタマー管理キーの管理に使用される場合がありますが、暗号化方法そのものではありません。

また、Azure Virtual Network（VNet）はネットワーク分離やアクセス制御には有効ですが、通信の暗号化を提供するものではありません。データ転送の暗号化は TLS/SSL によって実現されます。

## 7. 最終回答

A. Azure Storage Service Encryption（SSE）を有効にする  
C. トランジット内のすべてのデータに対して TLS/SSL を有効にする

## 8. まとめ

Azure Blob Storage に保存された機密データを保護するためには、保存時と送信時の両方で暗号化を実装する必要があります。

保存時の暗号化には Azure Storage Service Encryption（SSE）を使用し、ストレージに保存されるデータを自動的に暗号化します。また、送信時の暗号化には TLS/SSL を使用し、ネットワーク通信中のデータを保護します。

この2つの方法を組み合わせることで、Azure Blob Storage におけるデータの機密性と安全性を確保することができます。
