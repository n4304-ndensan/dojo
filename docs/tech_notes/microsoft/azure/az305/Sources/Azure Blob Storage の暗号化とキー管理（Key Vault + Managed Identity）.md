# Azure Blob Storage の暗号化とキー管理（Key Vault + Managed Identity）

## 1. 背景（シナリオ）

ある企業では、Azure Blob Storage に機密データを保存しています。このデータには個人情報やビジネス上の重要な情報が含まれているため、セキュリティを確保することが非常に重要です。

組織は、データが保存されているとき（保存時）だけでなく、ネットワーク経由で転送されているとき（転送中）にも暗号化されている必要があります。また、暗号化を実装するだけではなく、その暗号化キーを安全に管理する仕組みを導入する必要があります。

このようなセキュリティ要件を満たすためには、Azure のネイティブサービスを利用した安全なキー管理アーキテクチャを設計することが重要になります。

## 2. 要件整理

このシナリオでは、単にデータを暗号化するだけではなく、暗号化キーの安全な管理が重要なポイントになっています。

問題文から読み取れる要件を整理すると次の通りです。

まず、Azure Blob Storage に保存されているデータは保存時に暗号化される必要があります。次に、データがネットワークを通じて送信される際にも暗号化される必要があります。

さらに、暗号化キーを安全に保存し、適切にアクセス管理できる仕組みが必要になります。

これらをまとめると次のような要件になります。

・保存時のデータ暗号化  
・転送中のデータ暗号化  
・暗号化キーの安全な管理  
・キーへのアクセス制御  
・認証情報管理の最小化  

これらの要件を満たすためには **Azure Key Vault と Managed Identity の組み合わせ**を使用します。

## 3. 技術の基本概念

Azure Blob Storage は、デフォルトで **Storage Service Encryption（SSE）** を使用して保存データを暗号化します。また、HTTPS を利用することで転送中のデータも暗号化されます。

しかし、暗号化キーの管理方法には大きく次の2種類があります。

Microsoft 管理キー（Microsoft-managed keys）と、顧客管理キー（Customer-managed keys）です。

顧客管理キーを利用する場合、暗号化キーは **Azure Key Vault** に保存されます。Key Vault は Azure の専用キー管理サービスであり、暗号鍵やシークレットを安全に保存できます。

## 4. アーキテクチャまたは設計のポイント

Azure Key Vault を使用すると、暗号化キーの保存、アクセス管理、監査ログなどを一元的に管理できます。

この仕組みでは、Azure Blob Storage が Key Vault に保存されたキーを使用してデータを暗号化します。その際、Key Vault へのアクセスは **Managed Identity** を使用して認証されます。

この構成の重要なポイントは次の通りです。

まず、暗号化キーは Key Vault に安全に保存されます。次に、Managed Identity を利用することで、アプリケーションコードに認証情報を保存する必要がなくなります。さらに、RBAC やアクセスポリシーによってキーへのアクセスを細かく制御できます。

この設計により、セキュリティと運用管理の両方を最適化できます。

## 5. 設計判断（なぜこの構成になるか）

Azure Key Vault を使用する最大のメリットは、暗号化キーの管理を Azure の専用サービスに任せることができる点です。

Key Vault ではキーへのアクセスを RBAC またはアクセスポリシーで制御でき、監査ログも取得できます。これにより、コンプライアンス要件やセキュリティ監査にも対応できます。

また、Managed Identity を使用することで、アプリケーション側で秘密情報を管理する必要がなくなります。Azure が認証トークンの管理を自動化するため、認証情報の漏洩リスクも低減できます。

## 6. 他の選択肢が誤りな理由

### B Azure Active Directory を使用してユーザーベースの暗号化を管理

Azure Active Directory はユーザー認証やアクセス管理を行うサービスです。しかし、暗号化キーを安全に保存・管理するための専用サービスではありません。

### C ストレージアカウント内でキーを自動ローテーション

キーのローテーションは重要なセキュリティ機能ですが、キーの安全な保存やアクセス制御を提供するものではありません。キー管理には Key Vault を使用するのがベストプラクティスです。

### D PowerShell で手動暗号化してアップロード

手動暗号化は運用負荷が高く、スケーラビリティやセキュリティの観点でも推奨されません。また、キー管理が複雑になります。

## 7. 最終回答

A. **キーを Azure Key Vault に保存し、アクセスには Managed Identity を使用する**

## 8. まとめ

Azure Blob Storage の機密データを安全に保護するためには、保存時暗号化と転送時暗号化の両方を実装する必要があります。

さらに、暗号化キーを安全に管理するためには Azure Key Vault を利用することが推奨されます。

Managed Identity を組み合わせることで、アプリケーション側で認証情報を管理する必要がなくなり、セキュリティと運用管理の両方を改善できます。

試験では **「暗号化キー管理」「Customer Managed Key」「Key Vault」「Managed Identity」** といったキーワードが出てきた場合、この構成が正解になるケースが多いです。
