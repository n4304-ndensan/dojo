# Azure Blob Storage を使用した動画ファイルストレージ設計

## 1. 背景（シナリオ）

ある企業では Azure 上で新しいアプリケーションを設計しています。このアプリケーションでは **100MB〜10GBの動画ファイル**を保存する必要があります。

動画コンテンツは一般的にサイズが大きく、ユーザーが再生する際には高い読み取り性能が求められます。また、動画データは時間の経過とともに増加するため、スケーラビリティとコスト効率も重要になります。

そのため、このシステムでは次のようなストレージソリューションが必要になります。

大容量ファイルの保存  
高い読み取り性能  
コスト効率の良いストレージ  
高いスケーラビリティ  

このような要件を満たす Azure のストレージサービスを選択する必要があります。

## 2. 要件整理

シナリオから読み取れる重要な要件を整理すると次の通りです。

まず、保存対象は **100MB〜10GBの動画ファイル**であり、これは典型的な非構造化データです。

次に、ユーザーが動画を再生するため、読み取り性能が重要になります。

さらに、動画データは大量に保存される可能性があるため、ストレージコストを最小化する必要があります。

この問題の要件をまとめると次のようになります。

非構造化データの保存  
大容量ファイルの保存  
高い読み取り性能  
低コスト  
高いスケーラビリティ  

これらの要件を満たす Azure ストレージが **Azure Blob Storage** です。

## 3. 技術の基本概念

Azure Blob Storage は、Azure が提供する **オブジェクトストレージサービス**です。

このサービスは画像、動画、バックアップファイルなどの **非構造化データを大量に保存するために設計**されています。

Blob Storage には主に次の3種類のストレージタイプがあります。

まず **Block Blob** は、大容量ファイルの保存に最適化されたストレージです。動画、画像、ドキュメントなどの保存に使用されます。

次に **Append Blob** はログファイルなどの追加書き込み用途に使用されます。

最後に **Page Blob** は仮想マシンディスクなどに使用されます。

動画ファイルの保存には **Block Blob** が最適です。

## 4. アーキテクチャまたは設計のポイント

動画配信アプリケーションでは、Blob Storage をバックエンドストレージとして使用します。

この構成では、アプリケーションが Blob Storage に動画を保存し、ユーザーは HTTP または HTTPS 経由で動画を取得します。

さらに、読み取り性能を向上させるために **Azure CDN** と組み合わせることができます。

CDN を利用すると、動画コンテンツがエッジロケーションにキャッシュされるため、ユーザーの近くから高速に配信できます。

また、Blob Storage は自動的にスケールするため、動画数が増加してもインフラ管理が不要です。

## 5. 設計判断（なぜこの構成になるか）

Azure Blob Storage が最適な理由は次の通りです。

まず、Blob Storage は動画などの **大容量非構造化データの保存に最適化されています**。

次に、Hot、Cool、Archive などのストレージ階層を利用することで **ストレージコストを最適化**できます。

さらに、Blob Storage は Azure のグローバルインフラ上で動作するため、高い可用性とスケーラビリティを提供します。

このような理由から、動画ファイルを保存するアプリケーションでは Blob Storage が標準的な選択肢になります。

## 6. 他の選択肢が誤りな理由

### A Azure Files

Azure Files は SMB プロトコルを使用したファイル共有サービスです。  
ファイルサーバーの置き換えには適していますが、大量の動画ストリーミング用途には最適ではありません。

### C Azure SQL Database

Azure SQL Database はリレーショナルデータベースサービスです。  
動画などの大容量バイナリデータを保存する用途には適していません。

### D Azure Data Lake Storage Gen2

Data Lake Storage Gen2 は主に **ビッグデータ分析**や **データレイク**用途に最適化されています。  
動画配信のような一般的なコンテンツストレージには Blob Storage の方が適しています。

## 7. 最終回答

B. **Azure Blob Storage**

## 8. まとめ

Azure Blob Storage は動画、画像、バックアップなどの非構造化データを保存するための Azure のオブジェクトストレージです。

このサービスは高い読み取り性能とスケーラビリティを提供し、ストレージ階層によってコスト最適化も可能です。

試験では **「動画」「大容量ファイル」「非構造化データ」「低コストストレージ」** といったキーワードが出てきた場合、Azure Blob Storage が正解になるケースが非常に多いです。
