# Azure Cosmos DB SQL API によるグローバルデータ集約アーキテクチャ

## 1. 背景（シナリオ）

ある企業では、世界中の地域から売上データを収集し、それを集約して分析するクラウドアプリケーションを設計しています。このアプリケーションでは、複数の地域からのデータを迅速に処理し、ユーザーにリアルタイムに近い形で結果を提供する必要があります。

そのため、データベースにはいくつかの重要な要件があります。まず、データは複数のリージョンで利用できる必要があり、災害対策と可用性の観点からジオレプリケーションが必要です。

さらに、アプリケーションは売上データを分析するためにSQLのようなクエリを利用するため、SQLクエリをサポートするデータベースが必要になります。

また、ユーザーは世界中のさまざまな地域からアクセスするため、読み取り操作は低遅延で実行できる必要があります。

## 2. 要件整理

この問題では、Azure Cosmos DB の API を選択する必要があります。

シナリオから読み取れる要件を整理すると次の通りです。

- 複数地域のデータを集約
- ジオレプリケーションが必要
- SQLクエリが必要
- 読み取り操作の低レイテンシ
- スケーラブルなデータベース

Azure Cosmos DB は複数のAPIを提供しており、それぞれ異なるデータモデルやクエリ方法をサポートしています。

## 3. 技術の基本概念

Azure Cosmos DBはマルチモデルNoSQLデータベースであり、複数のAPIをサポートしています。

代表的なAPIには次の種類があります。

まず **SQL API（Core API）** です。  
これはCosmos DBのネイティブAPIであり、JSONドキュメントデータをSQLライクなクエリ言語で操作できます。

次に **MongoDB API** があります。  
これはMongoDB互換のAPIであり、MongoDBアプリケーションをCosmos DB上で実行できます。

次に **Cassandra API** です。  
これはApache Cassandra互換のAPIであり、大規模分散データベース用途で利用されます。

さらに **PostgreSQL** があります。  
これはAzure Cosmos DBとは別のサービスであり、リレーショナルデータベースです。

## 4. アーキテクチャまたは設計のポイント

このシナリオでは、以下の要件が重要になります。

まず、SQLクエリを利用できることです。売上データの分析では、フィルタリング、集計、並び替えなどのクエリを頻繁に使用します。

次に、ジオレプリケーションです。Cosmos DBのSQL APIはグローバル分散機能をネイティブにサポートしています。

また、Cosmos DBは複数リージョンへのデータレプリケーションを自動的に行い、世界中から低レイテンシでデータにアクセスできます。

さらに、Cosmos DBは自動インデックス機能を持っているため、読み取り性能が非常に高く、低遅延のクエリが可能になります。

## 5. 設計判断（なぜこの構成になるか）

この問題の要件を満たす最適なAPIは **SQL API（Core API）** です。

SQL APIはAzure Cosmos DBの標準APIであり、SQLライクなクエリを使用してJSONドキュメントデータを検索できます。

また、Cosmos DBのグローバル分散機能と完全に統合されているため、複数リージョンのデータアクセスに最適です。

さらに、Cosmos DBは読み取り操作を非常に高速に処理できるよう設計されており、低レイテンシのデータ取得が可能です。

そのため、売上データの集約や分析などのユースケースに非常に適しています。

## 6. 他の選択肢が誤りな理由

### A. PostgreSQL

PostgreSQLはリレーショナルデータベースですが、これはAzure Cosmos DBのAPIではありません。

この問題ではCosmos DBのAPIを選択する必要があります。

### B. MongoDB API

MongoDB APIはMongoDB互換のAPIです。

しかし、SQLクエリを直接使用することはできません。MongoDBのクエリ構文はSQLとは異なるため、このシナリオの要件には最適ではありません。

### D. Cassandra API

Cassandra APIは大規模分散データベース用途に適していますが、SQLクエリサポートが限定的です。

また、データ分析や集計用途ではSQL APIの方が一般的に使いやすいです。

## 7. 最終回答

この要件を満たす最適なAzure Cosmos DB APIは次です。

**SQL API**

したがって **正解は C** です。

## 8. まとめ

Azure Cosmos DBはグローバル分散型NoSQLデータベースであり、複数のAPIをサポートしています。

その中でも **SQL API（Core API）** は最も一般的に利用されるAPIです。

SQL APIの主な特徴は次の通りです。

- SQLライクなクエリ言語
- グローバル分散
- 低レイテンシ読み取り
- 自動スケーリング
- 高可用性

そのため、複数リージョンからデータを集約し、SQLクエリで分析するようなアプリケーションでは **Cosmos DB SQL APIが最も適した選択肢**になります。
