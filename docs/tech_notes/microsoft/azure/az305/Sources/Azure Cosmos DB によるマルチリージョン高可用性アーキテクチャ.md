# Azure Cosmos DB によるマルチリージョン高可用性アーキテクチャ

## 1. 背景（シナリオ）

ある金融機関では、顧客情報を扱うエンタープライズアプリケーションを Azure 上で運用しています。このアプリケーションは機密性の高いデータを扱うため、厳格なセキュリティおよびコンプライアンス要件を満たす必要があります。

特に重要なのは、アプリケーションの可用性とデータ保護です。金融システムではサービス停止が大きなビジネス影響を与えるため、リージョン障害が発生した場合でもサービスが継続するよう設計する必要があります。

さらに、このシステムでは世界中のユーザーが利用するため、異なる地域からのアクセスに対して低遅延でデータを提供する必要があります。また、保存されるデータは顧客が管理する暗号化キーを使用して暗号化する必要があります。

## 2. 要件整理

このシナリオでは、複数のアーキテクチャ要件が同時に提示されています。単にデータを保存するだけではなく、可用性・セキュリティ・パフォーマンスをすべて満たす必要があります。

問題文から読み取れる要件を整理すると次の通りです。

まず、顧客が管理するキー（Customer Managed Key）を使用して保存データを暗号化する必要があります。次に、複数リージョンからのアクセスに対して低遅延でデータを提供する必要があります。

さらに、リージョン障害が発生した場合でもサービスが停止しないように、自動フェイルオーバーをサポートする必要があります。

これらをまとめると、次の要件になります。

・保存データの暗号化（顧客管理キー）  
・マルチリージョン対応  
・低遅延アクセス  
・自動フェイルオーバー  
・高可用性  
・データレジデンシー規制への対応  

これらすべてを満たす Azure サービスが **Azure Cosmos DB** です。

## 3. 技術の基本概念

Azure Cosmos DB は、グローバル分散型の NoSQL データベースサービスであり、高可用性と低遅延を実現するために設計されています。

Cosmos DB の最大の特徴は、Azure の複数リージョンにデータを自動的にレプリケーションできることです。この機能により、世界中のユーザーが最も近いリージョンからデータにアクセスできます。

また、Cosmos DB は **自動フェイルオーバー機能**を備えており、プライマリリージョンで障害が発生した場合でも別のリージョンに自動的に切り替えることができます。

さらに、**マルチリージョン書き込み（Multi-region writes）**を有効化することで、複数のリージョンから同時にデータを書き込むことが可能になります。

## 4. アーキテクチャまたは設計のポイント

金融アプリケーションのようなミッションクリティカルなシステムでは、マルチリージョン構成を採用することが重要です。

Azure Cosmos DB を利用した設計では、データを複数のリージョンにレプリケーションします。これにより、ユーザーは最も近いリージョンからデータにアクセスできるため、アクセス遅延を大幅に削減できます。

このアーキテクチャでは次のような構成が実現できます。

まず、Cosmos DB は複数リージョンにデータをレプリケーションします。次に、アプリケーションは最も近いリージョンのデータベースエンドポイントを利用します。さらに、リージョン障害が発生した場合には自動フェイルオーバーによって別リージョンに切り替わります。

この設計により、高可用性と低遅延アクセスの両方を実現できます。

## 5. 設計判断（なぜこの構成になるか）

Azure Cosmos DB が最適な理由は、グローバル分散と自動フェイルオーバーをネイティブに提供している点にあります。

特に次の機能が重要になります。

まず、マルチリージョン書き込みによって複数リージョンから同時にデータを書き込むことができます。これにより、地理的に分散したユーザーのレスポンス時間を短縮できます。

次に、自動フェイルオーバー機能によってリージョン障害が発生してもサービスが継続します。

さらに、Azure Key Vault と連携することで、顧客管理キー（Customer Managed Keys）による暗号化を実装できます。

## 6. 他の選択肢が誤りな理由

### A Azure Blob Storage + 顧客管理キー

Azure Blob Storage は顧客管理キーによる暗号化をサポートしていますが、グローバル分散データベースとしての機能はありません。また、マルチリージョン書き込みや自動フェイルオーバーをネイティブに提供していません。

### B Azure SQL Database + Active Geo-Replication

Azure SQL Database は地理レプリケーションを提供しますが、Cosmos DB のようなグローバル分散アーキテクチャやマルチリージョン書き込み機能は提供していません。また、グローバルスケールの低遅延アクセスには最適ではありません。

### D Azure Data Lake Storage

Azure Data Lake Storage は主にビッグデータ分析やデータレイク用途に使用されるストレージサービスです。トランザクション処理やグローバル低遅延アクセスを目的としたサービスではありません。

## 7. 最終回答

C. **Azure Cosmos DB（マルチリージョン書き込み + 自動フェイルオーバー）**

## 8. まとめ

Azure Cosmos DB は、グローバル分散アプリケーション向けに設計されたデータベースサービスです。

このサービスを利用することで、複数リージョンへのデータレプリケーション、低遅延アクセス、自動フェイルオーバーを実現できます。

さらに Azure Key Vault と連携することで顧客管理キーによる暗号化も可能になります。

試験では **「グローバル分散」「低遅延アクセス」「自動フェイルオーバー」「マルチリージョン書き込み」** といったキーワードが出てきた場合、Azure Cosmos DB が正解になるケースが非常に多いです。
