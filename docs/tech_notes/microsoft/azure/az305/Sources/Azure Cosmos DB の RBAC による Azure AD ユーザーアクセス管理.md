# Azure Cosmos DB の RBAC による Azure AD ユーザーアクセス管理

## 1. 背景（シナリオ）

ある組織では、SQL API を使用した Azure Cosmos DB データベースを展開する予定です。このデータベースには機密データが保存されるため、アクセス制御を適切に管理する必要があります。

特に、特定の Azure Active Directory（Azure AD / Microsoft Entra ID）ユーザーに対して **読み取り専用アクセス** を付与する必要があります。つまり、ユーザーはデータを閲覧できる必要がありますが、データの変更や削除などの操作は許可されていません。

この要件を満たすためには、Azure AD を利用した認証と、適切なアクセス権限の管理を行う必要があります。

## 2. 要件整理

このシナリオでは、Cosmos DB のアクセス管理に関する重要な要件があります。

まず、Azure AD のユーザーアカウントを使用して認証を行う必要があります。これにより、組織の ID 管理を Azure AD に統合できます。

次に、ユーザーには読み取り専用のアクセス権限のみを付与する必要があります。これにより、データの安全性を維持しながら必要な情報にアクセスできるようになります。

さらに、アクセス管理は Azure のセキュリティベストプラクティスに従い、最小特権の原則に基づいて構成する必要があります。

この問題の要件は次のように整理できます。

・Azure AD ユーザーによる認証  
・Cosmos DB への読み取りアクセス  
・最小特権の原則に基づくアクセス管理  
・Azure ネイティブのアクセス制御  

## 3. 技術の基本概念

Azure Cosmos DB では、リソースへのアクセス制御を管理するために **Role-Based Access Control（RBAC）** を利用できます。

RBAC は Azure のアクセス管理モデルであり、ユーザー、グループ、またはサービスプリンシパルに対してロールを割り当てることでリソースへのアクセス権限を制御します。

Azure RBAC を使用すると、次のようなスコープでアクセス制御を行うことができます。

・サブスクリプション  
・リソースグループ  
・特定の Azure リソース  

Azure Cosmos DB では、Azure AD ユーザーに対して適切なロールを割り当てることで、データベースへの読み取りアクセスなどを安全に管理できます。

## 4. アーキテクチャまたは設計のポイント

このシナリオでは、Azure AD のユーザーを Cosmos DB リソースに関連付け、RBAC を利用してアクセス権限を付与します。

具体的には、Azure Portal の **IAM（Identity and Access Management）** 機能を使用して、対象ユーザーに読み取りアクセス権を持つロールを割り当てます。

この方法により、ユーザーは Azure AD 認証を使用して Cosmos DB にアクセスでき、必要な操作のみを実行できるようになります。

また、RBAC は Azure の監査ログと統合されているため、誰がどのリソースにアクセスしているかを追跡することも可能です。

## 5. 設計判断（なぜこの構成になるか）

RBAC を使用することで、Azure AD ユーザーに対してきめ細かいアクセス制御を実現できます。

また、IAM を通じてロールを割り当てることで、アクセス管理を集中管理できるため、セキュリティ管理が容易になります。

さらに、RBAC は Azure のセキュリティモデルの中心的な仕組みであり、最小特権の原則に基づいたアクセス管理を実現できます。

## 6. 他の選択肢が誤りな理由

共有アクセス署名（SAS）は主に Azure Storage のアクセス制御に使用される仕組みであり、Cosmos DB の Azure AD ユーザーアクセス管理には適していません。

マスターキーは Cosmos DB アカウントへの完全なアクセスを提供するため、特定ユーザーに読み取り専用アクセスを提供する用途には適していません。

リソーストークンはアプリケーションレベルのアクセス制御に使用されることが多く、Azure AD ユーザーの IAM 管理には適していません。

## 8. まとめ

Azure Cosmos DB で Azure AD ユーザーにアクセス権を付与する場合、Azure RBAC を使用することが最も安全で管理しやすい方法です。

IAM を使用してユーザーにロールを割り当てることで、必要な権限のみを付与することができます。

この方法により、セキュリティを維持しながら Cosmos DB へのアクセスを適切に管理できます。
