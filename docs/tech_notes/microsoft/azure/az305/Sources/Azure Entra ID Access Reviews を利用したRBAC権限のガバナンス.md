# Azure Entra ID Access Reviews を利用したRBAC権限のガバナンス

## 1. 背景（シナリオ）

ある企業では、外部ベンダーが開発したカスタムアプリケーション「App1」をAzure環境で運用しています。現在、外部ベンダーの開発者はこのアプリケーションのさまざまなコンポーネントに対してRBAC（Role-Based Access Control）権限を持っています。

しかし、セキュリティとコンプライアンスを強化するためには、開発者が本当に必要な権限のみを保持するように管理する必要があります。特に、不要または確認されていない権限が長期間残っていると、内部不正や情報漏洩のリスクが高まります。

そのため、組織では次のような管理を実施する必要があります。

- 外部開発者の権限を定期的に確認する  
- 不要または未確認の権限を自動的に取り消す  
- 権限の状態に関するレポートを作成する  
- そのレポートを毎月アプリケーション所有者へ送信する  

これらの要件を満たすAzureのソリューションを選択する必要があります。

---

## 2. 要件整理

この問題では、RBAC権限を管理するためのIdentity Governance機能を選択する必要があります。

シナリオから読み取れる主な要件は次の通りです。

まず、ユーザーのアクセス権限を定期的にレビューする必要があります。これにより、不要な権限を検出できます。

次に、確認されていない権限は自動的に取り消される必要があります。つまり、レビュー結果を基にアクセスを自動的に変更できる仕組みが必要です。

さらに、権限の状況に関するレポートを生成する必要があります。そして、そのレポートを毎月アプリケーション所有者へ送信する必要があります。

要件をまとめると次の通りです。

- RBAC権限の定期レビュー  
- 未確認権限の自動削除  
- 詳細なアクセスレポート  
- 月次レポートの自動送信  

---

## 3. 技術の基本概念

Azure Entra ID（旧Azure AD）には、Identity Governance機能として「Access Reviews」が提供されています。

Access Reviewsは、ユーザーがアプリケーションやリソースに対して持っているアクセス権を定期的に確認するための機能です。

この機能では次のようなことが可能です。

- ユーザーアクセスの定期レビュー  
- 管理者またはアプリ所有者による承認  
- 未承認アクセスの自動削除  
- レビュー結果のレポート作成  

これにより、組織は継続的にアクセス権を管理し、不要な権限を排除できます。

---

## 4. アーキテクチャまたは設計のポイント

Access Reviewsを利用することで、アプリケーションへのアクセスを定期的に確認できます。

この構成では次のような流れになります。

まず、App1に対するRBAC権限のAccess Reviewを作成します。

次に、レビューの頻度を月次に設定します。レビューはアプリケーション所有者または管理者が実施します。

レビューの結果として、不要または未確認のアクセスが特定されます。

その後、Logic Appsを使用してレビュー結果を処理します。Logic AppsはAzureのワークフロー自動化サービスであり、レビュー結果に基づいて権限の取り消しを自動化できます。

また、Logic Appsを利用することで次の処理も自動化できます。

- レポート生成  
- メール通知  
- 月次レポート送信  

---

## 5. 設計判断（なぜこの構成になるか）

この問題では「アクセスレビュー」「自動権限削除」「月次レポート」の3つが重要です。

Azure Entra ID Access Reviewsは、アクセス管理のレビュー機能を提供します。これにより、外部ベンダーのアクセス権を定期的に確認できます。

さらに、Logic Appsを組み合わせることで次の処理を自動化できます。

- 未確認アクセスの削除  
- レポート生成  
- 月次通知  

この組み合わせにより、セキュリティとコンプライアンスの両方を強化できます。

---

## 6. 他の選択肢が誤りな理由

Azure Policyはリソース構成のコンプライアンスを管理するサービスであり、RBAC権限のレビューや自動取り消しには適していません。

Azure AD Identity GovernanceのEntitlement Managementは主にアクセス要求管理に使用されますが、定期的なアクセスレビューにはAccess Reviewsの方が適しています。

Azure AD Privileged Identity Management（PIM）は特権ロールの管理に使用されるサービスであり、通常は一時的な特権アクセス管理を目的としています。このシナリオのような継続的なRBACレビューには最適ではありません。

---

## 7. 最終回答

A.  
Azure AD Access Reviews を実装し、  
Logic Apps を使用して権限取り消しを自動化する

---

## 8. まとめ

Azure Entra ID Access Reviewsは、アクセス権限を定期的に確認し、不要なアクセスを排除するIdentity Governance機能です。

この機能を利用すると次のことが可能になります。

- RBAC権限の定期レビュー  
- 不要アクセスの削除  
- アクセス状況のレポート生成  

さらにLogic Appsと組み合わせることで、権限の取り消しや月次レポート送信を自動化できます。

そのため、このシナリオでは **Access Reviews + Logic Apps** の組み合わせが最適なソリューションになります。
