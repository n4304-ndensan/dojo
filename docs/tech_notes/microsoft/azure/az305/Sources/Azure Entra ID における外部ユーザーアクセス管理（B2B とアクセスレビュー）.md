# Azure Entra ID における外部ユーザーアクセス管理（B2B とアクセスレビュー）

## 1. 背景（シナリオ）

あるグローバル企業では、複数の子会社を含む組織構造を持っています。現在、この企業では **Azure Entra ID（旧 Azure Active Directory）** を中心としたアイデンティティ管理基盤を設計しています。

今回の要件では、子会社 **Fabrikam** のユーザーが、親会社の Azure テナント内にある **特定のアプリケーションにアクセスする必要があります**。しかし、そのアクセス権は常に有効であってはならず、次のガバナンス要件があります。

- Fabrikam ユーザーのアクセスは **四半期ごとにレビューされる必要がある**
- レビューは **Fabrikam 側のマネージャー（外部テナント）** が実施する
- IT 管理部門の **管理オーバーヘッドは最小限に抑える必要がある**

つまり、このシナリオでは **外部ユーザーの安全なアクセス管理と定期的なガバナンス**を両立させる仕組みが必要になります。

---

## 2. 要件整理

この問題の設計要件を整理すると、以下のポイントが重要になります。

まず、ユーザーの所属です。

Fabrikam のユーザーは **自社の Azure テナントを持つ外部ユーザー**です。  
したがって、親会社のテナント内で **ローカルユーザーとして完全管理する必要はありません**。

次に、アクセス管理です。

外部ユーザーのアクセスは以下を満たす必要があります。

- 特定アプリケーションへのアクセス
- 四半期ごとのアクセスレビュー
- 外部マネージャーによる承認

さらに運用要件として次があります。

- IT 部門の管理負荷を最小化
- 自動化されたアクセス管理

---

## 3. 技術の基本概念

### Azure Entra ID B2B Collaboration

Azure Entra ID B2B（Business-to-Business）コラボレーションは、**外部組織のユーザーを安全に自社リソースへ招待する仕組み**です。

特徴は以下です。

- 外部ユーザーは **自分のテナントのIDを使用**
- 自社テナントには **ゲストユーザーとして登録**
- 外部組織との安全なコラボレーションを実現

つまり、ユーザーは Fabrikam テナントに所属したまま、親会社のアプリケーションにアクセスできます。

---

[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Access Reviews]]
### Azure Entra ID Access Reviews

Access Reviews は **アクセスガバナンス機能**の一つです。

これは次の目的で使用されます。

- ユーザーのアクセス権を定期的に確認
- 不要なアクセスを削除
- コンプライアンスを維持

Access Reviews の特徴は以下です。

- 定期レビュー（例：四半期）
- 外部ユーザーも対象可能
- レビュワーを指定可能（マネージャーなど）
- 未承認の場合の自動削除

これにより **自動化されたアクセス管理**が可能になります。

---

[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Identity Governance]]
### Identity Governance

Access Reviews は **Azure Entra ID Identity Governance** の一部です。

Identity Governance では以下を実現できます。

- アクセスレビュー
- エンタイトルメント管理
- ライフサイクル管理

これにより、**大規模な組織でも自動化されたアクセス管理**が可能になります。

---

## 4. アーキテクチャまたは設計のポイント

今回の設計では、次の2つを組み合わせることが重要です。

まず **Azure Entra ID B2B Collaboration** を使用します。

これは次の理由によります。

- Fabrikam ユーザーをゲストとして招待
- 元のテナントの ID を使用
- ユーザー管理を Fabrikam 側に維持

次に **Access Reviews** を設定します。

具体的には次のような設定になります。

- 対象：Fabrikam ドメインのユーザー
- レビュワー：Fabrikam のマネージャー
- 頻度：四半期
- 未承認：自動アクセス削除

この構成により、外部ユーザーのアクセスを **継続的に監査・管理**できます。

---

## 5. 設計判断（なぜこの構成になるか）

この構成が最適な理由は、**外部ユーザー管理とアクセスガバナンスを同時に実現できるため**です。

まず、B2B コラボレーションにより、Fabrikam ユーザーは **自社テナントの ID を維持したままアクセスできます**。

これにより、次の利点があります。

- アカウント管理の重複がない
- オンボーディングが簡単
- オフボーディングも自動化可能

次に、Access Reviews により **四半期レビューが自動化**されます。

これにより

- 外部マネージャーがレビュー
- 不要アクセスの自動削除
- IT部門の手動管理削減

というメリットがあります。

---

## 6. 他の選択肢が誤りな理由

### A. Fabrikam ユーザーごとにゲストアカウントを作成

この方法では、IT 部門が以下を管理する必要があります。

- ユーザー作成
- ライフサイクル管理
- パスワード管理

また、四半期レビューの自動化が困難です。

結果として **管理オーバーヘッドが増加します**。

---

### C. Fabrikam 用の別テナント + B2B Direct Connect

この構成は次の問題があります。

- テナント管理の複雑化
- クロステナント設定
- 運用負荷増加

また、アクセスレビューの要件を **直接満たすわけではありません**。

---

### D. Microsoft アカウント（MSA）を発行

MSA は **個人アカウント**です。

企業環境では次の問題があります。

- 中央管理不可
- セキュリティポリシー適用困難
- 監査ログ不足

そのため **企業 IAM には適していません**。

---

## 7. 最終回答

**B. Azure AD（Entra ID）B2B コラボレーションを使用し、Fabrikam ユーザー向けにアクセスレビューを作成する**

---

## 8. まとめ

この問題の重要なポイントは **外部ユーザーアクセス管理とアクセスガバナンスの自動化**です。

Azure Entra ID では、外部ユーザー管理のベストプラクティスとして **B2B Collaboration** が提供されています。

これにより

- 外部ユーザーをゲストとして招待
- 元の組織の ID を維持
- 安全なコラボレーション

が可能になります。

さらに **Access Reviews** を組み合わせることで

- 定期的なアクセス確認
- 外部マネージャーによる承認
- 不要アクセスの自動削除

が実現できます。

結果として、**セキュリティと管理効率を両立した IAM 戦略**を構築できます。
