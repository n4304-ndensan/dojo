# Azure ExpressRoute Global Reach を用いたマルチサイトネットワーク設計

（Azure → オンプレミス インターネット出口の自動フェールオーバー）

---

# 1 背景

企業は複数のデータセンターを運用しており、オンプレミス環境と Azure クラウドを接続してハイブリッドネットワークを構築している。

今回の構成では、以下の環境が存在する。

オンプレミス拠点

- ニューヨーク（NY）
    
- ロサンゼルス（LA）
    

Azureリージョン

- East US
    
- West US
    

[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Azure Virtual Network (VNet)]]
各 Azure 仮想ネットワーク (VNet) は **ExpressRoute** を利用してオンプレミスネットワークと接続されている。また、**ExpressRoute Global Reach** によってオンプレミス拠点同士も接続されている。

この環境では、Azure からインターネットへ出るトラフィックを **オンプレミスのインターネット出口（オンプレミスサイト）経由**で送る必要がある。

さらに次の要件がある。

- Azure のアウトバウンドインターネット通信はオンプレミス経由
    
- 最も近いオンプレミスサイトを使用
    
- サイト障害時は自動フェールオーバー
    
- 手動操作なしでルーティング変更
    

---

# 2 ネットワーク課題

Azure → インターネット通信のルートは次のようになる。

```
Azure VNet
     ↓
ExpressRoute
     ↓
オンプレミス（NY または LA）
     ↓
インターネット
```

問題は次の点である。

1. Azure はどちらのオンプレミス拠点を使うか判断する必要がある
    
2. どちらかの拠点が停止した場合、自動的に切り替える必要がある
    
3. 手動ルート変更は不可
    

このような **動的ルーティングとフェールオーバー** を実現するには、Azure とオンプレミス間で **BGP (Border Gateway Protocol)** を使用する必要がある。

---

# 3 BGP の役割

ExpressRoute ではルーティングに **BGP** が使用される。

BGP の主な機能は次の通り。

- ルート情報の動的交換
    
- 最適経路の選択
    
- 障害検知
    
- 自動フェールオーバー
    

Azure はオンプレミスのルーターと BGP セッションを確立し、以下のルート情報を交換する。

```
Azure VNet → オンプレミスネットワーク
オンプレミス → Azure VNet
```

---

# 4 ExpressRoute + BGP の動作

今回の構成では次のようなルーティングになる。

```
          NY Datacenter
            │
            │
            ▼
East US VNet ── ExpressRoute
            │
            │
            ▼
West US VNet ── ExpressRoute
            │
            │
            ▼
          LA Datacenter
```

BGP により Azure は次の情報を学習する。

- NY 経由のインターネットルート
    
- LA 経由のインターネットルート
    

そして BGP 属性に基づいて最適なルートを選択する。

---

# 5 フェールオーバーの仕組み

例えば次の状態を考える。

通常時

```
East US → NY → Internet
West US → LA → Internet
```

もし NY データセンターが停止した場合

```
East US → LA → Internet
West US → LA → Internet
```

この切り替えは **BGP のルート再収束 (convergence)** によって自動的に行われる。

BGP は以下を検知する。

- BGP セッションダウン
    
- ルート撤回
    
- 経路更新
    

これにより Azure は別ルートへ自動的に切り替える。

---

# 6 BGP を使うメリット

### 動的ルーティング

ネットワーク変更があってもルートを自動更新できる。

---

### 自動フェールオーバー

リンクやサイトがダウンした場合、別のルートへ自動切替される。

---

### 最適経路選択

BGP は以下の属性を利用してルート選択できる。

- AS Path
    
- Local Preference
    
- MED
    
- Weight
    

これにより **最も近いオンプレミスサイト**を優先させることが可能。

---

# 7 他の選択肢が不適切な理由

## UDR (User Defined Route)

UDR は Azure 内で静的ルートを設定する機能である。

問題点

- 動的ルート変更不可
    
- フェールオーバー不可
    
- 手動変更が必要
    

よってマルチサイト構成には適さない。

---

## HSRP

HSRP は Cisco ルーターのゲートウェイ冗長化プロトコルである。

特徴

- LAN 内の仮想ゲートウェイ
    
- Azure ルーティングに影響しない
    

Azure 側のトラフィック制御には利用できない。

---

## VRRP

VRRP もオンプレミス LAN のゲートウェイ冗長化プロトコルである。

Azure ExpressRoute のルート選択には関与しない。

---

# 8 推奨アーキテクチャ

最適な構成は次の通り。

```
Azure VNet
    │
    │ BGP
    ▼
ExpressRoute
    │
    │
    ├── NY Datacenter
    │
    └── LA Datacenter
```

BGP を使用することで

- 最適ルート選択
    
- 自動フェールオーバー
    
- 動的ルーティング
    

が実現できる。

---

# 10 まとめ

Azure とオンプレミスのハイブリッドネットワークでは、ExpressRoute 接続のルーティングは **BGP を利用するのが標準的な設計**である。

BGP を利用することで

- 複数拠点のルート学習
    
- 自動フェールオーバー
    
- 最適ルート選択
    

を実現できる。

そのため、今回の要件である

- 最も近いオンプレミスサイト利用
    
- 自動フェールオーバー
    
- マルチリージョン
    

を満たす最適な構成は **BGP によるルーティングとフェールオーバー**である。
