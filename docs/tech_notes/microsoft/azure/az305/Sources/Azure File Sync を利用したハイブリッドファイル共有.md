# Azure File Sync を利用したハイブリッドファイル共有

## 1. 背景（シナリオ）

ある組織ではオンプレミスネットワークと Azure サブスクリプションの両方を使用しています。複数の支店やオフィスに分散しているユーザーが、同じ共有ファイルサーバーにアクセスできる環境を構築する必要があります。

しかし、この環境では単にファイルを共有できるだけでは不十分です。主要な拠点のサーバーが障害によってアクセス不能になった場合でも、ユーザーがファイルへアクセスできるようにする必要があります。つまり、システムには高可用性と災害耐性が求められます。

また、ユーザーの多くは従来のファイルサーバーと同じように SMB プロトコルを使用した共有フォルダを利用しているため、既存のファイルアクセス方式との互換性も重要になります。

## 2. 要件整理

このシナリオの要件を整理すると、次のようなポイントが見えてきます。

まず、複数の支店やオフィスのユーザーが共有ファイルにアクセスできる必要があります。つまり、ファイル共有サービスを提供する必要があります。

次に、オンプレミス環境と Azure 環境の両方を活用するハイブリッド構成が必要です。これにより、ローカルアクセスのパフォーマンスとクラウドの可用性を両立できます。

さらに、主要な拠点が利用できなくなった場合でも、ユーザーがファイルへアクセスできるようにする必要があります。つまり、災害復旧（Disaster Recovery）と高可用性が重要です。

このシナリオの要件は次の通りです。

- 複数拠点からの共有ファイルアクセス  
- オンプレミスと Azure のハイブリッド構成  
- 高可用性と災害復旧  
- SMB ファイル共有のサポート  

## 3. 技術の基本概念

Azure ではクラウドベースのファイル共有サービスとして **Azure Files** が提供されています。Azure Files は SMB プロトコルを使用してアクセスできるマネージドファイル共有サービスです。

これにより、ユーザーは Azure のファイル共有をネットワークドライブとしてマウントし、オンプレミスのファイルサーバーと同じように利用できます。

さらに、**Azure File Sync** を使用すると、オンプレミスの Windows Server と Azure Files を同期できます。これにより、クラウドとオンプレミスの両方にファイルデータを保持できます。

## 4. アーキテクチャまたは設計のポイント

Azure File Sync を使用したハイブリッドファイル共有では、Azure Files をクラウド側の中央ストレージとして使用します。

次に、各支店やオフィスの Windows Server に Azure File Sync エージェントをインストールします。これにより、オンプレミスサーバーと Azure Files の間でファイルが同期されます。

この構成では次のような仕組みが実現されます。

- Azure Files がクラウドの中央ファイルストレージとして機能する  
- 各支店の Windows Server がローカルキャッシュとして動作する  
- Azure File Sync がファイルを自動同期する  
- 主要拠点が停止しても Azure Files にアクセスできる  

この仕組みにより、ローカルアクセスの高速性とクラウドの可用性を両立できます。

## 5. 設計判断（なぜこの構成になるか）

Azure File Sync はオンプレミスのファイルサーバーと Azure Files を同期するためのサービスです。これにより、ユーザーはローカルサーバーから高速にファイルへアクセスできる一方、クラウドにもデータが保存されます。

もし主要拠点のファイルサーバーが障害で停止した場合でも、ユーザーは Azure Files に直接アクセスできます。これにより、業務の継続性を確保できます。

また、Azure File Sync は複数のサーバー間で同期できるため、複数の支店に同じデータを配布できます。この機能により、分散環境でも効率的なファイル共有が可能になります。

## 6. 他の選択肢が誤りな理由

Azure Blob Storage と Azure CDN は主に静的コンテンツ配信のために使用されます。ファイル共有や共同編集用途には適していません。

Azure Backup はバックアップと復元のためのサービスであり、ユーザーが直接ファイル共有として利用するためのサービスではありません。

Azure Recovery Services Vault もバックアップおよび災害復旧の管理サービスであり、リアルタイムのファイル共有機能は提供しません。

## 7. 最終回答

A. Azure Files と Azure File Sync

## 8. まとめ

複数の支店から共有ファイルサーバーへアクセスできる環境を構築する場合、オンプレミスとクラウドのハイブリッド構成が有効です。

Azure Files はクラウド上の SMB ファイル共有を提供し、Azure File Sync を利用することでオンプレミスサーバーとクラウドのデータを同期できます。

この構成により、ローカルアクセスの高速性とクラウドの高可用性を両立できるため、拠点障害時でもユーザーはファイルにアクセスできます。
