# Azure Front Door と Application Gateway によるグローバルアプリケーションのトラフィック最適化

## 1. 背景（シナリオ）

ある組織では、Azure 上でグローバルに利用されるアプリケーションを設計しています。このアプリケーションは世界中のユーザーからアクセスされるため、ユーザーができるだけ近いアプリケーションインスタンスへ接続できるようにする必要があります。

また、ユーザー体験を最大化するために、単に地理的な距離だけではなく、リアルタイムのパフォーマンス指標（レイテンシなど）を考慮してトラフィックを分散する仕組みが必要です。

さらに、このアプリケーションでは高可用性が必須であり、ユーザーの通信データを保護するために SSL 終了（SSL Termination）を実装する必要があります。そのため、グローバルレベルのトラフィック管理とアプリケーションレベルのロードバランシングを組み合わせたアーキテクチャが求められています。

## 2. 要件整理

このシナリオの要件を整理すると、グローバルアプリケーションに必要なネットワーク機能が明確になります。

まず、ユーザーの地理的位置やネットワーク遅延を考慮して、最も近いアプリケーションインスタンスへルーティングする必要があります。これはグローバルロードバランシングの機能に該当します。

次に、アプリケーションの高可用性を確保する必要があります。障害が発生した場合でもトラフィックを別のリージョンへルーティングできる仕組みが必要です。

さらに、通信の安全性を確保するために SSL 終了を実装する必要があります。これにより、ユーザーとサービス間の通信が暗号化されます。

この問題の要件は次のように整理できます。

・グローバルレベルのトラフィック分散  
・地理的位置およびリアルタイムパフォーマンスに基づくルーティング  
・高可用性の確保  
・SSL 終了による通信の保護  
・アプリケーション層でのインテリジェントルーティング  

## 3. 技術の基本概念

Azure ではアプリケーションのトラフィック管理のために複数のロードバランシングサービスが提供されています。その中でもグローバルアプリケーションでよく利用されるサービスが **Azure Front Door** と **Azure Application Gateway** です。

まず Azure Front Door はグローバルレベルのロードバランシングサービスです。Microsoft のエッジネットワークを利用して、ユーザーのリクエストを最も適したバックエンドへルーティングします。

Azure Front Door には次のような特徴があります。

・グローバルロードバランシング  
・レイテンシベースルーティング  
・SSL 終了  
・動的サイトアクセラレーション  
・Web Application Firewall（WAF）

次に Azure Application Gateway はアプリケーション層（Layer 7）で動作するロードバランサーです。HTTP / HTTPS トラフィックを理解し、URL やホスト名に基づくルーティングが可能です。

Application Gateway の主な特徴は次の通りです。

・Layer 7 ロードバランシング  
・URL パスベースルーティング  
・SSL 終了  
・Web Application Firewall  
・自動スケーリング

## 4. アーキテクチャまたは設計のポイント

グローバルアプリケーションでは、エッジレベルのトラフィック管理とリージョン内部のロードバランシングを組み合わせる設計が一般的です。

Azure Front Door を使用すると、ユーザーの地理的位置やネットワークレイテンシを考慮して最適なリージョンへトラフィックをルーティングできます。これにより、世界中のユーザーに対して低遅延のアクセスを提供できます。

一方、Azure Application Gateway は各リージョン内でのトラフィック分散を担当します。URL パスやホストヘッダーに基づいてバックエンドサービスへリクエストを振り分けることができます。

この構成では、Front Door がグローバル入口として機能し、Application Gateway がリージョン内のアプリケーションロードバランサーとして機能します。

## 5. 設計判断（なぜこの構成になるか）

Azure Front Door はグローバルレベルのトラフィック最適化機能を提供します。ユーザーの位置とリアルタイムのパフォーマンスデータを基に最適なバックエンドへルーティングすることができます。

また、Front Door はエッジで SSL 終了を行うことができるため、バックエンドサービスの負荷を軽減できます。

さらに Application Gateway を組み合わせることで、リージョン内部での Layer 7 ルーティングが可能になります。URL ベースのルーティングや WAF によるセキュリティ保護を実装できます。

この2つのサービスを組み合わせることで、グローバルスケールのトラフィック管理とアプリケーションレベルのロードバランシングを同時に実現できます。

## 6. 他の選択肢が誤りな理由

この問題では他にも複数のサービスの組み合わせが提示されていますが、それぞれ機能が不足しています。

Azure Traffic Manager は DNS ベースのトラフィックルーティングを提供しますが、SSL 終了やアプリケーションレベルのルーティング機能を提供しません。

Azure Load Balancer は Layer 4 ロードバランサーであり、HTTP / HTTPS の内容に基づくルーティングや SSL 終了機能を提供しません。

そのため、グローバルアプリケーションで必要となる高度なトラフィック管理や SSL 終了の要件を満たすことができません。

## 7. 最終回答

B. Azure Front Door と Azure Application Gateway

## 8. まとめ

グローバルアプリケーションでは、ユーザーの地理的位置やネットワークパフォーマンスを考慮したトラフィック管理が重要になります。

Azure Front Door はエッジネットワークを利用したグローバルロードバランシングを提供し、ユーザーを最適なリージョンへルーティングします。

さらに Azure Application Gateway を組み合わせることで、リージョン内部でのアプリケーションレベルのロードバランシングやセキュリティ機能を実装できます。

このアーキテクチャにより、高可用性・低遅延・セキュリティを兼ね備えたグローバルアプリケーションを構築することができます。
