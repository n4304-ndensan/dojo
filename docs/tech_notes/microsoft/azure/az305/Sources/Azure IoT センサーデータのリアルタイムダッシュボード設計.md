# Azure IoT センサーデータのリアルタイムダッシュボード設計

[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Azure Event Hubs]]
（Azure Event Hubs + Azure Time Series Insights）

---

# 1 背景とシナリオ

製造業や設備監視の分野では、センサーから生成される大量のデータをリアルタイムで収集し、状態監視や異常検知を行うことが重要になる。例えば、工場の設備には温度、振動、圧力、電流などのセンサーが取り付けられており、これらの値を継続的に収集することで、機器の異常や故障の兆候を早期に検知できる。

今回のシナリオでは、製造設備から送信されるセンサーデータをクラウドで収集し、リアルタイムダッシュボードで可視化する仕組みを構築する必要がある。データは Azure Event Hubs に取り込まれ、ユーザーはダッシュボード上で以下の操作を行うことが求められている。

- ほぼリアルタイムでデータを表示する
    
- インタラクティブなフィルタリング（設備・時間・センサー）
    
- 集約処理（平均・最大・最小など）
    
- 大量の時系列データを高速に検索
    

このような要件を満たす Azure サービスとして最も適しているのが **Azure Time Series Insights（TSI）** である。

---

# 2 全体アーキテクチャ

典型的な IoT データ可視化アーキテクチャは次のようになる。

```
Manufacturing Sensors
        │
        ▼
IoT Devices / Gateway
        │
        ▼
Azure Event Hubs
        │
        ▼
Azure Time Series Insights
        │
        ▼
Interactive Dashboard
```

各コンポーネントの役割は以下の通り。

|コンポーネント|役割|
|---|---|
|Sensors|温度・振動などのデータ生成|
|Event Hubs|大量のイベントデータの取り込み|
|Time Series Insights|時系列データの保存・分析|
|Dashboard|データ可視化|

---

# 3 Azure Event Hubs の役割

Azure Event Hubs は、大量のイベントデータをリアルタイムで取り込むためのストリーミングプラットフォームである。IoT シナリオでは、数千台〜数百万台のデバイスから送信されるテレメトリデータを受信するために使用される。

特徴は以下の通り。

- 毎秒数百万イベントを処理可能
    
- パーティションによるスケーリング
    
- ストリームデータのバッファリング
    
- IoT データパイプラインとの統合
    

製造設備のセンサーデータは Event Hubs に送信され、そこから分析サービスへとストリームされる。

---

# 4 Azure Time Series Insights (TSI)

Azure Time Series Insights は **IoT やセンサーデータのような時系列データを分析・可視化するためのサービス**である。

時系列データとは、時間を軸として連続的に記録されるデータのことである。

例

```
Time        Temperature
10:00       22.1
10:01       22.3
10:02       22.4
10:03       23.0
```

TSI はこのようなデータを効率的に保存し、リアルタイムに近い形で検索・分析できる。

---

# 5 Time Series Insights の主な機能

## 5.1 リアルタイムデータ取り込み

TSI は Event Hubs と直接接続できるため、データはほぼリアルタイムで取り込まれる。

```
Event Hubs → TSI ingestion
```

そのため、センサーの状態変化を即座にダッシュボードに反映できる。

---

## 5.2 時系列データの高速検索

TSI は時間軸ベースのデータに最適化されたストレージを使用している。そのため、次のようなクエリを高速に実行できる。

例

- 特定設備の温度推移
    
- 直近1時間の平均値
    
- 異常値の検出
    

---

## 5.3 インタラクティブなフィルタリング

TSI のダッシュボードでは、ユーザーが次の条件でデータをフィルタできる。

- デバイスID
    
- センサータイプ
    
- 時間範囲
    
- 工場ライン
    

例

```
Factory A
   └ Machine 12
        └ Temperature sensor
```

---

## 5.4 集約処理

TSI は次のような集約関数をサポートしている。

- Average
    
- Max
    
- Min
    
- Percentile
    

例

```
平均温度
最大振動値
1時間ごとの平均電流
```

---

## 5.5 ビルトインダッシュボード

TSI には標準の可視化 UI があり、追加ツールなしでデータ探索が可能である。

表示できるもの

- 時系列グラフ
    
- 集約グラフ
    
- センサーデータ比較

# 6 他の選択肢との比較

[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Power BI]]
## Power BI Streaming Dataset

Power BI でもストリーミングデータを表示することは可能である。

しかし次の課題がある。

- データを Power BI にプッシュする必要がある
    
- 大量 IoT データに最適化されていない
    
- 時系列分析機能が限定的
    

そのため IoT センサーデータ用途では最適ではない。

---

## Azure Dashboard

Azure Dashboard は Azure リソースのメトリクスを可視化するためのダッシュボードである。

しかし以下の理由で不適切である。

- Event Hubs のメトリクスは取得できるが
    
- センサーデータの分析はできない
    
- 時系列分析機能がない
    

---

[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Azure Data Explorer]]
## Azure Data Explorer

Azure Data Explorer（Kusto）はログ分析やビッグデータ分析に強力なサービスである。

ただし IoT シナリオでは次の課題がある。

- Event Hubs からの ingestion 設定が必要
    
- ダッシュボード構築が必要
    
- IoT 特化ではない
    

TSI は IoT 向けに最適化されているため、より簡単に構築できる。

---

# 7 なぜ Azure Time Series Insights が最適なのか

今回の要件を整理すると次の通り。

|要件|必要な機能|
|---|---|
|リアルタイム可視化|ストリーム取り込み|
|フィルタリング|時系列クエリ|
|集約|時系列分析|
|大量センサーデータ|スケーラブルなストレージ|

TSI はこれらを **ネイティブで提供する唯一のサービス**である。

---

# 8 最終アーキテクチャ

製造設備ダッシュボードの構成は次のようになる。

```
Sensors
   │
   ▼
IoT Gateway
   │
   ▼
Azure Event Hubs
   │
   ▼
Azure Time Series Insights
   │
   ▼
Real-Time Dashboard
```

この構成により

- 高速データ取り込み
    
- ほぼリアルタイム分析
    
- インタラクティブダッシュボード
    

を実現できる。

---

# 9 まとめ

製造設備のセンサーデータをリアルタイムに可視化する場合、以下の要件が重要になる。

- 大量の時系列データ処理
    
- リアルタイム分析
    
- インタラクティブダッシュボード
    
- Event Hubs との統合
    

これらの要件を満たす最適な Azure サービスは

**Azure Time Series Insights**

である。

TSI は IoT センサーデータに特化した分析プラットフォームであり、リアルタイムダッシュボードを構築する最も効率的な方法となる。
