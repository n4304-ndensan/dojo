# Azure Key Vault のディザスタリカバリと地域コンプライアンス設計

## 1. 背景（シナリオ）
Azure Key Vault「SecureVault」を使用している組織では、米国東部リージョンで運用されています。災害時にすべての鍵と秘密を復旧できる戦略を設計する必要があります。さらに、コンプライアンス要件により、データは同じ地理的境界内で保持されなければなりません。

## 2. 要件整理
- **災害復旧:** リージョン全体の停電や障害が発生した場合でも、鍵と秘密を復元可能にする。
- **コンプライアンス:** データは作成されたAzureリージョン（East US）内に留まる必要がある。
- **技術的制約:** Key Vault のバックアップは同じリージョンでのみ復元可能。

## 3. 技術の基本概念
- **Azure Key Vault バックアップ:** バックアップ機能により、鍵や秘密の保護と復元が可能。
- **地域限定復元:** Key Vault の設計上、バックアップされたアイテムは作成リージョン内でのみ復元できる。
- **地理的境界:** データ主権やコンプライアンス要件により、復元可能なリージョンは制限される。

## 4. アーキテクチャまたは設計のポイント
- SecureVault のバックアップは必ず**東アメリカリージョン**内で復元。
- 別リージョンへの復元はコンプライアンス違反となる。
- 災害復旧計画では、リージョン内での可用性確保に注力する。

## 5. 設計判断（なぜこの構成になるか）
- TDEやキー管理などの機密情報は、同じリージョン内で復元することで地理的コンプライアンスを確保。
- Azure の公式ドキュメントでも、Key Vault バックアップの復元は作成リージョンでのみ可能であると明示。

## 6. 他の選択肢が誤りな理由
- **A. 米国内すべてのリージョン:** コンプライアンス要件で指定されたリージョン外への復元は不可。
- **C. 東アメリカ内の別リージョン:** Azure では「East US」という単一リージョンが指定され、サブリージョン間復元はサポートされない。
- **D. 東アメリカと同じ地政学的境界のすべてのリージョン:** 地政学的境界は複数リージョンにまたがる場合があるが、Key Vault バックアップ復元は作成リージョンに限定。

## 7. 最終回答
**B. 東アメリカ地域のみ**

## 8. まとめ
- Azure Key Vault バックアップは作成したリージョンでのみ復元可能。
- コンプライアンス要件に従い、SecureVault は東アメリカ地域内でのみ復元する。
- この構成により、災害復旧の要件と地域的なデータ保持要件を同時に満たす。
