# Azure Monitor を使用した VM の包括的モニタリング（Metrics Alerts + Log Analytics）

## 1. 背景（シナリオ）

ある企業では、Azure 上で複数の仮想マシン（VM）を運用しており、その上で重要なアプリケーションが稼働しています。これらのアプリケーションの可用性とパフォーマンスを維持するためには、VM の状態を継続的に監視することが必要です。

企業は、CPU 使用率やメモリ使用率などのパフォーマンス指標を監視し、異常が発生した場合には即座にアラートを受け取れるようにしたいと考えています。また、トラブルシューティングやパフォーマンス改善のために、過去のデータを分析してトレンドを把握することも重要です。

さらに、VM 上で実行されているアプリケーションのログやシステムログを収集し、それらを分析することで問題の原因を特定できるようにする必要があります。このような包括的な監視環境を実現するためには、Azure Monitor の複数の機能を組み合わせる必要があります。

## 2. 要件整理

このシナリオでは、Azure VM の監視ソリューションとして次の要件が提示されています。

まず、CPU 使用率やディスク I/O などのパフォーマンス指標を収集する必要があります。これにより、システムの状態をリアルタイムに把握できます。

次に、特定のパフォーマンスの閾値を超えた場合にアラートを発生させる必要があります。これにより、問題が発生した際に迅速な対応が可能になります。

さらに、ログやパフォーマンスデータを長期間保存し、過去のデータを分析してトレンドを確認できるようにする必要があります。

これらの要件をまとめると次の通りです。

- 仮想マシンのパフォーマンスメトリクスを収集する  
- 特定の閾値を超えた場合にアラートを発生させる  
- ログデータを収集して分析する  
- 過去のデータを分析してトレンドを把握する  

## 3. 技術の基本概念

Azure Monitor は Azure 環境の監視と診断を行うための中心的なサービスです。Azure Monitor には、主に **Metrics** と **Logs** という2種類のデータがあります。

Metrics は数値データであり、CPU 使用率やメモリ使用率などのパフォーマンス指標を表します。これらのデータは高頻度で収集されるため、リアルタイムに近い監視が可能です。

Logs はシステムイベントやアプリケーションログなどの詳細な情報を含むデータです。ログデータは Log Analytics ワークスペースに保存され、Kusto Query Language（KQL）を使用して分析できます。

Azure Monitor では、これらのデータを使用してアラートを作成することができます。特に Metrics Alerts は、CPU 使用率などの数値データが指定した閾値を超えた場合にアラートを発生させる機能です。

## 4. アーキテクチャまたは設計のポイント

このシナリオでは、リアルタイムのパフォーマンス監視とログ分析の両方が必要になります。そのため、Azure Monitor の Metrics Alerts と Log Analytics を組み合わせる構成が適しています。

まず、Azure Monitor Metrics を利用して VM のパフォーマンスメトリクスを収集します。これには CPU 使用率、メモリ使用率、ディスク I/O、ネットワークトラフィックなどが含まれます。

次に、Metrics Alerts を設定して、特定の閾値を超えた場合にアラートを発生させます。例えば、CPU 使用率が 80% を超えた場合に通知を送るように設定できます。

さらに、ログデータを Log Analytics ワークスペースに送信し、KQL クエリを使用してログ分析を行います。これにより、履歴データの分析やトレンド分析が可能になります。

## 5. 設計判断（なぜこの構成になるか）

この問題の正解は **A. パフォーマンスのしきい値には Metrics Alerts を使用し、ログデータ分析には Log Analytics を使用する** です。

Metrics Alerts は Azure Monitor のメトリクスデータに基づいてアラートを発生させる機能です。CPU 使用率やディスク使用率などの指標に閾値を設定することで、パフォーマンス問題を迅速に検知できます。

一方、Log Analytics はログデータを収集・分析するためのサービスです。KQL クエリを使用することで、過去のログデータを分析し、パフォーマンスのトレンドや問題の原因を特定できます。

この2つを組み合わせることで、リアルタイムの監視と履歴データ分析の両方を実現できます。

## 6. 他の選択肢が誤りな理由

Application Insights は主にアプリケーションレベルの監視を目的としたサービスであり、VM のインフラメトリクスの監視には適していません。また、Azure Monitor Workbook はデータの可視化ツールであり、アラート機能を提供するものではありません。

Log Analytics ワークスペースとアクショングループの組み合わせはログ分析には有効ですが、パフォーマンスメトリクスに基づくリアルタイムアラートを直接提供するものではありません。

Metrics Explorer はメトリクスの可視化には役立ちますが、アラート機能や履歴分析機能を包括的に提供するものではありません。また、診断設定はログを送信する設定であり、分析やアラート機能そのものではありません。

## 7. 最終回答

A. パフォーマンスのしきい値には Metrics Alerts を使用し、ログデータ分析には Log Analytics を使用します。

## 8. まとめ

この問題は、Azure Monitor の主要機能を理解しているかを確認するものです。Azure Monitor はメトリクスとログの2種類のデータを中心に監視機能を提供します。

Metrics Alerts を使用することで、CPU 使用率などのパフォーマンス指標に基づいてリアルタイムにアラートを発生させることができます。一方、Log Analytics を使用すると、ログデータを収集して過去のデータ分析やトレンド分析を行うことができます。

したがって、Azure Virtual Machines の包括的な監視、アラート、履歴分析を実現するための最適な組み合わせは Metrics Alerts と Log Analytics です。
