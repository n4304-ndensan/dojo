# Azure Network Security Group (NSG) によるネットワークセキュリティ制御

## 1. 背景（シナリオ）

Azure環境でアプリケーションを運用する場合、仮想ネットワーク内のリソースを安全に保護することが重要になります。特にインターネットなどの外部ソースからのトラフィックが直接内部リソースに到達してしまうと、セキュリティリスクが高まります。

そのため、仮想ネットワーク内の特定のリソースに対してのみトラフィックを許可し、それ以外のアクセスを制御する仕組みが必要になります。具体的には、どのIPアドレスからどのポートへの通信を許可するのか、または拒否するのかといったネットワークレベルでの制御を行う必要があります。

このような要件を満たすためには、Azureのネットワーク層でトラフィックをフィルタリングできるサービスを選択する必要があります。

---

## 2. 要件整理

この問題のシナリオから読み取れる要件を整理すると、次のようになります。

まず、外部からのトラフィックをフィルタリングする必要があります。これは、不要なアクセスを遮断し、許可された通信のみを通過させるためです。

次に、仮想ネットワーク内の特定のリソースのみアクセス可能にする必要があります。つまり、IPアドレスやポート番号に基づいたアクセス制御が必要になります。

さらに、ネットワーク層でトラフィックの流れを制御する必要があります。

要件をまとめると次の通りです。

- 外部トラフィックのフィルタリング  
- 仮想ネットワーク内リソースの保護  
- IPアドレスやポートによるアクセス制御  
- ネットワーク層でのトラフィック制御  

---

## 3. 技術の基本概念

Azureでは、仮想ネットワークのトラフィック制御を行うためにNetwork Security Group（NSG）を使用します。

Network Security Groupは、Azure Virtual Network内のトラフィックを制御するためのセキュリティルールの集合です。これらのルールを利用することで、インバウンドおよびアウトバウンド通信を制御できます。

NSGでは、次のような条件に基づいてトラフィックを制御できます。

- 送信元IPアドレス  
- 宛先IPアドレス  
- プロトコル（TCP / UDP）  
- ポート番号  
- 通信方向（Inbound / Outbound）

このように、ネットワーク層（Layer 3 / Layer 4）でトラフィックを制御できます。

---

## 4. アーキテクチャまたは設計のポイント

NSGはAzureネットワークのさまざまなレベルに適用できます。どこに適用するかによって、制御範囲が変わります。

代表的な適用先は次の2つです。

まず、サブネットに適用する方法があります。この場合、そのサブネットに属するすべてのリソースに同じセキュリティルールが適用されます。

次に、ネットワークインターフェース（NIC）に適用する方法があります。この方法では、特定の仮想マシンのみを対象にルールを適用できます。

このようにNSGを利用することで、ネットワークレベルでのセキュリティ境界を作ることができます。

例えば、Webサーバーに対しては次のようなルールを設定できます。

- インターネットからのHTTP（80）を許可  
- インターネットからのHTTPS（443）を許可  
- その他のポートを拒否  

このようなルールにより、最小権限のネットワークアクセスを実現できます。

---

## 5. 設計判断（なぜこの構成になるか）

この問題の重要なポイントは「ネットワーク層でトラフィックを制御する」という点です。

Network Security Groupは、Azureの仮想ネットワークにおいて最も基本的なネットワークセキュリティ機能です。IPアドレスやポート番号を基に通信を許可または拒否することができ、仮想マシンやその他のリソースを直接保護できます。

さらに、NSGは次のような特徴を持っています。

- レイヤー3 / レイヤー4でのトラフィック制御  
- サブネットまたはNIC単位で適用可能  
- Azureの基本ネットワークセキュリティ機能  
- 追加コストなしで利用可能  

そのため、仮想ネットワーク内のリソースを保護する場合には最適な選択になります。

---

## 6. 他の選択肢が誤りな理由

Azure Firewallは高度なネットワークセキュリティサービスであり、中央集約型のファイアウォールとして利用されます。しかし、この問題のような基本的なネットワークアクセス制御にはNSGの方が適しています。

Azure Load Balancerはトラフィックを複数のサーバーに分散するためのサービスであり、トラフィックのフィルタリングやセキュリティ制御を目的としたサービスではありません。

Azure Application GatewayはLayer7のロードバランサーであり、主にWebアプリケーションのトラフィック管理やWAF機能に使用されます。一般的なネットワークトラフィック制御には適していません。

---

## 7. 最終回答

A.  
ネットワークセキュリティグループ（NSG）

---

## 8. まとめ

Azureで仮想ネットワークのトラフィックを制御する場合、Network Security Group（NSG）が基本となるセキュリティ機能です。

NSGを使用すると次のことが可能になります。

- インバウンド / アウトバウンド通信の制御  
- IPアドレスやポートに基づくフィルタリング  
- サブネットまたはNIC単位でのセキュリティ適用  

これにより、仮想ネットワーク内のリソースをネットワーク層で安全に保護することができます。
