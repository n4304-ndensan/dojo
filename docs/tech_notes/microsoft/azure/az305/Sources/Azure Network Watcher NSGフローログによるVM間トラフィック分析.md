# Azure Network Watcher NSGフローログによるVM間トラフィック分析

## 1. 背景（シナリオ）
企業は Azure とオンプレミスで複数の仮想マシン(VM)を展開しており、ExpressRoute を介して接続されています。これらのVM間で通信上の問題が報告されており、ネットワークトラフィックを詳細に分析して、接続性に影響する可能性のあるパケットフィルタリングの問題を特定する必要があります。

## 2. 要件整理
- 仮想マシン間通信の問題を特定する。
- ネットワーク設定が複雑で、VMレベルでのトラフィックフローやフィルタリングルールの詳細な洞察が必要。
- トラブルシューティングに包括的なデータが求められる。

## 3. 技術の基本概念
**Azure Network Security Group (NSG) フローログ** は、NSG を通過するIPトラフィックに関する詳細な情報を収集するツールです。主な情報は以下の通りです。

- 送信元/宛先 IP アドレス、ポート、プロトコル
- NSG ルールに基づく許可/拒否の判定
- ログは Azure Storage に保存可能で、過去のトラフィック分析も可能

NSGフローログは Azure Network Watcher の機能の一部であり、VM間通信やパケットフィルタリングのトラブルシューティングに特化しています。

## 4. アーキテクチャまたは設計のポイント
- NSG フローログは、仮想マシンレベルでトラフィックの詳細を提供。
- トラフィックフローの可視化により、問題のあるパケットや誤ったフィルタリングルールを特定可能。
- Azure Monitor や Azure Sentinel と連携することで、アラート設定や可視化を通じた包括的な分析が可能。

## 5. 設計判断（なぜこの構成になるか）
このシナリオでは、VM間通信を正確に分析し、フィルタリングに起因する問題を特定する必要があります。そのため、以下の理由で **NSGフローログ** が最適です。

- 包括的なトラフィック分析が可能で、許可・拒否された通信を正確に把握できる。
- NSGルールに基づいたフィルタリングの可視性を提供。
- 過去の通信データを分析可能で、トラブルシューティング精度が高い。
- 他のツール（ファイアウォールやアプリケーションゲートウェイ）はVMレベルのトラフィック分析には特化していない。

## 6. 他の選択肢が誤りな理由
- **B. Azure ファイアウォールポリシー**: 中央集権的なトラフィック管理向けで、VMレベルの詳細なフロー分析には不十分。
- **D. Azure Application Gateway 診断**: Webアプリケーション層のトラフィック監視に特化しており、VM間通信のパケットフィルタリング問題とは関連が薄い。

## 7. 最終回答
**A. Azure Network Watcher NSG フローログ**

## 8. まとめ
NSGフローログは、VM間通信のトラフィックを詳細に分析し、パケットフィルタリングの問題を特定する最も包括的な手段です。ExpressRoute 経由の複雑なネットワーク環境でも、許可/拒否ルールの可視化と過去のログ分析により、通信障害の原因を効率的に特定できます。
