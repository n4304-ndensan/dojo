# Azure Network Watcher と NSG を使用したネットワークトラフィックのパケットレベル分析

## 1. 背景（シナリオ）

ある企業では、オンプレミス環境と Azure 環境を接続したハイブリッドネットワークを運用しています。最近、仮想マシンへ送信されるネットワークトラフィックに異常な挙動が確認されました。具体的には、送信されたパケットの一部が拒否されていることが判明しています。

この問題を解決するためには、ネットワークトラフィックの流れを詳細に分析し、どの時点でパケットが拒否されているのかを特定する必要があります。単にログを確認するだけではなく、パケットレベルで通信内容を確認できる仕組みが求められています。

また、この環境では継続的なセキュリティ監視も重要です。ネットワークトラフィックを監視しながら、どのルールによってトラフィックが拒否されているのかを分析できるツールの組み合わせが必要になります。

## 2. 要件整理

問題文から読み取れる要件を整理すると、ネットワーク診断には次の機能が必要になります。

まず、仮想マシンに送信されるトラフィックをパケットレベルで確認できる必要があります。つまり、通信の詳細を確認できるパケットキャプチャ機能が必要です。

次に、トラフィックが許可または拒否される理由を理解する必要があります。特に、どのセキュリティルールによってパケットがブロックされているのかを特定する必要があります。

さらに、ネットワークセキュリティの観点から、トラフィック制御ルールと監視機能の両方が必要になります。

これらの要件をまとめると次のようになります。

- パケットレベルでネットワークトラフィックを分析する  
- 仮想マシンへのトラフィックの流れを可視化する  
- 拒否されたパケットの原因を特定する  
- セキュリティルールによるトラフィック制御を確認する  

## 3. 技術の基本概念

Azure ではネットワーク監視と診断のために **Azure Network Watcher** が提供されています。Network Watcher は Azure 仮想ネットワークの監視、トラブルシューティング、診断を行うためのサービスです。

Network Watcher の主要機能の一つが **Packet Capture（パケットキャプチャ）** です。この機能を使用すると、Azure 仮想マシンのネットワークインターフェースで送受信されるパケットをキャプチャできます。キャプチャしたパケットは PCAP ファイルとして保存され、Wireshark などのツールで詳細に分析できます。

一方で、Azure のネットワークトラフィックを制御する主要な仕組みが **Network Security Group（NSG）** です。NSG は仮想ネットワークやサブネット、ネットワークインターフェースに適用できるセキュリティルールの集合であり、送信元 IP、宛先 IP、ポート番号、プロトコルなどの条件に基づいてトラフィックを許可または拒否します。

## 4. アーキテクチャまたは設計のポイント

このシナリオでは、ネットワークトラフィックの詳細分析とセキュリティ監視を同時に実現する必要があります。そのため、Network Watcher と NSG を組み合わせた構成が適しています。

まず、Azure Network Watcher の Packet Capture 機能を使用して、仮想マシンのネットワークインターフェースで送受信されるパケットを取得します。これにより、どのトラフィックが到達しているのか、どのパケットが拒否されているのかを詳細に分析できます。

次に、Network Security Group を利用して、トラフィックの許可または拒否ルールを管理します。NSG のルールを確認することで、どの条件によってパケットがブロックされているかを特定できます。

この2つのツールを組み合わせることで、ネットワーク通信の可視化とセキュリティ監視を同時に実現できます。

## 5. 設計判断（なぜこの構成になるか）

この問題の正解は **A. Azure Network Watcher Packet Capture と Azure Network Security Group** です。

Azure Network Watcher の Packet Capture 機能を使用すると、仮想マシンのネットワークトラフィックをパケットレベルでキャプチャできます。これにより、拒否されたパケットの詳細や通信内容を確認することができます。

一方、Network Security Group は Azure 仮想ネットワークのトラフィック制御を担当します。NSG のセキュリティルールを確認することで、どのトラフィックが許可され、どのトラフィックが拒否されているのかを理解できます。

この2つのサービスを組み合わせることで、ネットワークトラフィックの詳細分析とセキュリティ監視を実現できます。

## 6. 他の選択肢が誤りな理由

Azure Traffic Analytics と Azure Monitor Logs はトラフィックパターンの分析には役立ちますが、パケットレベルの検査は行えません。そのため、詳細なネットワーク診断には適していません。

Azure Application Gateway と Azure Firewall はトラフィック制御やアプリケーションセキュリティのためのサービスです。しかし、パケットキャプチャ機能や詳細なネットワーク診断機能は提供していません。

Azure Network Watcher の IP Flow Verify と NSG Flow Logs はトラフィックフローの確認には役立ちますが、パケットの内容を直接確認することはできません。そのため、パケットレベルの検査には適していません。

## 7. 最終回答

A. Azure Network Watcher Packet Capture および Azure Network Security Group

## 8. まとめ

この問題は、Azure におけるネットワーク診断ツールの役割を理解しているかを確認するものです。ネットワークトラフィックの詳細な分析を行うためには、通信内容を確認できるパケットキャプチャ機能が必要になります。

Azure Network Watcher の Packet Capture 機能を使用することで、仮想マシンのネットワークパケットを取得し、詳細なトラフィック分析が可能になります。また、Network Security Group を利用することで、トラフィック制御ルールを確認し、どのルールによって通信が拒否されているかを特定できます。

したがって、パケットレベルの検査とネットワークセキュリティ監視を実現する最適な組み合わせは Azure Network Watcher Packet Capture と Network Security Group です。
