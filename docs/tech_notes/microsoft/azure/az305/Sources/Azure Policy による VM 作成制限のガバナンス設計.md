# Azure Policy による VM 作成制限のガバナンス設計

## 1. 背景（シナリオ）

ある組織では、開発者が必要に応じて Azure 仮想マシンを作成できる **セルフサービス型のインフラ提供モデル** に移行しています。これにより、開発者は迅速に開発環境を用意できますが、一方で組織ポリシーへの準拠を確保する必要があります。

組織では、次のようなガバナンスルールを定めています。

- 仮想マシンは **指定された Azure リージョンのみ**で作成できる
- **承認された VM サイズ（SKU）** のみ使用できる
- 将来、許可リージョンやサイズを **簡単に更新できる必要がある**

このようなポリシーを実装する場合、開発者の自由度を保ちながら、組織のガバナンスを強制できる仕組みが必要になります。

---

## 2. 要件整理

この問題の要件を整理すると次のようになります。

まず、ガバナンス要件です。

- 許可されたリージョンのみ使用
- 許可された VM サイズのみ使用

次に、運用要件です。

- セルフサービスで VM 作成可能
- ポリシー違反のリソース作成を防止

さらに、柔軟性要件です。

- 許可リージョンの変更
- 許可 SKU の更新
- 再デプロイ不要

このような条件を満たすには **ポリシーベースのガバナンス機能**が必要になります。

---

## 3. 技術の基本概念

Azure ではリソースのガバナンスを実現するために **Azure Policy** が提供されています。

Azure Policy は、Azure リソースに対してルールを定義し、コンプライアンスを強制するサービスです。

Azure Policy の主な機能には次のものがあります。

まず、ポリシー定義です。

- リソースの構成ルールを定義
- 許可 / 拒否ルール

次に、ポリシー効果です。

- Deny（作成拒否）
- Audit（監査）
- Append（設定追加）
- Modify（設定変更）

さらに、コンプライアンス評価です。

- ポリシー違反リソースの検出
- コンプライアンスレポート

これにより、組織のガバナンスを自動的に適用できます。

---

## 4. アーキテクチャまたは設計のポイント

Azure Policy では **カスタムポリシー定義**を作成できます。

このポリシーでは次の条件を設定できます。

まず、リージョン制限です。

- 許可リージョンリスト
- 許可リージョン外のデプロイを拒否

次に、VMサイズ制限です。

- 許可された SKU
- 未承認サイズの VM 作成拒否

さらに、パラメータ化です。

- allowedLocations
- allowedSkus

このように **ポリシーをパラメータ化することで、将来の変更を簡単に行うことができます。**

---

## 5. 設計判断（なぜこの構成になるか）

この問題では **ガバナンスと柔軟性の両立**が重要です。

Azure Policy を使用すると次のメリットがあります。

まず、コンプライアンス強制です。

- 不正リソースの作成を拒否
- 組織ポリシーの自動適用

次に、運用効率です。

- 自動ポリシー適用
- 手動レビュー不要

さらに、柔軟性です。

- パラメータ変更でポリシー更新
- 再デプロイ不要

このため、セルフサービス環境でもガバナンスを維持できます。

---

## 6. 他の選択肢が誤りな理由

まず **ARMテンプレート**です。

ARMテンプレートではパラメータ検証はできますが、テンプレートを使わないデプロイを防ぐことはできません。

次に **Azure Governance + Management Groups** です。

これはガバナンスの概念ですが、実際の制御は Azure Policy によって行われます。

最後に **Azure Blueprints** です。

Blueprints は環境テンプレートのデプロイに適していますが、リソース構成の継続的な制御には Azure Policy がより適しています。

---

## 7. 最終回答

**A. Azure Policy（場所とSKU制限のカスタムポリシー + パラメータ）**

---

## 8. まとめ

Azure ガバナンスでは、複数のサービスを組み合わせて使用します。

主なサービスを整理すると次の通りです。

まず、ポリシー強制です。

- **Azure Policy**
  → リソース構成制御

次に、階層管理です。

- **Management Groups**
  → サブスクリプション管理

さらに、デプロイテンプレートです。

- **ARM Templates**

そして、環境テンプレートです。

- **Azure Blueprints**

そのため、**VM作成リージョンやサイズを制限しつつ、柔軟な更新を可能にするには Azure Policy のカスタムポリシーとパラメータを使用するのが最適**です。
