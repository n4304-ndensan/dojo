# Azure Policy によるリソースタグの強制とコンプライアンス管理

## 1. 背景（シナリオ）

ある組織では Azure 環境内で多数のリソースを運用しています。これらのリソースには仮想マシン、データベース、ストレージ、ネットワークなどが含まれており、運用管理を効率化するために **タグ（Tags）** を使用しています。

タグはリソースに付与するメタデータであり、コスト管理、所有者の識別、環境分類（Production / Development など）などに使用されます。組織では、すべての Azure リソースに必須のタグを設定することをポリシーとして定めています。

さらに、このタグ付けルールが守られているかを継続的に監視し、コンプライアンス状況をレポートできる仕組みが必要です。そのため、Azure 環境全体にタグ付けルールを強制できるガバナンス機能を導入する必要があります。

## 2. 要件整理

この問題のシナリオから、次の要件が明確になります。

まず、すべての Azure リソースに必須のタグを適用する必要があります。これはリソース管理とコスト追跡のために重要です。

次に、タグ付けがルールに従っているかを監視し、コンプライアンス状況を確認できる必要があります。つまり、タグが付いていないリソースを検出する仕組みが必要です。

さらに、これらのルールは Azure 環境全体に適用される必要があります。つまり、個別のリソースではなく、サブスクリプションやリソースグループ単位で適用できる仕組みが求められます。

これらをまとめると、次の要件になります。

- 必須タグの適用  
- タグ付けルールの強制  
- コンプライアンス監視  
- レポート機能  

## 3. 技術の基本概念

Azure にはクラウドガバナンスを実現するためのサービスとして **Azure Policy** が提供されています。

Azure Policy は、Azure 環境内のリソースが組織のルールに従っているかを評価し、必要に応じて強制するサービスです。これにより、企業のセキュリティポリシーや運用ポリシーをクラウド環境全体に適用できます。

Azure Policy では、次のようなポリシーを定義できます。

- 必須タグの強制  
- 特定リージョンのみ許可  
- 特定の VM サイズのみ許可  
- ストレージ暗号化の強制  

また、ポリシーは継続的に評価されるため、コンプライアンス違反のリソースを自動的に検出できます。

## 4. アーキテクチャまたは設計のポイント

Azure Policy を利用したタグ管理では、まず **カスタムポリシー定義**を作成します。このポリシーでは、リソースに必須のタグが存在するかをチェックするルールを定義します。

次に、そのポリシーをサブスクリプションまたはリソースグループに割り当てます。これにより、その範囲内のすべてのリソースに対してポリシーが適用されます。

この構成では次のような動作が行われます。

- 新しいリソースが作成されるとポリシーが評価される  
- 必須タグがない場合、リソース作成を拒否または警告する  
- 既存リソースも継続的に評価される  
- Azure ポータルでコンプライアンスレポートを確認できる  

このように、Azure Policy はタグ付けルールを強制するだけでなく、コンプライアンスの可視化も提供します。

## 5. 設計判断（なぜこの構成になるか）

Azure Policy は Azure 環境のガバナンスを管理するための専用サービスであり、タグ付けルールの強制に最適です。

カスタムポリシー定義を作成することで、必須タグを指定し、それらのタグが存在しないリソースの作成を拒否することができます。これにより、タグ付けルールを確実に適用できます。

また、Azure Policy は継続的にリソースを評価し、コンプライアンス違反のリソースを検出します。その結果は Azure ポータルのコンプライアンスレポートで確認できるため、監査や報告にも利用できます。

このように、Azure Policy はタグ付けの強制とコンプライアンス監視を同時に実現できるため、このシナリオに最も適しています。

## 6. 他の選択肢が誤りな理由

Azure Blueprints は、リソースの標準構成をテンプレートとして展開するためのサービスです。タグを含めることはできますが、既存リソースのコンプライアンス監視には適していません。

Azure Automation Runbook を使用するとタグ付けの自動化は可能ですが、これは定期的なスクリプト実行による修正であり、リアルタイムのポリシー強制ではありません。

ARM テンプレートはリソースデプロイ時にタグを設定できますが、既存リソースのコンプライアンス管理や監査機能は提供されません。

## 7. 最終回答

A. カスタムポリシー定義を採用し、必須のタグ付けと監査コンプライアンスを義務付ける Azure Policy

## 8. まとめ

Azure 環境でタグ管理を徹底するためには、単にタグを設定するだけでなく、組織ポリシーとして強制する必要があります。

Azure Policy を利用すると、必須タグのルールを定義し、Azure 環境全体に適用できます。また、ポリシーは継続的にリソースを評価し、コンプライアンスレポートを提供します。

そのため、Azure リソースに必須タグを適用し、コンプライアンスを監視・報告する最適な方法は **Azure Policy のカスタムポリシー定義を使用すること**です。
