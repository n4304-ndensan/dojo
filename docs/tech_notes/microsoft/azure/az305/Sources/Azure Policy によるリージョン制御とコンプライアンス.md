# Azure Policy によるリージョン制御とコンプライアンス

## 1. 背景（シナリオ）

企業がクラウド環境を利用する際には、データの所在地やコンプライアンスに関する規制を満たす必要があります。多くの業界では、データが保存されるリージョンを制限する必要があり、特定の地域以外にリソースが作成されないように管理する必要があります。

この問題では、Azure App Service と Azure SQL Database を使用したアプリケーションを構築する予定です。しかし、すべてのリソースは承認されたAzureリージョンにのみデプロイされる必要があり、さらに App Service とそのデータベースは同じリージョンに配置する必要があります。

このような要件は、単なる設計ではなく **クラウドガバナンス（Governance）** によって強制する必要があります。

## 2. 要件整理

問題文の要件を整理すると、次のようなコンプライアンスルールを適用する必要があります。

このシナリオでは、ユーザーの手動操作やテンプレート設定に依存するのではなく、組織レベルでポリシーを適用する必要があります。

具体的な要件は次の通りです。

・すべてのAzureリソースは承認されたリージョンのみで作成する  
・App Service と SQL Database は同じリージョンに配置する  
・規制要件を自動的に適用する必要がある  
・デプロイ時に違反を防止する必要がある  

つまり、リソース作成時にルールを自動的に適用できる仕組みが必要になります。

## 3. 技術の基本概念

Azureでは、クラウドガバナンスを実現するためのサービスとして **Azure Policy** が提供されています。

Azure Policyは、Azure環境のリソースに対してルールを定義し、それを強制するサービスです。これにより、組織の標準やコンプライアンス要件を自動的に適用できます。

Azure Policyでは次のようなことが可能です。

・特定リージョンのみリソース作成を許可する  
・特定のリソース構成を強制する  
・セキュリティ設定を標準化する  
・コンプライアンス状況を監査する  

この問題で重要なのは、Azure Policyの **「Allowed Locations（許可された場所）」** ポリシーです。

## 4. アーキテクチャまたは設計のポイント

Azure Policy の組み込みポリシーには **Allowed Locations（許可された場所）** という定義があります。このポリシーを使用すると、リソースを作成できるAzureリージョンを制限できます。

このポリシーを適用すると、次のような制御が可能になります。

・承認されたリージョンのみリソース作成を許可  
・承認されていないリージョンへのデプロイを拒否  
・サブスクリプションまたは管理グループ単位で適用  
・リアルタイムでポリシー違反を検出  

さらに、必要に応じてカスタムポリシーを作成することで、App Service と SQL Database が同じリージョンに配置されることも確認できます。

## 5. 設計判断（なぜこの構成になるか）

この問題の要件は、リソース配置に関するコンプライアンスを **強制（Enforce）すること**です。

そのためには次の機能が必要になります。

・リソース作成時にルールを適用できる  
・リージョン制限を設定できる  
・組織全体でポリシーを適用できる  
・違反するリソース作成を拒否できる  

Azure Policyはこれらの機能を提供するAzureの標準的なガバナンスサービスです。

特に「Allowed Locations」ポリシーを使用することで、承認されたリージョン以外でのリソース作成を防ぐことができます。

## 6. 他の選択肢が誤りな理由

まず ARMテンプレートについて説明します。ARMテンプレートではリージョンパラメータを指定できますが、ユーザーが別のリージョンでリソースを作成することを防ぐことはできません。つまり、組織レベルのガバナンスを強制できません。

次に Azure Blueprints です。Blueprints はポリシーやリソース構成をまとめてデプロイする仕組みですが、リージョン制御のような単純なガバナンス要件には Azure Policy の方が適しています。

最後に Microsoft Defender for Cloud です。Defender for Cloud はセキュリティ状態の監視やコンプライアンス評価を行うサービスですが、リソース作成を制御する機能はありません。

## 7. 最終回答

A. 組み込みの「許可された場所（Allowed Locations）」定義を使用した Azure Policy

## 8. まとめ

Azure環境でコンプライアンスやガバナンスを適用する場合、Azure Policyを使用してリソース構成ルールを強制できます。Azure Policyの「Allowed Locations」ポリシーを使用することで、リソースを特定のAzureリージョンに限定することができます。

この仕組みにより、組織の規制要件やデータ所在地ポリシーを自動的に適用することができます。そのため、Azureリソースを承認されたリージョンのみに制限する場合には **Azure Policy（Allowed Locations）** を使用するのが最適なソリューションです。
