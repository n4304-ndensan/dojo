# Azure Policy によるリージョン制約とリソース配置のコンプライアンス管理

## 1. 背景（シナリオ）

ある組織では、多層アーキテクチャのクラウドアプリケーションをAzure上にデプロイする準備を進めています。このアプリケーションは次の3つの主要サービスで構成されています。

- Azure App Service  
- Azure Cosmos DB  
- Azure Functions  

この環境では厳しい規制要件が存在し、すべてのAzureリソースは **指定されたAzureリージョン内にのみデプロイされる必要**があります。

さらに、データの居住要件（Data Residency）を満たすために、同じアプリケーション層に属するすべてのリソースが **同じAzureリージョンに配置されること**が重要です。

例えば、App Serviceのインスタンスがあるリージョンに配置されている場合、同じ層の関連リソースも同じリージョンに配置される必要があります。

## 2. 要件整理

この問題では、Azure環境においてリソースの配置場所をガバナンスレベルで制御する必要があります。

シナリオから読み取れる要件は以下の通りです。

- すべてのリソースは指定されたリージョンにデプロイする
- データ居住要件を満たす必要がある
- 同じアプリケーション層のリソースは同じリージョンに配置する
- コンプライアンスを継続的に強制する必要がある
- 手動管理ではなく自動的に制御する必要がある

これらの要件はAzureの **ガバナンス機能** を利用することで実現できます。

## 3. 技術の基本概念

Azureでは、リソースのコンプライアンスや構成を強制するために **Azure Policy** というサービスが提供されています。

Azure Policyは、リソースの作成や変更を監視し、組織のルールに従っていないリソースを制御する仕組みです。

Azure Policyでは、以下のような制御が可能です。

まず、リソースのリージョン制限です。ポリシーを設定することで、指定されたリージョン以外でリソースを作成できないようにすることができます。

次に、リソース構成の強制です。例えば、特定の設定を必須にしたり、タグを強制することができます。

さらに、ポリシーは新規リソースだけでなく既存リソースにも適用され、継続的に評価されます。

## 4. アーキテクチャまたは設計のポイント

このシナリオでは、リージョン制約とリソースの共配置（コロケーション）を確実に強制する必要があります。

Azure Policyを使用すると、以下のようなポリシーを作成できます。

まず、リソースが特定リージョンにのみデプロイされるように制限するポリシーを作成します。これにより、指定されたリージョン以外でのリソース作成が拒否されます。

次に、同じアプリケーション層のリソースが同じリージョンに配置されているかを検証するポリシーを定義します。

Azure Policyはリソース作成時に自動評価を行うため、コンプライアンス違反のリソースを防ぐことができます。

この仕組みにより、規制要件を満たしながら安全なクラウド環境を維持できます。

## 5. 設計判断（なぜこの構成になるか）

この問題の重要なポイントは **コンプライアンスの強制（Enforcement）** です。

ARMテンプレートでは、リソースのデプロイ時にリージョンを指定することはできます。しかし、テンプレート以外の方法でリソースが作成された場合、その制御はできません。

Azure Policyは、Azure環境全体でリソースを評価し、ポリシー違反のリソース作成を防止できます。

また、ポリシーは継続的に評価されるため、コンプライアンスを長期的に維持できます。

そのため、リージョン制約やリソース配置の要件を満たす最適な方法は **Azure Policyを使用すること**です。

## 6. 他の選択肢が誤りな理由

### A. ARMテンプレート

ARMテンプレートはインフラをコードとして定義するためのツールです。

確かにテンプレート内でリージョンを指定できますが、テンプレート外で作成されたリソースを制御することはできません。

そのため、コンプライアンスの強制には不十分です。

### C. Azure Blueprints

Azure Blueprintsは、ガバナンス構成をテンプレート化するためのサービスです。

Blueprintはポリシーやリソース構成をまとめて定義できますが、実際の制御はAzure Policyによって実行されます。

この問題の直接的な解決策はAzure Policyです。

### D. Azure Management Groups

管理グループはサブスクリプションを階層的に管理するための仕組みです。

ポリシーを適用するスコープとして利用できますが、リージョン制約を直接強制する機能はありません。

## 7. 最終回答

地域展開およびコロケーション要件を確実に遵守するための最適な方法は次です。

**Azure Policy を使用してリージョン制約を適用する**

したがって **正解は B** です。

## 8. まとめ

Azure環境でコンプライアンス要件を満たすためには、ガバナンスサービスの利用が重要です。

Azure Policyはリソース構成を評価し、組織のルールを強制する仕組みを提供します。

Azure Policyを利用することで、次のような制御が可能になります。

- リソース作成リージョンの制限
- セキュリティ設定の強制
- リソース構成の標準化
- コンプライアンス監視

そのため、規制要件やデータ居住要件を満たすAzure環境では **Azure Policyによるガバナンスが最も重要な仕組み**になります。
