# Azure Policy のタグ自動付与（Append エフェクト）

## 1. 背景（シナリオ）

Azure 環境では、リソース管理やコスト管理を効率化するために **タグ（Tag）** を利用することが一般的です。タグはキーと値のペアで構成され、次のような運用情報をリソースに付与できます。

- 部門
- 環境（Production / Dev / Test）
- 所有者
- コストセンター

企業のガバナンスでは、すべてのリソースに対してこれらのタグが適用されていることが求められる場合が多くあります。

今回のシナリオでは、Azure サブスクリプション全体のリソースに対して **企業ポリシーに従ったタグ付けを自動化**する必要があります。

さらに、次の要件があります。

- 新しいリソースにタグを自動追加する
- リソースグループのタグと一致させる
- 手動運用ではなくポリシーで強制する

---

## 2. 要件整理

この問題では、Azure Policy を使用してタグ管理を実装します。

シナリオから読み取れる重要な要件を整理します。

まず、ガバナンスに関する要件があります。

- すべての新規リソースにタグを適用
- 組織ポリシーへの準拠

次に、タグ管理の要件です。

- リソースグループのタグを継承
- タグを自動的に追加
- リソース作成を阻害しない

このことから、**リソース作成時にタグを自動追加できる Azure Policy 効果**を選択する必要があります。

---

## 3. 技術の基本概念

Azure Policy では、ポリシー違反に対する動作を **Effect（エフェクト）** として定義します。

主要なエフェクトには次のものがあります。

まず、ポリシー違反のリソースを制御するエフェクトです。

- **Deny**  
  ポリシー違反の場合、リソース作成を拒否する

次に、リソース構成を変更するエフェクトです。

- **Append**  
  リソース作成時に追加プロパティを自動付与
- **Modify**  
  既存のリソース設定を変更

最後に、追加リソースをデプロイするエフェクトです。

- **DeployIfNotExists**  
  条件を満たさない場合に追加リソースをデプロイ

この問題では、**タグを自動追加すること**が目的です。

---

## 4. アーキテクチャまたは設計のポイント

Azure 環境でタグ管理を実装する場合、Azure Policy を使用してタグ付与を自動化するのがベストプラクティスです。

特に、リソースグループのタグをリソースに継承させる場合には、Append エフェクトを使用します。

この設計のポイントは次の通りです。

- リソースグループにタグを定義
- Azure Policy をサブスクリプションまたは管理グループに適用
- Append エフェクトでタグを追加

この仕組みにより、新しいリソースが作成されると次の処理が行われます。

1. リソース作成リクエストを Azure Policy が評価  
2. 必要なタグが存在するか確認  
3. 不足しているタグを自動追加  

これにより、タグ管理を自動化できます。

---

## 5. 設計判断（なぜこの構成になるか）

この問題の要件は **新規リソースにタグを自動追加すること**です。

この要件に最も適した Azure Policy エフェクトが **Append** です。

Append の特徴を整理すると次の通りです。

- リソース作成をブロックしない
- 不足しているプロパティを自動追加
- タグ管理の自動化に最適

つまり、ユーザーがタグを設定し忘れても、Azure Policy がタグを追加します。

これにより

- 運用ミスの防止
- ガバナンス強化
- 管理負荷の削減

を実現できます。

---

## 6. 他の選択肢が誤りな理由

まず **Deny** について説明します。

Deny はポリシー違反のリソース作成を拒否します。しかし、この問題では **タグを自動追加する必要**があります。

Deny を使用すると、タグがない場合にリソース作成が失敗するため、運用効率が低下する可能性があります。

次に **Modify** です。

Modify は既存のプロパティを変更する場合に使用されることが多く、新しいタグを追加する用途としては Append が一般的です。

最後に **DeployIfNotExists** です。

DeployIfNotExists は、必要なリソースが存在しない場合に追加リソースをデプロイするためのエフェクトです。

タグ付与の用途には適していません。

---

## 7. 最終回答

**A. Append（付加）**

---

## 8. まとめ

Azure Policy は Azure ガバナンスを実現する重要なサービスです。

特に覚えておくべき Policy Effect は次の通りです。

まず、コンプライアンスを強制するものです。

- **Deny**  
  ポリシー違反のリソース作成を拒否

次に、自動設定を行うものです。

- **Append**  
  プロパティやタグを自動追加
- **Modify**  
  リソース設定を変更

最後に追加リソースをデプロイするものです。

- **DeployIfNotExists**

このため、**リソースグループのタグを新しいリソースに自動付与する場合は Append エフェクトを使用する**のが適切です。
