# Azure SQL Database Elastic Pool によるクロスデータベーストランザクション設計

## 1. 背景（シナリオ）

ある企業では、オンプレミス環境で運用している **Microsoft SQL Server データベース DB1 と DB2** を Azure に移行する計画があります。移行の目的は、クラウド環境のスケーラビリティと高可用性を活用しながら、運用管理の負担を軽減することです。

このシステムでは、2つのデータベース間で **サーバー側トランザクション**を実行する必要があります。つまり、DB1 と DB2 の両方にまたがるトランザクション処理が必要です。

さらに、このソリューションは将来の成長を見据えて設計される必要があり、以下の要件を満たす必要があります。

・管理オーバーヘッドの最小化  
・データベース間トランザクションのサポート  
・高可用性  
・スケーラビリティ  

## 2. 要件整理

この問題では、単にデータベースを Azure に移行するだけではなく、運用負荷を減らしながらトランザクション処理を維持する必要があります。

シナリオから読み取れる要件を整理すると次の通りです。

まず、DB1 と DB2 の両方にまたがるトランザクション処理をサポートする必要があります。次に、運用チームがインフラ管理に多くの時間を費やさないように、管理オーバーヘッドを最小限に抑える必要があります。

さらに、クラウド環境としての利点である高可用性とスケーラビリティも重要です。

これらをまとめると、次の要件になります。

・クロスデータベーストランザクション  
・マネージドサービスによる管理負担の削減  
・高可用性  
・スケーラビリティ  

この要件に最も適したソリューションが **Azure SQL Database Elastic Pool** です。

## 3. 技術の基本概念

Azure SQL Database は、Microsoft が提供する **完全マネージド型のリレーショナルデータベースサービス**です。

Elastic Pool は Azure SQL Database の機能の一つで、複数のデータベースが共有リソースを利用できるようにする仕組みです。

このモデルでは、データベースごとに専用のリソースを割り当てるのではなく、複数のデータベースが同じプール内で CPU やメモリなどのリソースを共有します。

これにより、リソース利用率を最適化しながらコスト効率の高い運用が可能になります。

また、Azure SQL Database は高可用性を標準機能として提供しており、自動バックアップやフェイルオーバーなどが Azure によって管理されます。

## 4. アーキテクチャまたは設計のポイント

Elastic Pool を利用したアーキテクチャでは、DB1 と DB2 を同一の Elastic Pool 内に配置します。

この構成では、両方のデータベースが同じ論理サーバー上に存在するため、クロスデータベーストランザクションを実行することが可能になります。

また、Elastic Pool によってリソースが共有されるため、トラフィックの変動にも柔軟に対応できます。

この構成の主なポイントは次の通りです。

まず、複数のデータベースが同じリソースプールを共有します。次に、Azure がバックアップや可用性管理を自動化します。さらに、必要に応じてプールのリソースをスケールアップすることで、将来的な負荷増加にも対応できます。

## 5. 設計判断（なぜこの構成になるか）

Elastic Pool が最適な理由は、管理負担を最小限に抑えながらスケーラブルなデータベース環境を提供できる点です。

Azure SQL Database は PaaS サービスであるため、パッチ管理、バックアップ管理、インフラ管理などを Azure が自動的に処理します。

そのため、運用チームはデータベース管理ではなくアプリケーション開発やビジネスロジックに集中できます。

また、Elastic Pool を利用することで複数データベースのリソース利用を効率化でき、将来的な負荷増加にも柔軟に対応できます。

## 6. 他の選択肢が誤りな理由

### A SQL Managed Instance を別々に配置して DTC を使用

この方法では分散トランザクションコーディネーターを使用する必要があり、構成が複雑になります。また、複数の Managed Instance を管理する必要があり、運用負荷が増加します。

### C 同じサーバー上の Azure SQL Database

この構成ではクロスデータベーストランザクションは可能ですが、Elastic Pool のようなリソース共有や効率的なスケーリングのメリットがありません。

### D Azure VM 上の SQL Server

SQL Server を仮想マシン上で運用する場合、OS 管理、パッチ管理、バックアップ管理などをすべて自分で行う必要があります。そのため管理オーバーヘッドが大きくなります。

## 7. 最終回答

B. **クロスデータベーストランザクションが有効な Elastic Pool 内の Azure SQL Database**

## 8. まとめ

Azure SQL Database Elastic Pool は、複数のデータベースを効率的に管理するための PaaS ソリューションです。

この構成では、DB1 と DB2 を同一プールに配置することでクロスデータベーストランザクションを実行できます。

さらに Azure がインフラ管理を自動化するため、管理オーバーヘッドを最小限に抑えることができます。

試験では **「オンプレ SQL 移行」「管理オーバーヘッド削減」「クロスデータベーストランザクション」「スケーラブル」** といったキーワードが出てきた場合、Azure SQL Database（Elastic Pool）が正解になるケースが多いです。
