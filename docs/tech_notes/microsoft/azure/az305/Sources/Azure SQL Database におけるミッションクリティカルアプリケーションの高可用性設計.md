# Azure SQL Database におけるミッションクリティカルアプリケーションの高可用性設計

## 1. 背景（シナリオ）

ある組織では、ミッションクリティカルなアプリケーションのために Azure SQL Database を使用した高可用性データベース基盤を設計する必要があります。このアプリケーションはビジネスの中核を担っており、システム停止やデータ損失が発生すると大きな影響が出る可能性があります。

そのため、データベースの設計には非常に厳しい可用性要件が設定されています。特に、フェイルオーバーが発生した場合でもデータ損失をゼロに抑えること、さらにリージョン障害が発生した場合でもサービスを継続できることが求められています。

また、このシステムは厳格なサービスレベル契約（SLA）を満たす必要があり、同時にコスト効率も考慮した設計でなければなりません。

## 2. 要件整理

このシナリオの要件を整理すると、次のような重要なポイントが明確になります。

- ミッションクリティカルなアプリケーション  
- フェイルオーバー時のデータ損失ゼロ  
- リージョン障害に対する耐障害性  
- 高可用性（High Availability）  
- 厳格なSLAの遵守  
- コスト効率  

これらの要件を満たすためには、単一リージョンの高可用性だけでなく、リージョン間の災害復旧（Disaster Recovery）も考慮する必要があります。

## 3. 技術の基本概念

Azure SQL Database では、高可用性と災害復旧を実現するために複数の機能が提供されています。これらの機能を組み合わせることで、ミッションクリティカルなワークロードを支えることができます。

まず、Azure SQL Database には複数のサービスレベルがあります。特に重要なのが Business Critical レベルです。

Business Critical レベルは、ミッションクリティカルなワークロード向けに設計されており、次のような特徴があります。

- 高性能ストレージ  
- インメモリ処理  
- 高速フェイルオーバー  
- 高い可用性（99.99% SLA）

さらに、可用性ゾーンを利用したゾーン冗長構成を設定することで、同一リージョン内の複数のデータセンターにデータを分散できます。

また、ジオレプリケーションを利用することで、別リージョンに読み取り可能なセカンダリデータベースを作成し、リージョン障害時の災害復旧を実現できます。

## 4. アーキテクチャまたは設計のポイント

ミッションクリティカルなデータベース設計では、単一障害点（Single Point of Failure）を排除することが重要です。そのためには、複数レベルの冗長化を実装する必要があります。

まず、リージョン内の可用性を高めるために、可用性ゾーンを利用したゾーン冗長構成を採用します。これにより、データベースは同一リージョン内の複数のゾーンに複製され、ゾーン障害が発生してもサービスが継続できます。

次に、リージョン障害に備えるために、ジオレプリケーションを使用して別リージョンにデータベースを複製します。これにより、大規模障害が発生した場合でもセカンダリリージョンへフェイルオーバーすることでサービスを継続できます。

このように、ゾーン冗長とリージョン冗長を組み合わせることで、多層的な可用性アーキテクチャを構築できます。

## 5. 設計判断（なぜこの構成になるか）

この問題では「ミッションクリティカル」「データ損失ゼロ」「リージョン障害対応」という3つの要件が重要です。

Business Critical サービスレベルは、Azure SQL Database の中で最も高い可用性とパフォーマンスを提供する層です。そのため、ミッションクリティカルなアプリケーションに最適です。

さらに、ゾーン冗長構成を使用することで、同一リージョン内のデータセンター障害に対応できます。そして、ジオレプリケーションを組み合わせることで、リージョンレベルの障害にも対応できます。

このように、Business Critical とゾーン冗長構成、そしてジオレプリケーションを組み合わせることで、要求される高可用性と災害復旧の両方を満たすことができます。

## 6. 他の選択肢が誤りな理由

この問題では他のAzure SQL Databaseの構成も提示されていますが、それぞれ要件を完全には満たしていません。

B. Azure SQL Database Hyperscale と Active Geo-Replication  

Hyperscaleは非常に大規模なデータベース向けのスケーラビリティを提供しますが、ミッションクリティカルな可用性要件においてBusiness Criticalほど最適ではありません。

C. Azure SQL Database Premium と Auto-failover Groups  

Premiumレベルは高性能ですが、Business Criticalほどの高可用性設計が提供されるわけではありません。また、バックアップ冗長だけでは即時フェイルオーバーには対応できません。

D. Azure SQL Database Standard  

Standardレベルはコストは低いものの、ミッションクリティカルな高可用性や災害復旧要件を満たす機能が不足しています。

## 7. 最終回答

A. Azure SQL Database Business Critical with Zone Redundant Configuration および Geo-Replication

## 8. まとめ

ミッションクリティカルなAzure SQL Database環境を設計する場合には、リージョン内の高可用性とリージョン間の災害復旧の両方を考慮する必要があります。

Azure SQL Database Business Critical レベルは高可用性と高性能を提供するサービスレベルであり、ゾーン冗長構成を利用することでゾーン障害に対応できます。さらに、ジオレプリケーションを組み合わせることでリージョン障害にも対応可能になります。

そのため、Business Critical、ゾーン冗長構成、ジオレプリケーションの組み合わせが、ミッションクリティカルなアプリケーションの要件を満たす最適な構成となります。
