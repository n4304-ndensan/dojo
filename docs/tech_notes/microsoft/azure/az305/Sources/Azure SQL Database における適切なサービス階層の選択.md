# Azure SQL Database における適切なサービス階層の選択

## 1. 背景（シナリオ）
ある企業がアプリケーションを Azure に移行しています。このアプリケーションは主に月次レポートに利用され、データベースは毎月1日のみアクセスされます。データは年間約5％の割合で増加します。シナリオでは、低コストでパフォーマンスが十分なデータベースサービス階層を選ぶ必要があります。

## 2. 要件整理
このシナリオから読み取れる要件は次の通りです。

- アプリケーションは主にレポート作成に使用されるため、トランザクション負荷は低い。
- データベースの利用は月に1回のみ。
- 年間データ成長率は5％と低め。
- コスト効率が重要。
- 将来的なスケーリングも可能であることが望ましい。

## 3. 技術の基本概念
Azure SQL Database には、DTUベースと vCoreベースの2種類のモデルがあります。

- **DTUベースモデル**: コンピューティング、メモリ、I/O を組み合わせたパッケージ型で、管理が簡単。
  - 標準層：軽量〜中程度のワークロード向け、コスト効率が高い。
  - プレミアム層：高トランザクション向け、より高価。
- **vCoreベースモデル**: コンピューティング、メモリ、ストレージを個別に管理可能で、柔軟性が高い。
  - 汎用：バランスの取れた性能、コスト効率重視。
  - ビジネスクリティカル：低遅延、高性能、大規模トランザクション向け。

## 4. アーキテクチャまたは設計のポイント
このシナリオでは、月次レポート専用で利用頻度が低く、データ増加率も低いため、過剰な性能や高コストの階層は不要です。DTUベースの標準階層は、軽量〜中程度の負荷向けであり、コスト効率が高く、十分なパフォーマンスを提供します。

## 5. 設計判断（なぜこの構成になるか）
- データベースは月1回のみ利用されるため、常時高性能は不要。
- 年間5％程度の成長であれば標準層で十分。
- DTUベースモデルは管理が簡単で、月次レポート用途に適する。
- 必要に応じて将来的にスケールアップ可能で、コスト管理が容易。

## 6. 他の選択肢が誤りな理由
- **A) vCoreベースのビジネスクリティカル**: 高性能かつ高価で、月1回の利用には過剰。
- **B) DTUベースのプレミアム**: 高トランザクション向けでコストが高く、低頻度利用には不向き。
- **C) vCoreベースの汎用**: コスト効率は良いが、管理や設定が少し複雑で、DTU標準で十分な性能。

## 7. 最終回答
**DTUベースの標準階層**

## 8. まとめ
月1回だけ使用される軽量ワークロードでは、DTUベースの標準階層が最適です。コスト効率が高く、パフォーマンスは十分で、将来必要に応じてスケーリングも可能です。この構成により、低コストで安定した月次レポート作成環境を構築できます。
