# Azure SQL Database（アクティブ ジオレプリケーション）によるSQL Server移行設計
## 1. 背景（シナリオ）

ある企業では、オンプレミス環境で稼働している複数のSQL ServerデータベースをAzureへ移行する計画を進めています。これらのデータベースを利用するアプリケーションは、リアルタイム性の高い処理を必要としており、低遅延でデータにアクセスできることが重要です。

さらに、このアプリケーションでは複数のテーブルを結合する複雑なクエリが頻繁に実行されます。そのため、リレーショナルデータベースとして高度なSQL機能をサポートするデータベースサービスが必要になります。

また、システムの可用性を高めるため、データは別のAzureリージョンにも複製される必要があります。これは災害復旧（Disaster Recovery）対策として重要な要件です。

加えて、クラウド移行ではコスト最適化も重要な要素であり、必要な機能を満たしつつ運用コストを抑えることが求められています。

---

## 2. 要件整理

この問題では、Azureに移行するデータベースのストレージおよびデータベースサービスを選択する必要があります。

シナリオから読み取れる要件を整理すると次のようになります。

まず、アプリケーションは低遅延でデータにアクセスする必要があります。そのため、高性能なデータベースサービスが必要です。

次に、複数テーブルを含む複雑なクエリを実行する必要があります。これはリレーショナルデータベース機能が必要であることを意味します。

さらに、データを別リージョンへ複製する必要があります。これは災害復旧のためのジオレプリケーション機能が必要になります。

最後に、コスト最適化が求められています。つまり、必要な機能を満たしながら無駄なインフラ管理を減らす必要があります。

要件をまとめると次の通りです。

- SQL Server互換のリレーショナルデータベース  
- 複雑なSQLクエリのサポート  
- 低遅延アクセス  
- 別リージョンへのデータレプリケーション  
- コスト効率  

---

## 3. 技術の基本概念

Azure SQL Databaseは、Microsoft SQL ServerエンジンをベースにしたPaaS型のリレーショナルデータベースサービスです。

このサービスは、オンプレミスSQL Serverからクラウドへ移行する際に最も一般的に利用されるサービスの一つです。

Azure SQL Databaseは次のような特徴を持っています。

- SQL Server互換エンジン  
- マネージドデータベースサービス  
- 自動バックアップ  
- 高可用性機能  
- 自動スケーリング  

また、災害復旧機能として「Active Geo-Replication（アクティブジオレプリケーション）」を提供しています。

この機能により、別のAzureリージョンに読み取り可能なセカンダリデータベースを作成できます。

---

## 4. アーキテクチャまたは設計のポイント

Azure SQL DatabaseのActive Geo-Replicationを利用すると、データベースは複数リージョンに複製されます。

この構成では次のような仕組みになります。

1. プライマリデータベースがメインリージョンで稼働  
2. データがセカンダリリージョンへレプリケーション  
3. セカンダリは読み取り専用として利用可能  
4. 障害時はセカンダリへフェイルオーバー  

この構成により、次のメリットがあります。

- 災害復旧対応  
- 高可用性の実現  
- 読み取りスケーリング  
- グローバルアプリケーション対応  

さらに、Azure SQL DatabaseはPaaSサービスであるため、インフラ管理の負担も大幅に削減されます。

---

## 5. 設計判断（なぜこの構成になるか）

この問題の最大のポイントは「複雑なクエリ」と「SQL Server移行」です。

Azure SQL DatabaseはSQL Server互換のリレーショナルデータベースであり、複雑なSQLクエリをネイティブにサポートしています。

また、Active Geo-Replicationにより次の機能を実現できます。

- 別リージョンへのデータ複製  
- 災害復旧対応  
- 高可用性  

さらに、PaaSサービスであるため、運用管理コストも削減できます。

そのため、このシナリオではAzure SQL Databaseが最適な選択になります。

---

## 6. 他の選択肢が誤りな理由

Azure Cosmos DBはNoSQLデータベースであり、複雑なリレーショナルクエリには適していません。SQL APIは存在しますが、SQL Serverのリレーショナル機能とは異なります。

Azure Table Storageはキー・バリュー型NoSQLストレージであり、複雑なテーブル結合や高度なSQLクエリをサポートしていません。

Azure Managed Diskは仮想マシン用のストレージであり、データベースサービスではありません。そのため、クエリ処理やデータ管理機能は提供されません。

---

## 7. 最終回答

A.  
アクティブジオレプリケーションを有効にした **Azure SQL Database**

---

## 8. まとめ

オンプレミスSQL ServerをAzureへ移行する場合、Azure SQL Databaseは最も自然な選択肢になります。

特にActive Geo-Replicationを利用すると次のメリットがあります。

- 別リージョンへのデータレプリケーション  
- 災害復旧機能  
- 高可用性  
- 読み取りスケール  

さらに、SQL Server互換のリレーショナルデータベースであるため、複雑なクエリにも対応できます。

そのため、このシナリオでは **Azure SQL Database（Active Geo-Replication）** が最適なソリューションです。
