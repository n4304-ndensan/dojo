# Azure SQL Managed Instance によるクロスデータベーストランザクション対応の移行

## 1. 背景（シナリオ）

企業では、オンプレミス環境で運用している **Microsoft SQL Server のデータベース DB1 と DB2** を Azure に移行する計画があります。クラウド移行の目的は、運用管理の負担を減らしながら、Azure の高可用性とスケーラビリティを活用することです。

このシステムでは、DB1 と DB2 の両方をまたぐ **サーバー側トランザクション**を実行する必要があります。つまり、2つのデータベースに対して同一トランザクションで処理を行う必要があります。

そのため、移行先の Azure ソリューションには次の条件が求められます。

オンプレ SQL Server と高い互換性  
クロスデータベーストランザクションのサポート  
管理オーバーヘッドの最小化  
高可用性とスケーラビリティ  

## 2. 要件整理

この問題では、単にデータベースをクラウドに移行するだけではなく、既存アプリケーションのトランザクション処理を維持しながら運用負荷を減らす必要があります。

シナリオから読み取れる重要な要件を整理すると次の通りです。

まず、DB1 と DB2 の両方にまたがるトランザクション処理をサポートする必要があります。これは SQL Server のサーバー側トランザクションや分散トランザクションに該当します。

次に、クラウド移行後の運用管理を簡素化する必要があります。そのため、インフラ管理が必要な IaaS よりも、PaaS サービスの方が適しています。

さらに、将来的なシステム拡張に対応するため、高可用性とスケーラビリティも必要になります。

これらをまとめると次の要件になります。

クロスデータベーストランザクション  
オンプレ SQL Server との互換性  
管理オーバーヘッドの最小化  
高可用性とスケーラビリティ  

## 3. 技術の基本概念

Azure SQL Managed Instance は、SQL Server と非常に高い互換性を持つ **PaaS 型データベースサービス**です。

このサービスは、オンプレミス SQL Server とほぼ同じ機能を提供するため、既存アプリケーションの変更を最小限に抑えて移行することができます。

Managed Instance は SQL Server インスタンスの概念を持つため、同じインスタンス内に複数のデータベースを配置できます。

この構成では、複数のデータベース間で **サーバー側トランザクション**を実行することが可能です。

さらに、Managed Instance は **Microsoft Distributed Transaction Coordinator（MSDTC）** をサポートしており、分散トランザクションにも対応できます。

## 4. アーキテクチャまたは設計のポイント

このシナリオでは、DB1 と DB2 を **同じ Azure SQL Managed Instance 上に配置する**ことが最も適切な設計になります。

この構成では、両方のデータベースが同一 SQL インスタンス上に存在するため、アプリケーションは従来の SQL Server と同様にトランザクションを実行できます。

この設計の主なポイントは次の通りです。

同一インスタンス上の複数データベース  
サーバー側トランザクションのサポート  
MSDTC による分散トランザクション  
Azure による自動バックアップとパッチ管理  

この構成により、オンプレミス SQL Server とほぼ同じ動作を Azure 上で実現できます。

## 5. 設計判断（なぜこの構成になるか）

Azure SQL Managed Instance が最適な理由は、**SQL Server 互換性と PaaS の利点を両立できること**です。

Managed Instance は PaaS サービスであるため、以下の管理作業が Azure によって自動化されます。

バックアップ管理  
パッチ適用  
高可用性管理  
インフラ管理  

そのため、運用チームはインフラ管理ではなくアプリケーション運用に集中できます。

また、同一インスタンス上の複数データベースはネイティブにクロスデータベーストランザクションをサポートしているため、既存システムとの互換性も高くなります。

## 6. 他の選択肢が誤りな理由

### B 弾性プール内の Azure SQL Database

Elastic Pool はリソース共有によるコスト最適化には優れていますが、SQL Managed Instance のようなフル SQL Server 互換性はありません。

また、クロスデータベーストランザクションのサポートも制限があります。

### C リンクサーバーを使用した Azure SQL Database

リンクサーバーを使用すると複雑な構成が必要になり、パフォーマンスや管理の観点でも最適ではありません。

### D Azure VM 上の SQL Server

Azure VM 上の SQL Server は IaaS であり、OS 管理やパッチ適用、バックアップなどを自分で管理する必要があります。

そのため、管理オーバーヘッドが増加します。

## 7. 最終回答

A. **同じ Azure SQL Managed Instance 上の 2 つのデータベース**

## 8. まとめ

Azure SQL Managed Instance は、オンプレミス SQL Server と高い互換性を持つ PaaS データベースサービスです。

同一インスタンス内に DB1 と DB2 を配置することで、クロスデータベーストランザクションをサポートできます。

さらに、Azure がインフラ管理を自動化するため、管理オーバーヘッドを大幅に削減できます。

試験では **「オンプレ SQL 移行」「クロスデータベーストランザクション」「SQL Server 互換性」「管理オーバーヘッド削減」** といったキーワードが出てきた場合、Azure SQL Managed Instance が正解になるケースが多いです。
