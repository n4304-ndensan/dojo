# Azure Storage アカウントのネットワーク制御（Private Endpoint と Storage Firewall）

## 1. 背景（シナリオ）

ある企業では、アプリケーションが Azure Storage アカウントを使用してデータを保存しています。このストレージには機密性の高いデータが含まれているため、新しいセキュリティ要件が追加されました。

セキュリティチームは、ストレージアカウントへのアクセスを厳格に制御することを求めています。特に、ストレージアカウントに対するアクセスは特定のネットワークからのみ許可される必要があります。

[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Azure Virtual Network (VNet)]]
具体的には、アプリケーションが動作する **特定の Azure 仮想ネットワーク（VNet）**からのみストレージへアクセスできるようにする必要があります。また、管理者がストレージアカウントを管理する際には、指定された **特定のパブリック IP 範囲**からのみアクセス可能である必要があります。

さらに重要な要件として、ストレージアカウントへの **パブリックアクセスは完全に無効化**する必要があります。つまり、インターネット上の任意の場所からストレージへ接続できないようにする必要があります。

このようなセキュリティ要件を満たすためには、Azure Storage のネットワークアクセス制御機能を適切に組み合わせる必要があります。

---

## 2. 要件整理

この問題では、ストレージアカウントのネットワークセキュリティに関する複数の要件が提示されています。設計を正しく理解するためには、これらの要件を整理することが重要です。

まず、アプリケーションのアクセス経路に関する要件があります。ストレージアカウントは特定の Azure 仮想ネットワークからのみアクセス可能でなければなりません。

次に、管理アクセスに関する要件があります。ストレージアカウントの管理操作は、特定のパブリック IP 範囲からのみ許可される必要があります。

最後に、最も重要なセキュリティ要件があります。それは、ストレージアカウントへのパブリックネットワークアクセスを完全に無効にすることです。

これらの条件を整理すると、次のようになります。

- 特定の Azure VNet からのみアクセス可能  
- 特定のパブリック IP 範囲から管理アクセス可能  
- パブリックアクセスを完全に無効化  

このような条件を満たすためには、単一の機能ではなく **複数のネットワークセキュリティ機能を組み合わせる必要があります**。

---

## 3. 技術の基本概念

### Private Endpoint（プライベートエンドポイント）

[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Azure Private Link]]
Private Endpoint は Azure Private Link の機能の一部であり、PaaS サービスに対して **仮想ネットワーク内のプライベート IP を使用して接続する仕組み**です。

通常、Azure Storage はパブリックエンドポイントを通じてアクセスされます。しかし Private Endpoint を使用すると、ストレージアカウントへの接続は VNet 内のプライベート IP を経由するようになります。

この仕組みにより、トラフィックはインターネットを経由せず Azure の内部ネットワーク内で完結します。その結果、より高いセキュリティを実現することができます。

Private Endpoint の重要な特徴を整理すると次の通りです。

- VNet 内のプライベート IP で PaaS に接続  
- インターネットを経由しない通信  
- 特定の VNet からのみアクセス可能  

この機能により、ストレージアカウントへのアクセスを特定の仮想ネットワークに制限できます。

---

### Azure Storage Firewall

Azure Storage Firewall は、ストレージアカウントへのアクセスをネットワークレベルで制御する機能です。ファイアウォールルールを使用することで、特定のネットワークや IP アドレスからのみアクセスを許可できます。

この機能は、ストレージアカウントのセキュリティを強化するために非常に重要です。特に、パブリックネットワークアクセスを制御する場合に使用されます。

Storage Firewall では、次のようなアクセス制御を設定できます。

まず、特定の仮想ネットワークやサブネットからのアクセスを許可する設定があります。これにより、信頼できるネットワークからのみアクセスが可能になります。

次に、特定のパブリック IP 範囲を許可する設定があります。この設定を使用することで、管理者のアクセスを制限できます。

また、すべてのパブリックアクセスを拒否する設定も可能です。

---

## 4. アーキテクチャまたは設計のポイント

今回のシナリオでは、Private Endpoint と Storage Firewall を組み合わせることで高いセキュリティを実現できます。

まず、Private Endpoint を作成することで、ストレージアカウントへの接続を特定の仮想ネットワークからのプライベート接続に限定します。この構成により、アプリケーションはインターネットを経由せずにストレージへアクセスできます。

次に、Storage Firewall を使用してネットワークアクセスルールを設定します。この設定では、パブリックネットワークからのアクセスを拒否し、必要な IP 範囲のみ許可します。

このように複数のセキュリティ機能を組み合わせることで、ストレージアカウントのアクセス経路を厳格に制御することができます。

---

## 5. 設計判断（なぜこの構成になるか）

この問題のポイントは **VNet アクセス制限とパブリック IP 制御の両方を実現すること**です。

Private Endpoint は VNet 内のプライベート接続を提供しますが、IP 範囲ベースの管理アクセス制御までは提供しません。一方、Storage Firewall は IP 範囲ベースのアクセス制御を提供しますが、プライベート接続を作成する機能はありません。

そのため、この2つの機能を組み合わせる必要があります。

Private Endpoint により VNet からのプライベートアクセスを実現し、Storage Firewall によりパブリックアクセスの拒否と管理 IP 制御を行います。この構成により、セキュリティ要件をすべて満たすことができます。

---

## 6. 他の選択肢が誤りな理由

### Service Endpoints

Service Endpoint は VNet から Azure サービスへの接続を Azure バックボーンに拡張する機能です。しかしパブリックエンドポイントは引き続き存在するため、完全なプライベート接続を保証するものではありません。

### Private Endpoint のみ

Private Endpoint は VNet からのアクセスをプライベート化できますが、IP 範囲ベースの管理アクセス制御までは提供できません。そのため要件を完全には満たせません。

### Network Security Group

NSG はサブネットや NIC レベルでトラフィックを制御する機能です。しかし Azure Storage のパブリックエンドポイントに対するアクセス制御は行えません。
