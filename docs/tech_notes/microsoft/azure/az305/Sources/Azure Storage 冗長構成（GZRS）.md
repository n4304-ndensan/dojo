# Azure Storage 冗長構成（GZRS）

## 1. 背景（シナリオ）

ある組織では、グローバルに利用されるアプリケーションのクラウドアーキテクチャを設計しています。このアプリケーションは世界中のユーザーが利用するため、データアクセスのパフォーマンスと可用性が非常に重要になります。

特に、ユーザー体験を維持するためには **低遅延でデータにアクセスできること**が重要です。

さらに、アプリケーションの可用性を確保するために、次のような災害対策も必要になります。

- リージョン障害からの保護
- 自動フェイルオーバー
- 高いデータ耐久性

このような要件を満たすためには、Azure Storage の冗長構成を適切に選択する必要があります。

---

## 2. 要件整理

この問題の要件を整理すると、次のポイントが重要になります。

まず、パフォーマンス要件です。

- グローバルアプリケーション
- 低レイテンシアクセス

次に、可用性要件です。

- リージョン障害からの保護
- 自動フェイルオーバー

最後に、データ整合性です。

- 強力な一貫性
- 最新データの保証

これらの条件を満たすには、**リージョン間冗長とゾーン冗長を組み合わせたストレージ構成**が必要になります。

---

## 3. 技術の基本概念

Azure Storage では、データの冗長性を確保するために複数の冗長オプションが提供されています。

主な冗長方式には次のものがあります。

まず、単一データセンターの冗長です。

- **LRS（Locally Redundant Storage）**  
  同一データセンター内で3コピー

次に、リージョン内冗長です。

- **ZRS（Zone Redundant Storage）**  
  同一リージョンの複数ゾーンにレプリケーション

さらに、リージョン間冗長です。

- **GRS（Geo Redundant Storage）**  
  別リージョンへデータ複製

そして、これらを組み合わせた方式があります。

- **GZRS（Geo-Zone Redundant Storage）**

GZRS は、次の2つの冗長性を組み合わせたストレージです。

- ゾーン冗長（Availability Zones）
- ジオ冗長（別リージョン）

---

## 4. アーキテクチャまたは設計のポイント

GZRS を使用すると、次のようなデータ保護構成になります。

まず、データはプライマリリージョンの **複数の可用性ゾーン**にレプリケートされます。

その後、同じデータが **セカンダリリージョン**にも非同期レプリケーションされます。

この構成の特徴を整理すると次の通りです。

- ゾーン障害への耐性
- リージョン障害への耐性
- 自動フェイルオーバー
- 高いデータ耐久性

この仕組みにより、アプリケーションは地域障害が発生しても継続的にデータにアクセスできます。

---

## 5. 設計判断（なぜこの構成になるか）

この問題では **リージョン障害対策と低遅延アクセス**が重要です。

GZRS は次の特徴を持っています。

まず、ゾーンレベルの可用性です。

- 複数の Availability Zone にデータ複製
- データセンター障害への耐性

次に、リージョンレベルの可用性です。

- セカンダリリージョンへのレプリケーション
- 災害復旧対応

さらに、自動フェイルオーバーにも対応しています。

このため、グローバルアプリケーションの可用性と耐障害性を両立できます。

---

## 6. 他の選択肢が誤りな理由

まず **RA-GRS** です。

RA-GRS はセカンダリリージョンから読み取り可能ですが、ゾーン冗長がありません。

つまり、ゾーン障害への耐性は GZRS より低くなります。

次に **GRS + Soft Delete** です。

Soft Delete は削除されたデータを復元するための機能であり、冗長性とは直接関係ありません。

最後に **LRS + Azure Backup** です。

LRS は単一データセンター内の冗長であり、リージョン障害からデータを保護できません。

そのため、このシナリオの要件を満たしません。

---

## 7. 最終回答

**A. Geo-Zone-Redundant Storage (GZRS)**

---

## 8. まとめ

Azure Storage の冗長オプションは、可用性要件に応じて選択する必要があります。

代表的な冗長方式を整理すると理解しやすくなります。

まず、データセンター冗長です。

- **LRS**
  → 同一データセンター冗長

次に、リージョン内冗長です。

- **ZRS**
  → 可用性ゾーン冗長

さらに、リージョン間冗長です。

- **GRS**
  → 地理冗長

最後に、複合冗長です。

- **GZRS**
  → ゾーン冗長 + 地理冗長

そのため、**グローバルアプリケーションで低遅延アクセス、リージョン障害保護、自動フェイルオーバーを実現する場合は GZRS が最適なストレージ冗長構成**になります。
