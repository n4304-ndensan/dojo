# Azure Storage 設計ドキュメント

（性能・コスト・耐障害性・データ保護を考慮したストレージアーキテクチャ）

---

# 1 Azure Storage の概要

Azure Storage は、Microsoft Azure が提供するクラウドストレージサービスであり、大規模データを高可用性・高耐久性で保存するための基盤を提供する。オンプレミスのファイルサーバーやアプリケーションのデータをクラウドへ移行する場合、Azure Storage は非常に重要な役割を果たす。

Azure Storage は、単なるデータ保存場所ではなく、**性能、コスト、可用性、耐障害性、セキュリティなど複数の要素を設計段階で考慮する必要があるサービス**である。そのため、ストレージ設計を行う際には、以下の観点を総合的に評価する必要がある。

- データアクセス頻度
    
- 必要なパフォーマンス（レイテンシ・IOPS）
    
- データ保存期間
    
- データセンター障害への耐性
    
- リージョン災害への耐性
    
- ストレージコスト
    

Azure Storage は、これらの要件に応じて様々な構成を選択できるようになっており、**適切なストレージアカウント種類、アクセス層、冗長構成を組み合わせて設計することが重要**となる。

---

# 2 Azure Storage の主要サービス

Azure Storage は用途に応じて複数のストレージサービスを提供している。代表的なものは以下の通りである。

|サービス|用途|
|---|---|
[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Azure Blob Storage]]
|Blob Storage|オブジェクトデータ保存|
[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Azure Files]]
|Azure Files|SMB/NFS ファイル共有|
|Queue Storage|メッセージキュー|
|Table Storage|NoSQL Key-Value ストア|

Azure試験や設計問題では、特に次の2つが頻繁に登場する。

**Blob Storage**  
ログ、バックアップ、動画、画像などの非構造化データを保存するオブジェクトストレージであり、Azure Storageの中心的サービスである。

**Azure Files**  
SMBまたはNFSプロトコルを使用してファイル共有を提供するサービスであり、オンプレミスのファイルサーバーをクラウドに移行する際によく利用される。

---

# 3 Storage Account の種類

Azure Storage を利用するためには、まず **Storage Account（ストレージアカウント）** を作成する必要がある。このストレージアカウントの種類によって、利用できる機能や性能、料金体系が異なる。

主なストレージアカウントの種類は以下の通りである。

|種類|特徴|
|---|---|
|Standard Storage|HDDベースで低コスト|
|Premium Storage|SSDベースで高性能|

---

## Standard Storage

Standard Storage は HDD を基盤としたストレージであり、Azure Storage の中でも最も一般的な構成である。コストが低く、長期保存やバックアップ用途に適している。

特徴としては次のような点が挙げられる。

- ストレージコストが低い
    
- Blob tier（Hot / Cool / Archive）をサポート
    
- 一般的なアプリケーションのデータ保存に適している
    

一方で、SSDベースのストレージと比較すると、レイテンシやIOPSの面では性能が劣るため、**高トランザクションのワークロードには向かない場合がある**。

---

## Premium Storage

Premium Storage は SSD を基盤とした高性能ストレージである。高IOPSと低レイテンシを実現できるため、パフォーマンス要求の高いアプリケーションに適している。

主な特徴は以下の通りである。

- SSDベースの高速ストレージ
    
- 低レイテンシ
    
- 高トランザクション処理
    
- データベースや分析ワークロード向け
    

Azure試験では

「**低遅延**」「**高トランザクション**」「**高IOPS**」

といったキーワードが出てきた場合、Premium Storage を選択するケースが多い。

---

# 4 GPv1 と GPv2

Azure Storage Account には世代があり、主に次の2つが存在する。

|種類|特徴|
|---|---|
|GPv1|旧世代|
|GPv2|最新世代|

GPv2（General Purpose v2）は、現在の Azure Storage の標準構成であり、以下の機能をサポートする。

- Blob tier
    
- Lifecycle management
    
- 最新のストレージ機能
    

そのため、設計問題では

**GPv2 が最も汎用的で推奨される構成**となる。

---

# 5 Blob Storage のアクセス層（Access Tier）

Blob Storage は、アクセス頻度に応じてストレージコストを最適化できるように、複数のアクセス層を提供している。

|Tier|用途|コスト|
|---|---|---|
|Hot|頻繁アクセス|高|
|Cool|低頻度アクセス|低|
|Archive|長期保存|最安|

---

## Hot Tier

Hot tier は頻繁にアクセスされるデータ向けのストレージ層である。読み取りおよび書き込みのレイテンシが低く、頻繁にアクセスされるデータに適している。

例えば次のような用途で利用される。

- Webアプリケーションのコンテンツ
    
- 動画配信
    
- 分析データ
    

---

## Cool Tier

Cool tier はアクセス頻度が低いデータ向けのストレージ層である。保存コストは低いが、アクセスコストはHot tierより高くなる。

主な用途は以下の通りである。

- バックアップデータ
    
- 長期保存ログ
    
- 災害対策用データ
    

---

## Archive Tier

Archive tier は長期間アクセスされないデータを最も低コストで保存するための層である。ただし、アーカイブデータを読み出す場合は、数時間の復元時間が必要になる。

そのため、次のような用途で利用される。

- 法令保存
    
- コンプライアンスデータ
    
- アーカイブログ
    

---

# 6 Azure Storage の冗長構成

Azure Storage は、データの耐久性と可用性を確保するために、複数の冗長構成を提供している。

|冗長性|説明|
|---|---|
|LRS|同一データセンター|
|ZRS|複数ゾーン|
|GRS|別リージョン|

---

## LRS

LRS（Local Redundant Storage）は、同一データセンター内で3つのコピーを保持する構成である。最も低コストであるが、データセンター障害には対応できない。

---

## ZRS

ZRS（Zone Redundant Storage）は、同一リージョン内の複数のアベイラビリティゾーンにデータを複製する構成である。

この構成では、1つのデータセンターが停止してもサービスを継続できるため、**高可用性を実現できる**。

---

## GRS

GRS（Geo Redundant Storage）は、別リージョンにデータを複製する構成であり、リージョン災害に備えることができる。

ただし、レプリケーションは非同期であり、フェイルオーバーには操作が必要となる。

---

# 7 ストレージ設計の判断手順

Azure Storage の設計問題では、以下の順序で考えると解きやすい。

```
1 パフォーマンス要件
2 コスト要件
3 耐障害性
4 データアクセスパターン
```

この順序で考えると、適切な構成が導きやすい。

---

# 8 実際の設計例

例えば次の要件を考える。

**オンプレミスのファイルサーバーを Azure Blob Storage に移行する。  
データセンター障害にも対応し、コストを最小化したい。  
また、ストレージ層を利用できる必要がある。**

この要件を分析すると次のようになる。

ストレージ層を利用する必要があるため、GPv2 が必要となる。  
また、コストを最小化するためには Standard Storage を使用する必要がある。  
さらに、データセンター障害に対応するためには ZRS を選択する必要がある。

したがって最適な構成は

**Standard GPv2 + ZRS**

となる。

---

# 9 まとめ

Azure Storage 設計では、次の3つの要素を組み合わせて構成を決定する。

```
Storage Type
Access Tier
Replication
```

Azure試験では特に次の対応関係を覚えておくと解きやすい。

|要件|答え|
|---|---|
|高性能|Premium|
|最低コスト|Cool|
|Tier必要|GPv2|
|データセンター障害|ZRS|
|リージョン災害|GRS|

これらを理解しておくことで、Azure Storage の設計問題を体系的に解くことができる。
