# Azure Storageの冗長性設計（RA-GZRSとAzure Files）

## 1. 背景（シナリオ）

ある企業では、オンプレミス環境で運用されているファイル共有システムを **Azure Storage** に移行する計画があります。  
このファイル共有は、オンプレミスのアプリケーションから **SMB（Server Message Block）プロトコル** を使用してアクセスされます。

そのため、Azureでは **Azure Files** を使用して、クラウド上のファイル共有を提供する設計が検討されています。

このシステムでは以下の要件を満たす必要があります。

- SMBプロトコルでアクセスできること
- 低遅延でアクセスできること
- 高可用性を確保すること
- リージョン障害（地域停電）から迅速に復旧できること

これらの要件を満たすために、Azure Storageの **冗長性オプション（Redundancy Option）** の選択が重要になります。

## 2. 要件整理

この問題では、Azure Storageの冗長性オプションの中から最適なものを選択する必要があります。

まず、システム要件を整理します。

このシナリオで重要な要件は次の通りです。

- オンプレミスアプリケーションが **SMBでファイル共有にアクセス**
- **低遅延**
- **高可用性**
- **リージョン障害からの迅速な復旧**
- 災害時にもデータアクセスを継続できる設計

これらの要件は、単なるローカル冗長ではなく **ゾーン冗長 + 地理冗長** の組み合わせが必要であることを意味します。

## 3. 技術の基本概念

Azure Storageには、データ保護のために複数の冗長性オプションが存在します。

ここでは主要な4種類の冗長構成について理解することが重要です。

### LRS（Local Redundant Storage）

まず、最も基本的な冗長方式が **LRS（ローカル冗長ストレージ）** です。

LRSでは次のような構成になります。

- 単一データセンター内で3コピー
- リージョン障害には対応不可
- 最も安価

つまり、**リージョン障害には対応できません。**

### ZRS（Zone Redundant Storage）

次に **ZRS（ゾーン冗長ストレージ）** です。

ZRSでは以下のような仕組みになります。

- 同一リージョンの **複数の可用性ゾーン**
- ゾーン障害に耐性
- リージョン障害には対応不可

つまり、**データセンター障害には強いがリージョン障害には弱い構成**です。

### GZRS（Geo-Zone Redundant Storage）

GZRSは次の特徴を持ちます。

- プライマリリージョンの **複数ゾーンにレプリケーション**
- セカンダリリージョンにも非同期コピー

つまり、

- ゾーン障害
- リージョン障害

両方に耐性があります。

ただし **セカンダリリージョンは読み取り不可**です。

### RA-GZRS（Read Access Geo-Zone Redundant Storage）

RA-GZRSはGZRSを拡張した冗長構成です。

この構成では次の仕組みになります。

- プライマリリージョンの **複数ゾーン**
- セカンダリリージョンに非同期コピー
- **セカンダリリージョンから読み取り可能**

つまり、以下の利点があります。

- 高可用性
- 災害耐性
- 読み取りアクセス継続

## 4. アーキテクチャまたは設計のポイント

この問題では、次の3つの要件が特に重要です。

まず **高可用性**です。

高可用性を実現するには、データが複数の可用性ゾーンに分散される必要があります。

次に **リージョン障害からの復旧**です。

これは **地理冗長（Geo-replication）** が必要になります。

最後に **継続的アクセス**です。

災害時でもアプリケーションがデータを読み取れる必要があります。

これらを整理すると、必要な機能は以下になります。

- ゾーン冗長
- 地理冗長
- セカンダリ読み取り

この3つをすべて満たすのが **RA-GZRS** です。

## 5. 設計判断（なぜこの構成になるか）

最適な選択肢は **RA-GZRS（Read Access Geo-Zone Redundant Storage）** です。

この構成の利点を整理します。

まず、低遅延についてです。

RA-GZRSでは、通常は **プライマリリージョン内のゾーン**からデータが提供されます。

そのため、

- ローカルアクセス
- 低遅延

を実現できます。

次に高可用性です。

RA-GZRSでは以下の構造になります。

- 複数の可用性ゾーン
- 自動レプリケーション

そのため **ゾーン障害でもサービスは継続します。**

さらに重要なのが **リージョン障害対応**です。

RA-GZRSでは

- セカンダリリージョン
- 読み取りアクセス

が可能です。

つまり、プライマリリージョン障害時でも **セカンダリから読み取り可能**です。

これにより **迅速な災害復旧（DR）** が可能になります。

## 6. 他の選択肢が誤りな理由

### A. GZRS

GZRSは以下の特徴を持ちます。

- ゾーン冗長
- 地理冗長

しかし問題は **セカンダリ読み取りができない**ことです。

つまり、災害時の継続アクセスが制限されます。

### B. LRS

LRSは最も基本的な冗長方式です。

特徴は以下です。

- 単一データセンター
- リージョン障害に弱い

そのため **災害復旧要件を満たせません。**

### D. ZRS

ZRSは以下の特徴を持ちます。

- 同一リージョン内のゾーン冗長

しかし **リージョン障害には対応できません。**

そのため要件の

「地域の停電からの迅速な復旧」

を満たせません。

## 7. 最終回答

最適な冗長性オプションは次です。

**RA-GZRS（読み取りアクセス可能なジオゾーン冗長ストレージ）**

したがって **正解は C** です。

## 8. まとめ

Azure Storageの冗長性設計では、可用性レベルに応じて適切なオプションを選択する必要があります。

重要なポイントを整理します。

まず、Azure Storageの冗長性には以下の種類があります。

- LRS（単一データセンター）
- ZRS（ゾーン冗長）
- GZRS（ゾーン + 地理冗長）
- RA-GZRS（ゾーン + 地理冗長 + セカンダリ読み取り）

今回の問題では以下の要件がありました。

- 高可用性
- リージョン障害対応
- 継続アクセス

これらをすべて満たすのは **RA-GZRSのみ**です。

そのため、Azureストレージ設計において最適な選択は **RA-GZRS** となります。
