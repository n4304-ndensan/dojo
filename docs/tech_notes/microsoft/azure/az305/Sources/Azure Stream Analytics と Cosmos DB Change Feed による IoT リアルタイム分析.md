# Azure Stream Analytics と Cosmos DB Change Feed による IoT リアルタイム分析

## 1. 背景（シナリオ）

ある企業では、約 10 万台の IoT デバイスを管理する大規模システムを構築しています。これらのデバイスは温度や湿度などのセンサーデータを毎秒送信し続けるため、非常に大量のデータが継続的に生成されます。

このデータは単に保存するだけでなく、リアルタイムに近い形で分析する必要があります。例えば、温度が閾値を超えた場合にアラートを出す、設備の状態をダッシュボードで監視する、または異常検知を行うといった用途が考えられます。

そのため、このシステムでは以下の3つの要素を満たすアーキテクチャが求められます。

1つ目は、大量のストリーミングデータを効率よく取り込むことです。  
2つ目は、低遅延でデータ処理を行うことです。  
3つ目は、処理されたデータをスケーラブルなストレージに保存し、リアルタイム分析を可能にすることです。

## 2. 要件整理

この問題では、IoT デバイスから送信されるストリーミングデータを処理するための Azure サービスの組み合わせを選択する必要があります。

問題文から読み取れる要件は次の通りです。

まず、10 万台の IoT デバイスが毎秒データを送信するため、大量のストリーミングデータを処理できるサービスが必要になります。

次に、収集したデータはほぼリアルタイムで分析できる必要があります。つまり、バッチ処理ではなくストリーム処理が必要です。

さらに、データは低遅延で保存され、リアルタイム分析やイベント処理に利用できる必要があります。

これらの要件をまとめると次の通りです。

- IoT データのリアルタイム処理  
- 大量データを処理できるスケーラブルなサービス  
- 低遅延のデータ保存  
- データ変更をトリガーにしたリアルタイム処理  

## 3. 技術の基本概念

IoT システムでは、一般的に **ストリーム処理 + スケーラブルなデータストア** の組み合わせが使用されます。

Azure Stream Analytics はリアルタイムデータストリームを処理するためのサービスです。SQL に似たクエリを使用して、ストリームデータをフィルタリング、集計、変換できます。

例えば次のような処理が可能です。

- 温度データの平均値を計算  
- 閾値を超えたイベントを検出  
- 時系列ウィンドウ分析  

一方、Azure Cosmos DB はグローバルに分散された NoSQL データベースです。低遅延アクセスと自動スケーリングを提供するため、大量の IoT データ保存に適しています。

さらに Cosmos DB には **Change Feed** という機能があります。これはデータの変更（追加や更新）をリアルタイムで取得できる仕組みです。

## 4. アーキテクチャまたは設計のポイント

このシナリオでは、IoT データのストリーム処理とリアルタイム分析を組み合わせたアーキテクチャが必要になります。

まず、Azure Stream Analytics を使用して IoT データストリームをリアルタイムで処理します。Stream Analytics は IoT Hub や Event Hub からデータを取り込み、SQL ライクなクエリを使用してデータを分析できます。

次に、処理されたデータを Azure Cosmos DB に保存します。Cosmos DB は低遅延アクセスと高スケーラビリティを提供するため、IoT データの保存に適しています。

さらに Cosmos DB の Change Feed を利用することで、新しいデータが追加された際にリアルタイムでイベントを検知できます。これにより、ダッシュボード更新やアラート処理などのリアルタイム分析が可能になります。

## 5. 設計判断（なぜこの構成になるか）

この問題の正解は次の2つです。

A. Azure Stream Analytics  
D. Azure Cosmos DB（Change Feed）

Azure Stream Analytics はストリーミングデータをリアルタイムで処理するためのサービスであり、大量の IoT データを分析するのに適しています。

Azure Cosmos DB は低遅延でスケーラブルな NoSQL データベースであり、IoT データを効率的に保存できます。さらに Change Feed を利用することで、データ変更に基づくリアルタイム処理が可能になります。

この2つを組み合わせることで、大規模 IoT システムに必要なリアルタイム分析、スケーラビリティ、低遅延処理を実現できます。

## 6. 他の選択肢が誤りな理由

Azure Blob Storage は大量データ保存には適していますが、リアルタイム分析には向いていません。Blob Storage は主にデータレイクやバッチ処理用途で使用されます。

Azure Synapse Analytics は大規模なデータウェアハウス分析を行うサービスです。主にバッチ分析や BI ワークロードに使用されるため、リアルタイム IoT ストリーム処理には適していません。

## 7. 最終回答

A. Azure Stream Analytics  
D. Azure Cosmos DB（Change Feed）

## 8. まとめ

この問題は、IoT データ処理のリアルタイムアーキテクチャを理解しているかを確認するものです。

IoT システムでは、大量のストリーミングデータをリアルタイムに処理する必要があります。そのため、ストリーム処理サービスとスケーラブルなデータストアの組み合わせが一般的です。

Azure Stream Analytics はリアルタイムデータ処理を担当し、Azure Cosmos DB は低遅延のデータ保存と Change Feed によるイベント処理を提供します。

この2つのサービスを組み合わせることで、大規模 IoT システムにおけるリアルタイム分析と低遅延処理を実現できます。
