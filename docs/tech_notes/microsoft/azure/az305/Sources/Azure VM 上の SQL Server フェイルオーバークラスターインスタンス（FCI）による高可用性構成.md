# Azure VM 上の SQL Server フェイルオーバークラスターインスタンス（FCI）による高可用性構成

## 1. 背景（シナリオ）

ある組織は、オンプレミス環境で運用している **SQL Server 2017** を Azure に移行する計画を進めています。このデータベースはミッションクリティカルなシステムで使用されているため、システム停止が業務に大きな影響を与える可能性があります。

そのため、移行後の環境では高可用性を確保しながら、フェイルオーバー発生時のダウンタイムを最小限に抑える必要があります。また、アプリケーション接続は既存の構成を維持するため、**仮想ネットワーク名（VNN）**を使用する必要があります。

さらに、組織はクラウド移行においてコストも重要視しており、可能な限りコスト効率の高いアーキテクチャを採用する必要があります。

## 2. 要件整理

このシナリオでは、高可用性構成の設計において次のような要件が存在します。

まず、インスタンス間で自動的にフェイルオーバーできる仕組みが必要です。これにより、障害が発生した場合でもサービスの継続性を維持できます。

次に、フェイルオーバーが発生した場合でもアプリケーションの停止時間を最小限に抑えるため、フェイルオーバー時間が短い構成が必要です。

さらに、アプリケーション接続では仮想ネットワーク名（VNN）を使用する必要があります。これにより、アプリケーション側の接続設定を変更せずに、アクティブなノードへ接続を切り替えることができます。

この問題の要件を整理すると、次のようになります。

・自動フェイルオーバー  
・高速フェイルオーバー  
・仮想ネットワーク名（VNN）による接続  
・高可用性構成  
・コストを最小化  

## 3. 技術の基本概念

SQL Server の高可用性を実現するためには、いくつかの技術が存在します。その中でも重要なのが **Failover Cluster Instance（FCI）** です。

SQL Server FCI は Windows Server Failover Clustering（WSFC）を利用して複数のノードで SQL Server を構成する仕組みです。1つのノードで障害が発生した場合、別のノードが自動的にサービスを引き継ぐことができます。

FCI の特徴として、以下のようなポイントがあります。

・インスタンス単位で高可用性を提供  
・自動フェイルオーバーが可能  
・単一の仮想ネットワーク名（VNN）で接続  
・共有ストレージを利用  

Azure 環境では、この共有ストレージとして **Shared Premium Disk** を利用することで FCI を構成できます。共有ディスクにより、複数の VM ノードが同じデータへアクセスできるようになります。

## 4. アーキテクチャまたは設計のポイント

今回のシナリオでは、Azure Virtual Machines 上に SQL Server を配置し、Windows Server Failover Clustering を構成します。

この構成では、複数の仮想マシンをクラスタノードとして設定し、SQL Server Failover Cluster Instance を構築します。クラスタノードは共有プレミアムディスクを使用して同じデータストレージにアクセスします。

この構成により、アクティブノードが障害を起こした場合でも、別のノードが自動的にインスタンスを引き継ぐことができます。また、アプリケーションは仮想ネットワーク名（VNN）を使用して接続するため、アクティブノードの変更を意識する必要がありません。

## 5. 設計判断（なぜこの構成になるか）

SQL Server Failover Cluster Instance はインスタンスレベルで高可用性を提供するため、アプリケーション接続を変更せずにフェイルオーバーを実現できます。

さらに FCI は仮想ネットワーク名（VNN）を利用するため、アプリケーションは常に同じ接続先を使用できます。フェイルオーバーが発生した場合でも、DNS の更新により新しいノードへ接続が切り替わります。

また、Shared Premium Disk を使用することで複数のノードから同じデータへアクセスできるため、迅速なフェイルオーバーが可能になります。

この構成は SQL Server のネイティブ機能を活用するため、既存の SQL Server 環境を Azure に移行する場合にも互換性が高く、コスト面でも効率的です。

## 6. 他の選択肢が誤りな理由

この問題では複数の Azure データベースソリューションが提示されていますが、それぞれ要件を完全には満たしていません。

まず、Always On Availability Groups は高可用性を提供しますが、仮想ネットワーク名（VNN）ではなく分散ネットワーク名（DNN）を使用する構成になります。そのため、今回の要件とは一致しません。

次に SQL Server Managed Instance は高可用性を提供しますが、インスタンスレベルのクラスタ制御や VNN の使用など、FCI と同じ構成を実現することはできません。

また Azure SQL Database Hyperscale は PaaS 型のデータベースサービスであり、SQL Server インスタンスレベルの構成や VNN 接続を使用することができません。

## 7. 最終回答

C. Azure Virtual Machines 上の SQL Server に、Always On フェイルオーバークラスターインスタンス（FCI）と共有プレミアムディスクを構成する

## 8. まとめ

ミッションクリティカルな SQL Server を Azure に移行する場合、高可用性構成を適切に設計することが重要です。

SQL Server Failover Cluster Instance（FCI）は、Windows Server Failover Clustering を利用して自動フェイルオーバーを実現し、仮想ネットワーク名（VNN）を使用したアプリケーション接続を提供します。

Azure Virtual Machines と Shared Premium Disk を組み合わせることで、オンプレミスと同様の高可用性構成を Azure 上で実現でき、フェイルオーバー時間を最小化しながらミッションクリティカルなデータベースを安全に運用することができます。
