# Azure VNet 間通信のセキュア接続（VPN Gateway + IPsec）

## 1. 背景（シナリオ）

ある組織では、2つの Azure Virtual Network（VNet）間でデータ交換を行う必要があります。これらのネットワークは異なる環境またはリージョンに存在しており、安全な通信経路を確立することが求められています。

さらに、この通信で扱われるデータには機密情報が含まれているため、通信中のデータを暗号化し、不正アクセスや盗聴から保護する必要があります。そのため、単にネットワーク接続を確立するだけではなく、強力な暗号化と認証機能を備えた通信方式を採用することが重要です。

このような要件を満たすためには、Azureネットワークサービスの中から適切なセキュア接続方法を選択する必要があります。

## 2. 要件整理

このシナリオから読み取れる要件を整理すると、次のポイントが重要になります。

- 2つのAzure Virtual Network間で通信する  
- 安全な通信チャネルを確立する  
- 転送中のデータを暗号化する  
- セキュリティのベストプラクティスに従う  
- 機密データの保護  

この要件から、単なるネットワーク接続ではなく、暗号化されたトンネルを使用した安全な接続が必要であることが分かります。

## 3. 技術の基本概念

Azureでは、VNet間の接続を実現するために複数の方法が提供されています。代表的な方法として次のものがあります。

- VNet Peering  
- VPN Gateway  
- ExpressRoute  

これらの接続方式の中で、暗号化されたトンネル通信を提供するのが VPN Gateway です。

VPN Gateway は IPsec（Internet Protocol Security）プロトコルを利用して安全な通信トンネルを構築します。IPsec はネットワーク通信の暗号化と認証を提供する標準プロトコルであり、多くの企業ネットワークで利用されています。

IPsecトンネルには次の特徴があります。

- データの暗号化  
- データの整合性検証  
- 通信相手の認証  
- 安全なトンネル通信

この仕組みにより、ネットワーク間で送信されるデータを安全に保護することができます。

## 4. アーキテクチャまたは設計のポイント

AzureでVNet間の安全な通信を設計する際には、VPN Gatewayを使用してトンネル接続を構築する方法が一般的です。

この構成では、それぞれのVNetにVPN Gatewayを配置し、両ネットワーク間でIPsecトンネルを確立します。このトンネルを通じて通信が行われるため、ネットワーク間のデータはすべて暗号化されます。

IPsecトンネルでは通常、次のような暗号化アルゴリズムが使用されます。

- AES（Advanced Encryption Standard）  
- SHAベースの認証アルゴリズム  

このような暗号化プロトコルを使用することで、通信内容の盗聴や改ざんを防ぐことができます。

## 5. 設計判断（なぜこの構成になるか）

この問題では「転送中データの暗号化」が最も重要な要件です。

VNet Peering は高速な接続を提供しますが、トラフィックの暗号化を提供するものではありません。ネットワーク内通信として扱われるため、暗号化されたトンネルは作成されません。

一方、VPN Gateway は IPsec を利用して暗号化トンネルを作成できるため、機密データを安全に転送することができます。

そのため、VNet間の安全な通信を実現するためには VPN Gateway と IPsec トンネルを使用する構成が最も適しています。

## 6. 他の選択肢が誤りな理由

A. VNet Peering + NSG  

VNet Peering は高速接続を提供しますが、トラフィックを暗号化する機能はありません。NSG はアクセス制御を行うための機能であり、暗号化は提供しません。

B. Azure ExpressRoute  

ExpressRoute はプライベート接続を提供しますが、必ずしもトラフィックの暗号化を保証するものではありません。また、VNet間通信のためには過剰なソリューションになる場合があります。

D. Application Gateway + WAF  

Application Gateway は主に HTTP/HTTPS トラフィックを処理するロードバランサーです。VNet間の一般的なネットワーク通信を暗号化する用途には適していません。

## 7. 最終回答

C. VPN Gateway で両方の VNet を設定し、IPsec トンネルを確立する

## 8. まとめ

Azure で 2つの VNet 間の安全な通信を確立するためには、暗号化されたトンネル通信を使用することが重要です。

VPN Gateway を利用すると、IPsec プロトコルによる暗号化トンネルを構築でき、データの機密性、整合性、認証を確保することができます。

そのため、VNet間通信を安全に実現する最も適切な方法は VPN Gateway と IPsec トンネルを使用する構成です。
