# Azure Virtual Machine Scale Sets による自動スケーリング

## 1. 背景（シナリオ）

クラウド環境でWebアプリケーションを運用する場合、トラフィックは時間帯やユーザー数によって大きく変動することがあります。例えば、キャンペーンやイベントなどによりアクセスが急増するケースもあります。

このような状況では、固定数の仮想マシンだけではピークトラフィックを処理できない可能性があります。一方で、常に大量の仮想マシンを稼働させるとコストが増加してしまいます。

そのためクラウド環境では、需要に応じてリソースを自動的に増減させる **オートスケーリング（Auto Scaling）** の仕組みが重要になります。

この問題では、Azure仮想マシンで実行されているWebアプリケーションのトラフィックが変動するため、**仮想マシンを自動的にスケーリングする方法**を選択する必要があります。

## 2. 要件整理

問題文から読み取れる重要な要件を整理すると、スケーラブルなインフラ構成が必要であることが分かります。

このシナリオでは次の要件があります。

・Webアプリケーションが複数のVMで実行されている  
・トラフィック量が変動する  
・需要に応じてVM数を増減する必要がある  
・自動スケーリングが必要  
・高可用性とスケーラビリティが必要  

つまり、**仮想マシンの数を自動的に増減できる仕組み**が必要になります。

## 3. 技術の基本概念

Azureでは仮想マシンをスケーラブルに運用するためのサービスとして **Virtual Machine Scale Sets（VMSS）** が提供されています。

VM Scale Set は、同じ構成を持つ複数の仮想マシンをまとめて管理できるサービスです。これにより、大規模な仮想マシン環境を効率的に運用できます。

VM Scale Set の主な特徴には次のようなものがあります。

・同一構成のVMを複数作成できる  
・需要に応じてVM数を自動的に増減できる  
・Azure Load Balancerと統合できる  
・高可用性を確保できる  
・自動修復（VMの置き換え）機能  

この仕組みにより、トラフィック量に応じて仮想マシンの数を動的に調整できます。

## 4. アーキテクチャまたは設計のポイント

VM Scale Set を使用すると、スケーリングポリシーを設定することで自動スケーリングを実現できます。

例えば、次のような条件でスケーリングルールを設定できます。

・CPU使用率が70%以上になったらVMを追加する  
・CPU使用率が30%以下になったらVMを削減する  
・特定の時間帯にVMを増やす  
・カスタムメトリクスを使用する  

このようなルールにより、システムはトラフィック負荷に応じて自動的にスケールアウト（VM追加）やスケールイン（VM削減）を行います。

さらに、VM Scale Set は Azure Load Balancer と組み合わせて使用することが多く、複数のVMへトラフィックを均等に分散できます。

## 5. 設計判断（なぜこの構成になるか）

この問題の要件は、仮想マシンの数をトラフィック負荷に応じて自動的に増減させることです。

この要件を満たすためには、次の機能が必要になります。

・VMの自動スケーリング  
・同一構成VMの管理  
・高可用性  
・トラフィック負荷に応じたスケール  

Azureのサービスの中で、これらの機能を直接提供するものが **Virtual Machine Scale Sets（VMSS）** です。

VMSSを使用することで、トラフィックが増加した際にはVMを追加し、トラフィックが減少した際にはVMを削減することができます。

## 6. 他の選択肢が誤りな理由

まず Azure Load Balancer について説明します。Load Balancer は複数の仮想マシンにトラフィックを分散するサービスですが、仮想マシンの数を自動的に増減する機能はありません。

次に Azure Application Gateway です。Application Gateway は Layer7 ロードバランサーであり、URLベースのルーティングやWeb Application Firewallなどの機能を提供しますが、VMのスケーリング機能はありません。

最後に Azure Traffic Manager です。Traffic Manager はDNSベースのグローバルトラフィックルーティングサービスであり、リージョン間のトラフィック制御を行うものです。仮想マシンの数を管理する機能はありません。

## 7. 最終回答

A. 仮想マシンスケールセット（Virtual Machine Scale Sets）

## 8. まとめ

クラウド環境ではトラフィックが変動するため、インフラを動的にスケールできる仕組みが重要になります。AzureではVirtual Machine Scale Sets（VMSS）を使用することで、同一構成の仮想マシンを自動的に増減させることができます。

VMSSはAzure Load Balancerと統合して使用されることが多く、高可用性とスケーラビリティを実現できます。そのため、仮想マシンベースのWebアプリケーションで自動スケーリングを実現する場合には **Virtual Machine Scale Sets** を使用するのが最適です。
