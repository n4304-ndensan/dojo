# Azure Virtual Machine Scale Sets による高可用性 Web アプリケーション設計

## 1. 背景（シナリオ）

ある組織では、トラフィック量の変動が大きい Web アプリケーションを Azure 上で設計しています。このアプリケーションは多くのユーザーが利用するため、アクセスの急増にも耐えられるスケーラブルなインフラストラクチャが必要です。

さらに、このアプリケーションは高可用性を維持する必要があり、ダウンタイムを最小限に抑える仕組みが求められます。そのため、アプリケーションは複数の Azure 仮想マシン（VM）上に展開され、複数リージョンで稼働することでレジリエンスとパフォーマンスを向上させる構成になります。

また、トラフィックが増加した場合には自動的にインスタンス数を増やし、トラフィックが減少した場合にはインスタンス数を減らす自動スケーリング機能も必要です。さらに、ユーザーからのリクエストを複数の VM に均等に分散するロードバランシング機能も必要になります。

## 2. 要件整理

このシナリオでは、高可用性とスケーラビリティを実現するためにいくつかの重要な要件があります。

まず、トラフィック量の変動に応じて自動的にリソースを増減させる自動スケーリング機能が必要です。これにより、需要の増減に応じて効率的にリソースを管理できます。

次に、複数の仮想マシン間でトラフィックを分散するロードバランシング機能が必要です。これにより、特定の VM に負荷が集中することを防ぎます。

さらに、障害が発生した VM を検出し、正常なインスタンスへトラフィックをルーティングするヘルスチェック機能も必要になります。

これらの要件をまとめると、次のようになります。

・複数 VM へのトラフィック分散  
・自動スケーリング  
・ヘルスチェックによるフェイルオーバー  
・高可用性の確保  
・トラフィック変動への対応  

## 3. 技術の基本概念

Azure では、スケーラブルな仮想マシン環境を構築するためのサービスとして **Azure Virtual Machine Scale Sets（VMSS）** が提供されています。

VMSS は、同一構成の仮想マシンをグループとして管理し、必要に応じて自動的にインスタンス数を増減できるサービスです。

VMSS の主な特徴を理解することが重要です。

まず、VMSS は自動スケーリングをサポートしています。CPU 使用率やメモリ使用率、またはカスタムメトリックに基づいて仮想マシンの数を自動的に増減できます。

次に、VMSS は Azure Load Balancer と統合されており、ユーザーリクエストを複数の VM に分散できます。

さらに、Load Balancer はヘルスプローブを使用して VM の状態を監視します。異常な VM が検出された場合、そのインスタンスにはトラフィックが送られなくなります。

## 4. アーキテクチャまたは設計のポイント

このシナリオでは、VMSS を利用して複数の仮想マシンを自動的に管理します。VMSS により、トラフィックの増減に応じて VM 数を自動的にスケールさせることができます。

また、Azure Load Balancer を組み合わせることで、ユーザーからのリクエストを複数の VM インスタンスへ均等に分散できます。これによりアプリケーションのパフォーマンスを維持できます。

さらに、Load Balancer のヘルスプローブを利用することで、正常に動作していない VM を自動的に検出し、その VM へのトラフィックを停止することができます。

この構成により、高可用性とスケーラビリティを両立した Web アプリケーション環境を構築できます。

## 5. 設計判断（なぜこの構成になるか）

Azure Virtual Machine Scale Sets は、大規模な仮想マシン環境を効率的に管理するために設計されたサービスです。

特に、トラフィック量が変動する Web アプリケーションでは、需要に応じてインスタンス数を自動的に増減する仕組みが重要になります。VMSS はこの要件をネイティブにサポートしています。

さらに、Azure Load Balancer と統合することで、複数の VM に対してトラフィックを分散し、アプリケーションの負荷を均等化できます。

また、ヘルスチェック機能により、障害が発生したインスタンスを自動的に検出し、正常なインスタンスへトラフィックをルーティングできます。

## 6. 他の選択肢が誤りな理由

Azure Kubernetes Service（AKS）はコンテナベースのアプリケーションを管理するためのオーケストレーションサービスです。アプリケーションがコンテナ化されていない場合、導入には追加の複雑さが生じます。

Azure App Service は PaaS の Web アプリケーションプラットフォームであり、仮想マシンを直接管理するシナリオには適していません。

Azure Functions と Event Grid はイベント駆動型のサーバーレスアーキテクチャ向けのサービスであり、常時稼働する Web アプリケーションのインフラストラクチャ管理には適していません。

## 7. 最終回答

C. Azure 仮想マシン スケール セット（ロードバランシング付き）

## 8. まとめ

トラフィック量が変動する Web アプリケーションでは、スケーラブルで高可用性なインフラストラクチャを構築することが重要です。

Azure Virtual Machine Scale Sets は、自動スケーリングと負荷分散機能を組み合わせることで、トラフィックの増減に柔軟に対応できます。

さらに、Azure Load Balancer のヘルスチェック機能により、障害が発生したインスタンスを自動的に除外し、アプリケーションの信頼性を維持することができます。
