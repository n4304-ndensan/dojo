# Azure リアルタイム動画分析アーキテクチャ

（複数カメラ映像の物体検出）

---

# 1 背景

ある組織では、複数の監視カメラや IoT カメラから送信される **リアルタイム動画ストリーム**を分析し、特定の物体を検出するシステムを構築する必要がある。例えば、スマートシティ、工場の安全監視、交通監視、店舗分析などの分野では、カメラ映像をリアルタイムに分析し、人や車両、物体などを識別することが重要である。

このようなシステムでは、単に動画を保存するだけでなく、動画フレームを AI によって解析し、検出された物体の情報（メタデータ）を生成する必要がある。例えば、以下のようなデータが生成される。

- 検出された物体の種類（人物、車両など）
    
- 物体の位置（バウンディングボックス）
    
- 検出の信頼度スコア
    
- 検出時間
    

これらの情報は動画データそのものではなく **メタデータ**として保存され、後続の分析やダッシュボード、機械学習、異常検知などに利用される。

このようなリアルタイム動画分析システムでは、以下の処理が必要になる。

1. カメラからの動画ストリームの取り込み
    
2. フレーム単位での分析
    
3. AI による物体検出
    
4. 検出結果メタデータの生成
    
5. メタデータの保存と後続分析
    

これらの要件を満たす Azure サービスの組み合わせとして最適なのは次の2つである。

**Azure Video Analyzer**  
**Azure Cognitive Services Custom Vision**

---

# 2 Azure Video Analyzer

Azure Video Analyzer は、動画データの取り込みと分析を目的とした Azure サービスである。IoT カメラや IP カメラなどから送信される動画ストリームを処理し、リアルタイムで動画解析を実行することができる。

このサービスは、動画データを処理するための **パイプライン構築機能**を提供しており、カメラからのストリーム入力、AI モデルによる解析、結果の保存などを統合的に管理することができる。

Video Analyzer は次のような機能を提供する。

|機能|内容|
|---|---|
|動画ストリーム入力|カメラからの映像取り込み|
|フレーム処理|動画フレームの抽出|
|AI推論統合|物体検出などのAI処理|
|メタデータ生成|検出結果の構造化データ化|

このサービスは、動画ストリームをリアルタイムに処理するパイプラインの中心となるコンポーネントである。

---

# 3 Azure Custom Vision

Azure Custom Vision は、Azure Cognitive Services の一部として提供される **カスタム画像認識 AI モデル作成サービス**である。開発者は自分のデータを利用して物体検出モデルを学習し、特定の対象物を識別する AI を作成できる。

例えば次のような物体検出モデルを作成することができる。

- 人物検出
    
- 車両検出
    
- 工場設備の異常検出
    
- 商品認識
    

Custom Vision の物体検出モデルは、画像や動画フレームを入力として、検出された物体の位置と種類を返す。

出力例

```text
Object: Person
Confidence: 0.92
Bounding Box: (x,y,width,height)
```

この情報が **メタデータ**として保存され、後続の分析に利用される。

---

# 4 システムアーキテクチャ

複数カメラのリアルタイム動画分析の構成は次のようになる。

```text
Camera Streams
      │
      ▼
Azure Video Analyzer
      │
      ▼
Frame Extraction
      │
      ▼
Custom Vision Model
      │
      ▼
Object Detection
      │
      ▼
Metadata Storage
```

処理の流れは次の通りである。

1. カメラから動画ストリームが送信される
    
2. Video Analyzer が動画を取り込みフレームを抽出する
    
3. Custom Vision モデルがフレームを分析する
    
4. 検出された物体情報がメタデータとして生成される
    
5. メタデータがストレージやデータベースに保存される
    

---

# 5 メタデータ例

動画分析システムでは、動画そのものではなく次のような構造化データが生成される。

```json
{
  "cameraId": "Camera01",
  "timestamp": "2026-03-07T10:15:20Z",
  "objects": [
    {
      "type": "person",
      "confidence": 0.93,
      "bbox": [120,200,50,90]
    },
    {
      "type": "car",
      "confidence": 0.88,
      "bbox": [300,220,120,80]
    }
  ]
}
```

このデータは次のような用途に利用される。

- ダッシュボード可視化
    
- アラート通知
    
- 異常検知
    
- 機械学習トレーニング

# 6 他の選択肢が適切でない理由

## Azure Media Services

Azure Media Services は動画エンコード、配信、ストリーミングを目的としたサービスであり、主に動画配信や VOD（Video On Demand）システムで使用される。動画の配信やトランスコードには適しているが、リアルタイム物体検出の処理基盤としては最適ではない。

---

[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Azure Stream Analytics]]
## Azure Stream Analytics

Azure Stream Analytics は IoT テレメトリやイベントストリームをリアルタイム処理するサービスである。数値データやイベントデータのストリーム分析には適しているが、動画フレーム解析や AI 推論には対応していない。

---

[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Azure Event Grid]]
## Azure Event Grid

Azure Event Grid はイベント駆動型アーキテクチャを構築するためのイベントルーティングサービスである。システム間のイベント通知には有用だが、動画フレーム解析や AI 推論機能は提供しない。

---

# 7 推奨アーキテクチャ

最適な Azure サービスの組み合わせは次の通りである。

```text
Camera
   │
   ▼
Azure Video Analyzer
   │
   ▼
Frame Processing
   │
   ▼
Custom Vision
   │
   ▼
Object Detection
   │
   ▼
Metadata Storage
```

この構成により、動画のリアルタイム分析と物体検出を効率的に実現できる。

---

# 8 メリット

この構成を採用することでいくつかのメリットが得られる。

まず、Video Analyzer によってカメラ映像をリアルタイムに処理できるため、動画解析パイプラインを簡単に構築できる。また、Custom Vision によって特定用途に最適化された物体検出 AI を作成できる。

さらに、動画フレームから抽出されたメタデータはデータベースやデータレイクに保存できるため、大規模な分析や機械学習にも利用できる。

---

# 9 まとめ

今回の要件は次の通りである。

- 複数カメラのリアルタイム動画分析
    
- 動画フレームの処理
    
- 物体検出 AI の利用
    
- 検出結果メタデータの保存
    

これらの要件を満たす Azure サービスの組み合わせは

**Azure Video Analyzer**  
**Azure Cognitive Services Custom Vision**

である。

この構成により、リアルタイム動画分析と物体検出を効率的に実装することができる。
