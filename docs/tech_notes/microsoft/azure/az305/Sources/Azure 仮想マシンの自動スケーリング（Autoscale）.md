# Azure 仮想マシンの自動スケーリング（Autoscale）

## 1. 背景（シナリオ）

ある企業では、トラフィックの多い Web アプリケーションを Azure 仮想マシン（VM）上で運用しています。このアプリケーションでは、時間帯や利用状況によってアクセス数が大きく変動するため、インフラリソースを動的に調整する必要があります。

トラフィックが急増した場合には、追加の仮想マシンを自動的に起動して処理能力を高める必要があります。一方で、アクセスが減少した場合には不要な仮想マシンを削減し、コストを最適化することも重要です。

また、このスケーリング操作は運用担当者が手動で対応するのではなく、**完全に自動化された仕組み**で実行される必要があります。

## 2. 要件整理

このシナリオでは、単に仮想マシンを増減させるだけではなく、アプリケーションの可用性とコスト効率を同時に満たす必要があります。

問題文から読み取れる要件を整理すると次の通りです。

まず、トラフィックの増減に応じて仮想マシンを自動的にスケールさせる必要があります。次に、スケーリング操作によってダウンタイムが発生しないようにする必要があります。

さらに、不要なリソースを停止することでコストを最適化する必要があります。

これらの要件をまとめると次のようになります。

・トラフィックに応じた自動スケーリング  
・手動操作なしでのスケーリング  
・ダウンタイムの最小化  
・コスト効率の最適化  

これらの要件を満たすためには、Azure の **Autoscale（自動スケーリング）機能**を利用します。

## 3. 技術の基本概念

Azure では、仮想マシンの自動スケーリングを実現するために **Virtual Machine Scale Sets（VMSS）** と **Autoscale** を組み合わせて使用します。

VM Scale Sets は、同じ構成を持つ仮想マシンのグループを管理するサービスです。これにより、大量の VM をまとめて管理することができます。

Autoscale は、特定のパフォーマンスメトリックに基づいて VM の数を自動的に増減させる機能です。これにより、システムはリアルタイムの負荷に応じてスケールアウト（VM追加）またはスケールイン（VM削減）を行います。

## 4. アーキテクチャまたは設計のポイント

自動スケーリングを実装する場合、重要なのは **スケーリングのトリガーとなるメトリックを定義すること**です。

Azure では、CPU使用率やメモリ使用率などのパフォーマンスメトリックをトリガーとして利用できます。

例えば、次のようなルールを設定することが可能です。

まず、CPU使用率が一定の閾値を超えた場合に VM を追加します。例えば CPU 使用率が 75% を超えた場合にインスタンスを1台追加するよう設定できます。

次に、CPU使用率が低下した場合には VM を削減します。例えば CPU 使用率が 30% 未満の場合にインスタンスを1台削除するよう設定できます。

このようなルールにより、システムは自動的にリソースを最適化します。

## 5. 設計判断（なぜこの構成になるか）

CPUやメモリ使用率を基準にした自動スケーリングは、アプリケーションの実際の負荷に応じてリソースを調整できるため、最も効率的な方法です。

負荷が高くなれば VM を追加し、負荷が低くなれば VM を削減することで、システムは常に適切なパフォーマンスを維持できます。

さらに、この仕組みは Azure Monitor と統合されており、リアルタイムのメトリックに基づいてスケーリングが実行されます。

これにより、アプリケーションの可用性を維持しながらコストを最適化できます。

## 6. 他の選択肢が誤りな理由

### B カスタムスクリプトによる手動スケーリング

カスタムスクリプトを使用すると VM を増減することは可能ですが、手動操作や複雑なスクリプト管理が必要になります。そのため、完全な自動スケーリングには適していません。

### C 予約インスタンス

予約インスタンスは長期利用の割引を提供する仕組みです。コスト削減には役立ちますが、リソースを自動的に増減する機能はありません。

### D Azure Monitor のアラート通知

Azure Monitor は監視やアラート通知を提供しますが、単体ではリソースのスケーリングを実行しません。スケーリングには Autoscale 機能が必要になります。

## 7. 最終回答

A. **CPU とメモリ使用率に基づいて自動スケーリングを設定する**

## 8. まとめ

Azure 仮想マシンの自動スケーリングを実現するためには、VM Scale Sets と Autoscale を利用します。

CPUやメモリ使用率などのメトリックに基づいてスケーリングルールを設定することで、トラフィックの変動に応じて VM を自動的に増減できます。

これにより、アプリケーションの可用性を維持しながら、不要なリソースを削減してコストを最適化できます。

試験では **「トラフィックに応じた自動スケーリング」「手動操作なし」「高可用性」「コスト最適化」** といったキーワードが出てきた場合、Autoscale または VM Scale Sets が正解になるケースが多いです。
