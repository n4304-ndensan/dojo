# Azure 管理アクセスを MFA で保護する方法（Conditional Access）

## 1. 背景（シナリオ）

ある企業では、Azure 環境のセキュリティを強化するために Azure ポータルへの管理アクセスを保護する必要があります。Azure ポータルはクラウドリソースの管理を行う重要なインターフェースであり、ここへのアクセスが侵害されると、環境全体のセキュリティに重大な影響を与える可能性があります。

そのため、Azure ポータルを使用して管理操作を行うユーザーには、多要素認証（MFA）を必須にする必要があります。MFA は、パスワードに加えて追加の認証要素を要求することで、アカウント乗っ取りのリスクを大幅に低減することができます。

しかし、この企業ではすべてのユーザーに MFA を強制したいわけではありません。特に、一般ユーザーが Microsoft 365 などのクラウドアプリケーションにアクセスする場合には、ユーザー体験を損なわないよう MFA を要求しない方針です。

つまり、このシナリオでは **管理アクセスだけを MFA で保護し、一般ユーザーの通常利用には影響を与えないようにする必要があります。**

[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Microsoft Entra ID]]
このような要件を満たす Azure Entra ID（旧 Azure AD）の機能を選択する必要があります。

---

## 2. 要件整理

この問題では、認証ポリシーの適用範囲を正確に制御することが重要になります。

まず、MFA を要求する対象は **Azure ポータルへの管理アクセス**です。これは主に管理者ロールを持つユーザーに関連する操作になります。

次に、一般ユーザーの通常のクラウドアプリケーション利用には影響を与えない必要があります。つまり、MFA をすべてのユーザーに適用する方法は適していません。

さらに、セキュリティ対策は Azure のネイティブな認証制御機能を使用して実装する必要があります。

これらの条件を整理すると次のようになります。

- Azure ポータル管理アクセスに MFA を要求  
- 管理者ユーザーのみ対象  
- 非管理者ユーザーには MFA を強制しない  
- 柔軟な認証ポリシー管理  

このような要件を満たす認証制御機能を利用する必要があります。

---

## 3. 技術の基本概念

### 多要素認証（MFA）

多要素認証（Multi-Factor Authentication）は、ユーザー認証において複数の認証要素を使用するセキュリティ手法です。

通常のログインではユーザー名とパスワードのみを使用しますが、MFA では追加の認証要素が必要になります。

例えば次のような要素があります。

- モバイル認証アプリ  
- SMS コード  
- ハードウェアトークン  
- 生体認証  

MFA はパスワード漏洩やフィッシング攻撃に対する防御として非常に効果的です。

---

### 条件付きアクセス（Conditional Access）

条件付きアクセスは、Azure Entra ID の認証ポリシー機能です。ユーザー、アプリケーション、デバイス、場所などの条件に基づいてアクセス制御を行うことができます。

例えば次のようなポリシーを設定できます。

- 管理者のみ MFA を要求  
- 特定アプリへのアクセス時に MFA を要求  
- 特定の場所からのアクセスを制限  
- 管理対象デバイスのみアクセス許可  

条件付きアクセスを使用すると、セキュリティを強化しながらユーザー体験を最適化することができます。

---

## 4. アーキテクチャまたは設計のポイント

このシナリオでは、条件付きアクセスを使用して **管理者ロールを持つユーザーのみを対象に MFA を要求するポリシー**を作成します。

このポリシーでは次のような条件を設定できます。

まず、対象ユーザーとして Azure 管理者ロールを指定します。これには Global Administrator や Privileged Role Administrator などのロールが含まれます。

次に、アクセス対象のアプリケーションとして Azure 管理ポータルや Azure Resource Manager を指定します。

そして、アクセス制御として MFA を要求する設定を適用します。

このようなポリシーにより、管理操作を行うユーザーには MFA が要求されますが、一般ユーザーには影響がありません。

---

## 5. 設計判断（なぜこの構成になるか）

この問題の重要なポイントは **MFA を管理者のみに限定すること**です。

セキュリティデフォルトを有効にすると、すべてのユーザーに MFA が適用されます。これは今回の要件には適していません。

また、すべてのユーザーに MFA を設定する方法も同様に過剰な制御になります。

条件付きアクセスを使用すれば、管理者ロールを持つユーザーのみを対象に MFA を要求することができます。

そのため、最も適切な方法は **管理者ロールに対する条件付きアクセスポリシーを作成すること**になります。

---

## 6. 他の選択肢が誤りな理由

### セキュリティデフォルト

セキュリティデフォルトはすべてのユーザーに MFA を適用します。そのため、一般ユーザーのクラウドアプリ利用にも MFA が要求されてしまいます。

### すべてのユーザーに MFA を設定

この方法では管理者だけでなくすべてのユーザーが MFA を使用する必要があります。ユーザー体験を制限してしまうため、要件に合いません。

### Azure AD Premium P2 ライセンス購入

ライセンスを購入するだけでは MFA は有効になりません。MFA を実際に強制するにはポリシー設定が必要です。

---

## 7. 最終回答

B. 管理職種に割り当てられたユーザーに対して MFA を必要とする条件付きアクセスポリシーを作成します。
