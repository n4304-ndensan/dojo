# Azure仮想WANにおけるExpressRoute接続導入の初期ステップ

## 1. 背景（シナリオ）
既存のAzure仮想WANに、標準層で設定されているセキュアかつ高性能なExpressRoute接続を追加する必要があります。仮想WANは複数のハブを地域ごとに持ち、ExpressRoute接続が複数の仮想ネットワークを効果的にサポートできるようにすることが目的です。

## 2. 要件整理
- 複数リージョンに分散するハブを持つ仮想WAN
- セキュアで高性能なExpressRoute接続
- 仮想ネットワーク間の効率的な接続サポート
- 初期ステップとして接続の基盤を確立する必要あり

## 3. 技術の基本概念
- **ExpressRoute**: オンプレミスとAzureデータセンター間でプライベート接続を提供するサービス。インターネット経由でなく、安全で低遅延の通信が可能。
- **Azure Virtual WAN**: 複数のリージョンとハブを統合管理し、ブランチやVPN接続、ExpressRouteを簡素化するネットワーキングサービス。
- **リージョン整合性**: ExpressRoute回路は、プライマリハブと同じリージョンに配置することがパフォーマンスと接続性確保の観点で重要。

## 4. アーキテクチャまたは設計のポイント
- プライマリハブと同じリージョンにExpressRoute回路を作成
- 遅延を最小化し、接続を最適化
- 複数仮想ネットワークへの接続を確実にする基盤を整備
- 後続ステップとして、仮想WANルーティングポリシー設定やプレミアム機能の有効化を行う

## 5. 設計判断（なぜこの構成になるか）
- ExpressRoute回路を正しいリージョンに作成することが、最初かつ最も重要なステップ
- この基盤が確立されないと、ルーティングやパフォーマンス最適化、追加の仮想ネットワーク接続の設定が正しく機能しない
- 後続ステップはこの基盤の上に構築される

## 6. 他の選択肢が誤りな理由
- **仮想WANルーティングポリシー設定**: 回路作成前に設定しても機能せず、初期ステップではない
- **ExpressRouteプレミアムアドオン有効化**: 基本回路が存在していないと利用不可
- **仮想WANプレミアム層へのアップグレード**: 高度な接続機能を提供するが、初期ステップとしては回路作成が優先

## 7. 最終回答
**B. プライマリハブと同じリージョンにExpressRoute回路を作成する**

## 8. まとめ
Azure仮想WANにおいて安全かつ高性能なExpressRoute接続を構築する場合、最初のステップはプライマリハブと同じリージョンに回路を作成することです。この設定により、後続のルーティングポリシーやプレミアム機能の適用が効率的かつ効果的に行えるようになります。
