# Azure動画ストリーミング向けストレージ設計（General-purpose v2 + Azure CDN）

## 1. 背景（シナリオ）

グローバルな動画ストリーミングサービスを提供するシステムでは、世界中のユーザーが同時に動画コンテンツへアクセスするため、非常に高い同時接続数（高い同時実行性）を処理できるストレージアーキテクチャが必要になります。

さらに、このようなサービスでは以下のような要件が重要になります。

- 世界中のユーザーに対して低遅延で動画を配信すること
- 高解像度動画などの大容量データを効率よく保存できること
- 大量アクセスに耐えるスケーラブルな構成
- 障害が発生してもサービスを継続できるレジリエンス
- ストレージコストを抑えた費用対効果の高い設計

このような要件を満たすために、Azure Storage アカウントの種類とコンテンツ配信方式を適切に選択する必要があります。

---

## 2. 要件整理

この問題では、動画ストリーミングサービス特有の要件を整理することが重要です。

シナリオから読み取れる主な要件は次の通りです。

まず、パフォーマンスとユーザー体験に関する要件があります。

- 高い同時アクセスに対応できる
- グローバルユーザーへの低遅延配信
- 動画などの大容量ファイルを効率的に保存

次に、システム運用の観点からの要件です。

- 複数リージョンのユーザー対応
- レジリエンス（障害耐性）
- コスト効率の高いストレージ

このような条件から、**大量のコンテンツ配信に適したストレージとCDNの組み合わせ**が必要であると判断できます。

---

## 3. 技術の基本概念

動画配信のようなコンテンツ配信サービスでは、ストレージ単体ではなく **CDN（Content Delivery Network）と組み合わせた設計**が重要になります。

まず、Azure Storage の **General-purpose v2（GPv2）ストレージアカウント**について理解します。

GPv2 ストレージアカウントは Azure Storage の標準的なストレージアカウントであり、次のような特徴があります。

このサービスの特徴を整理すると以下のようになります。

- Blob Storage（オブジェクトストレージ）をサポート
- ファイル、キュー、テーブルなど複数サービスを利用可能
- 大容量データ保存に適している
- コスト効率が高い
- スケーラブルな設計

動画ファイルは一般的に **Block Blob** として保存されます。Block Blob は動画や画像などの大容量ファイル保存に最適です。

次に **Azure CDN** の役割を理解する必要があります。

Azure CDN は、世界中のエッジロケーションにコンテンツをキャッシュし、ユーザーに最も近い場所からデータを配信するサービスです。

これにより以下のメリットが得られます。

- レイテンシ（遅延）の削減
- オリジンストレージの負荷軽減
- 高い同時アクセス処理能力

---

## 4. アーキテクチャまたは設計のポイント

グローバル動画配信サービスでは、次のようなアーキテクチャが一般的に採用されます。

まず、動画コンテンツは Azure Blob Storage に保存されます。  
そのストレージを **CDN のオリジン**として設定します。

この構成のポイントを整理すると次の通りです。

- 動画データを **Azure Blob Storage（GPv2）に保存**
- Azure CDN が **エッジロケーションでキャッシュ**
- ユーザーは **最も近い CDN エッジからコンテンツ取得**
- ストレージへの直接アクセスを減らす

この設計により

- グローバル低遅延配信
- 高同時接続処理
- ストレージ負荷削減

を実現できます。

---

## 5. 設計判断（なぜこの構成になるか）

この問題の最適解が **「Standard General-purpose v2 + Azure CDN」** となる理由は以下の通りです。

まず、動画配信では **IOPSよりも配信性能とキャッシュ**が重要になります。

プレミアムストレージは高性能ですが、動画配信では CDN によって大部分のトラフィックがキャッシュされるため、必ずしも必要ではありません。

また、GPv2 ストレージは次の理由から適しています。

- Blob Storage による動画保存
- 高いスケーラビリティ
- 低コスト
- CDN とのネイティブ統合

さらに CDN を組み合わせることで

- 世界中のエッジノードでキャッシュ
- 低遅延配信
- ストレージ負荷軽減

という利点が得られます。

---

## 6. 他の選択肢が誤りな理由

まず **A. プレミアムブロックブロブ + GRS** について説明します。

プレミアムストレージは高性能ですが、動画ストリーミングでは CDN キャッシュが主に利用されるため、プレミアム性能は過剰になる可能性があります。またコストも高くなります。

次に **C. プレミアムページブロブ + GZRS** です。

ページブロブは主に次の用途で使用されます。

- 仮想マシンのディスク（Azure VM）

動画ファイル保存用途では通常 **Block Blob** を使用するため、この選択肢は適していません。

最後に **D. 標準 Blob + ライフサイクル管理** です。

ライフサイクル管理はデータ階層化やコスト管理には有効ですが、動画配信に重要な **低遅延配信の仕組み（CDN）** が含まれていません。

したがって、このシナリオでは最適解とは言えません。

---

## 7. 最終回答

**B. Azure CDN と統合した標準汎用 v2 ストレージアカウント**

## 8. まとめ

この問題は、クラウドでの **コンテンツ配信アーキテクチャ**を理解しているかを問う問題です。

特に重要なポイントは以下です。

まず、動画ストリーミングでは次の設計が基本になります。

- コンテンツ保存 → **Azure Blob Storage**
- グローバル配信 → **Azure CDN**

またストレージアカウントの選択では次の理解が重要です。

- **General-purpose v2**
  → 最も一般的でコスト効率が高い

- **Premium Blob**
  → 超低レイテンシ用途

- **Page Blob**
  → VMディスク用途

そのため、**グローバル動画配信では「Blob Storage + CDN」の構成がベストプラクティス**となります。
