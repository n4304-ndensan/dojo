# Hub-Spoke アーキテクチャにおけるユーザー定義ルート（UDR）

## 1. 背景（シナリオ）

Azure 上で複数の仮想ネットワークを利用したシステムを設計する場合、一般的に **Hub-Spoke（ハブ・スポーク）トポロジー** が採用されます。この構成では、中央にあるハブネットワークが共通サービスや共有リソースを提供し、複数のスポークネットワークがそのサービスを利用します。

今回のシナリオでは、ハブネットワークに共有リソースが配置されています。スポークネットワークはそれらのリソースへアクセスする必要がありますが、スポーク同士が直接通信できないようにする必要があります。

つまり、ネットワーク構成では **ハブ経由の通信のみ許可し、スポーク間通信は許可しない** という設計が必要になります。

## 2. 要件整理

この問題のネットワーク要件を整理すると、次のようになります。

まず、スポークネットワークはハブネットワーク内のサービスへアクセスできる必要があります。これは共有サービスや共通インフラを利用するためです。

次に、スポークネットワーク同士は直接通信できないようにする必要があります。これはネットワーク分離とセキュリティを維持するためです。

さらに、ネットワーク構成は効率的で安全である必要があります。そのため、トラフィックの経路を明確に制御できる仕組みが必要です。

要件をまとめると次の通りです。

- スポーク → ハブ通信を許可  
- スポーク ↔ スポーク通信を禁止  
- トラフィックルーティングの制御  
- セキュアで効率的なネットワーク  

## 3. 技術の基本概念

Azure Virtual Network では、トラフィックのルーティングを制御するために **ユーザー定義ルート（UDR: User Defined Route）** を使用できます。

Azure では通常、システムルートと呼ばれるデフォルトルーティングが自動的に設定されます。しかし、UDR を使用することで、トラフィックの次ホップをカスタマイズできます。

UDR を使用すると次のようなことが可能になります。

- 特定のトラフィックを特定のネットワークへルーティング  
- トラフィックをネットワーク仮想アプライアンスへ送信  
- 特定の通信経路を強制  

この機能により、Hub-Spoke アーキテクチャの通信経路を制御できます。

## 4. アーキテクチャまたは設計のポイント

Hub-Spoke ネットワークでは、ハブネットワークが中央ルーティングポイントとして機能します。

この設計では、スポークネットワークのサブネットに対して UDR を設定します。UDR の次ホップをハブネットワークに設定することで、スポークからの通信は必ずハブを経由するようになります。

この構成では次のような通信経路が実現されます。

- スポーク → ハブの共有サービスへ通信  
- スポーク → スポークの直接通信は行われない  
- すべての通信がハブ経由になる  

この構造により、ネットワークの分離とトラフィック管理を実現できます。

## 5. 設計判断（なぜこの構成になるか）

Hub-Spoke トポロジーでは、ネットワークトラフィックを中央のハブで管理することが重要です。

UDR を使用することで、スポークネットワークから送信されるトラフィックのルートを明示的に指定できます。これにより、スポークからの通信をハブへ強制的にルーティングできます。

この設計により、次のメリットがあります。

- ネットワークトラフィックの集中管理  
- スポークネットワークの分離  
- セキュリティの向上  
- ルーティングの柔軟な制御  

そのため、Hub-Spoke アーキテクチャでは UDR がよく利用されます。

## 6. 他の選択肢が誤りな理由

VNet Peering で転送トラフィックを許可すると、スポークネットワーク間の通信が可能になる場合があります。これはスポーク間通信を防ぐ要件に適していません。

ゲートウェイトランジットは VPN Gateway や ExpressRoute を共有するための機能であり、スポーク間通信の制御とは関係ありません。

Network Security Group（NSG）はトラフィックの許可・拒否を制御するセキュリティ機能ですが、ルーティング制御を行うものではありません。そのため、通信経路の管理には適していません。

## 7. 最終回答

B. スポークネットワーク内にユーザー定義ルート（UDR）を設定してハブネットワークを指す

## 8. まとめ

Hub-Spoke ネットワークでは、中央のハブがネットワーク通信の中心として機能します。

スポークネットワークからハブへの通信を確実に行い、スポーク同士の直接通信を防ぐためには、トラフィックルーティングを制御する必要があります。

Azure のユーザー定義ルート（UDR）を使用することで、スポークからのトラフィックをハブへルーティングでき、セキュアで効率的なネットワーク構成を実現できます。
