# NSG フローログによるネットワークトラフィック分析

## 1. 背景（シナリオ）

ある組織では、オンプレミス環境と Azure の両方に仮想マシンを配置した **ハイブリッドクラウド環境** を運用しています。

最近、ユーザーから次のような問題が報告されています。

- アプリケーションへの接続が断続的に失敗する
- 特定の時間帯で通信が不安定になる
- Azure VM への接続が時々ブロックされる

この問題を調査するためには、オンプレミスと Azure 間のネットワーク通信を詳細に分析する必要があります。

特に次の点を確認する必要があります。

- セキュリティルールによる通信ブロック
- インバウンドトラフィックのフィルタリング
- アウトバウンドトラフィックのフィルタリング

つまり、**VM レベルでどのトラフィックが許可または拒否されているかを確認できるツール**が必要になります。

---

## 2. 要件整理

この問題では、ネットワークトラフィックの分析に関する要件を整理することが重要です。

まず、診断対象です。

- Azure 仮想マシン
- ハイブリッドネットワーク環境

次に、調査内容です。

- インバウンド通信
- アウトバウンド通信
- セキュリティルールによるブロック

最後に、必要な機能です。

- トラフィックの詳細ログ
- セキュリティルールの影響確認
- トラフィックの流れの可視化

このことから、**Network Security Group（NSG）のトラフィックログを確認できる機能**が必要になります。

---

## 3. 技術の基本概念

Azure 仮想ネットワークでは、VM の通信制御に **Network Security Group（NSG）** が使用されます。

NSG は以下のトラフィックを制御します。

- インバウンド通信
- アウトバウンド通信

NSG では、ルールによって通信を **許可（Allow）または拒否（Deny）** します。

しかし、通信トラブルが発生した場合には、どのルールが通信をブロックしているかを確認する必要があります。

このとき利用できる機能が **NSG Flow Logs（NSG フローログ）** です。

NSG フローログは次の情報を記録します。

- ソース IP
- 宛先 IP
- ポート番号
- プロトコル
- トラフィック方向
- Allow / Deny の結果

これにより、どのトラフィックがブロックされているかを確認できます。

---

## 4. アーキテクチャまたは設計のポイント

NSG フローログは **Azure Network Watcher** の機能として提供されます。

フローログを有効にすると、次のような処理が行われます。

まず、NSG を通過するすべてのトラフィックが監視されます。

次に、通信イベントがログとして保存されます。

このログには次のような情報が含まれます。

- 通信元 IP
- 通信先 IP
- ポート番号
- プロトコル
- NSG ルール結果

これらのログは Azure Storage または Azure Monitor に保存されます。

その後、ログを分析することで次のような問題を特定できます。

- 特定ポートがブロックされている
- 特定 IP が拒否されている
- NSG ルールの誤設定

---

## 5. 設計判断（なぜこの構成になるか）

この問題では、通信トラフィックの **許可 / 拒否の詳細分析** が必要です。

NSG フローログを使用すると、次のメリットがあります。

まず、トラフィック可視化です。

- どの通信が許可されたか
- どの通信が拒否されたか

次に、トラブルシューティングです。

- 接続失敗の原因特定
- NSG ルールの確認

さらに、長期分析です。

- トラフィックパターン分析
- セキュリティ監査

そのため、ネットワーク通信のフィルタリング問題を調査する際に最も適したツールになります。

---

## 6. 他の選択肢が誤りな理由

まず **Network Watcher Connection Troubleshoot** です。

このツールは 2 つのリソース間の接続可否をテストできますが、トラフィックの詳細ログを提供するものではありません。

次に **Azure Firewall Policy** です。

Azure Firewall Policy はファイアウォールの設定管理ツールであり、既存トラフィックの詳細分析には適していません。

最後に **Application Gateway Diagnostics** です。

これは Application Gateway の診断用ツールであり、VM のネットワークトラフィック分析には使用できません。

---

## 7. 最終回答

**C. Azure Network Security Group Flow Logs**

---

## 8. まとめ

Azure ネットワークトラブルシューティングでは、適切なツールを選択することが重要です。

主なツールを整理すると次の通りです。

まず、トラフィック分析です。

- **NSG Flow Logs**
  → トラフィックの許可 / 拒否ログ

次に、接続テストです。

- **Connection Troubleshoot**
  → 接続可否診断

さらに、セキュリティ管理です。

- **Azure Firewall Policy**
  → ファイアウォール管理

このため、**VM レベルのトラフィックフィルタリング問題を診断する場合は NSG Flow Logs を使用するのが最適な方法**となります。
