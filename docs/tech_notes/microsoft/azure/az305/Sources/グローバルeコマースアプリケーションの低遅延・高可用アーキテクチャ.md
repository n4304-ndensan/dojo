# グローバルeコマースアプリケーションの低遅延・高可用アーキテクチャ

## 1. 背景（シナリオ）

ある企業では、世界中のユーザーが利用するグローバルなeコマースアプリケーションをAzure上で構築しています。このアプリケーションは複数のリージョンに展開されており、ユーザーがどの地域からアクセスしても低遅延で利用できる必要があります。

また、eコマースサービスではシステム停止がビジネスに大きな影響を与えるため、リージョン障害が発生してもサービスを継続できる高可用性アーキテクチャが必要です。ユーザーのトラフィックは地理的位置に基づいて最適なリージョンへルーティングされ、障害発生時には自動的に別リージョンへ切り替えられる必要があります。

このような要件を満たすためには、グローバルレベルのトラフィック管理とフェールオーバー機能を組み合わせたアーキテクチャを設計することが重要です。

---

## 2. 要件整理

このシナリオから読み取れる設計要件を整理すると、次のようになります。

まず、ユーザーが世界中に分散しているため、ユーザーの地理的位置に基づいて最適なリージョンへトラフィックをルーティングする必要があります。これにより、ユーザー体験を向上させる低遅延アクセスが実現できます。

次に、高可用性が必要です。特定のリージョンが停止した場合でも、別のリージョンへトラフィックを自動的に切り替えることでサービス停止を防ぐ必要があります。

さらに、リアルタイムのヘルスチェックを行い、利用可能なバックエンドへトラフィックをルーティングする仕組みも必要になります。

これらの要件を整理すると次の通りです。

- グローバルユーザー向けの低遅延アクセス  
- 地理位置ベースのトラフィックルーティング  
- 自動フェールオーバー  
- 高可用性とレジリエンス  

---

## 3. 技術の基本概念

Azureにはグローバルトラフィック管理を行うサービスとして「Azure Front Door」と「Azure Traffic Manager」があります。

まずAzure Front Doorは、グローバルレベルのHTTP/HTTPSロードバランサーです。ユーザーの位置やネットワークパフォーマンスを基に、最も応答が速いバックエンドへトラフィックをルーティングできます。

Azure Front Doorには次のような特徴があります。

- グローバルHTTPロードバランシング  
- パフォーマンスベースルーティング  
- 自動ヘルスチェック  
- Web Application Firewall（WAF）  
- SSL終端  

次にAzure Traffic Managerです。Traffic ManagerはDNSベースのトラフィックルーティングサービスであり、複数リージョンのエンドポイント間でトラフィックを分散します。

Traffic Managerでは次のようなルーティング方式を利用できます。

- パフォーマンスルーティング  
- 地理ベースルーティング  
- 優先順位ルーティング（フェールオーバー）  

この2つのサービスを組み合わせることで、グローバルアプリケーションのトラフィック管理と可用性を強化できます。

---

## 4. アーキテクチャまたは設計のポイント

グローバルアプリケーションでは、トラフィック管理を多層構造で設計することが重要です。

まず、ユーザーからのトラフィックはAzure Front Doorが受け取ります。Front Doorはユーザーの場所やパフォーマンスを考慮して最適なリージョンへリクエストを送信します。

次に、Azure Traffic ManagerがDNSレベルでトラフィックを管理し、リージョン障害が発生した場合には別のリージョンへフェールオーバーします。

この構成では次のような処理が行われます。

1. ユーザーがアプリケーションのURLへアクセス  
2. Azure Traffic Managerが最適なリージョンを選択  
3. Azure Front Doorがバックエンドの健全性を確認  
4. 最適なアプリケーションインスタンスへルーティング  

この仕組みにより、ユーザーは常に最も近く、かつ利用可能なリージョンへ接続されます。

---

## 5. 設計判断（なぜこの構成になるか）

この問題では「低遅延アクセス」と「自動フェールオーバー」の両方が重要な要件です。

Azure Front Doorは、グローバルレベルでの高速なトラフィックルーティングとリアルタイムヘルスチェックを提供します。これにより、ユーザーは最も応答速度の高いリージョンへ接続されます。

一方、Azure Traffic ManagerはDNSベースで複数リージョン間のトラフィック管理を行い、障害時には別リージョンへ自動的にフェールオーバーします。

この2つを組み合わせることで、次のメリットが得られます。

- グローバルなトラフィック最適化  
- 地理的位置に基づくルーティング  
- 自動フェールオーバー  
- 高可用性アーキテクチャ  

そのため、グローバルeコマースアプリケーションにはこの組み合わせが最適です。

---

## 6. 他の選択肢が誤りな理由

まずAzure Load Balancerを使用する構成です。Azure Load Balancerはリージョン内でトラフィックを分散するサービスであり、グローバルレベルでのトラフィックルーティングやフェールオーバーには対応していません。

次にAzure API ManagementとAzure CDNの組み合わせです。API ManagementはAPI管理サービスであり、トラフィックルーティング機能は提供していません。Azure CDNは静的コンテンツ配信の高速化には有効ですが、アプリケーションレベルのトラフィック管理には適していません。

最後にAzure Application GatewayとAzure Site Recoveryの組み合わせです。Application Gatewayは主にリージョン内のレイヤー7ロードバランサーであり、グローバルルーティング機能はありません。またSite Recoveryは災害復旧用のサービスであり、リアルタイムのユーザートラフィックルーティングには使用されません。

---

## 7. 最終回答

A.  
Azure Front Door と Azure Traffic Manager の組み合わせ

## 8. まとめ

グローバルアプリケーションでは、ユーザーの地理的位置に応じたトラフィックルーティングとリージョン障害時のフェールオーバーが重要です。

Azure Front Doorは次の役割を担います。

- グローバルHTTPロードバランシング  
- パフォーマンスベースルーティング  
- ヘルスモニタリング  

Azure Traffic Managerは次の機能を提供します。

- DNSベースのトラフィック管理  
- 地理位置ルーティング  
- 自動フェールオーバー  

この2つを組み合わせることで、**低遅延・高可用性・高耐障害性のグローバルアーキテクチャ**を実現できます。
