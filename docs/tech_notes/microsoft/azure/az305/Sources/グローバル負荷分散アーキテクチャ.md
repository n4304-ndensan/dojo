# グローバル負荷分散アーキテクチャ

[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Azure Front Door]]
（Azure Front Door + NSG Service Tag）

## 1 概要

複数リージョンに配置されたアプリケーションをユーザーに提供する場合、**グローバル負荷分散**と**セキュリティ制御**が重要になる。

この問題の構成では次の要件がある。

[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Azure Virtual Machines]]
- 異なるリージョンのVM間で負荷分散
    
- Azure Front Door を使用
    
- VMは **Front Door経由のトラフィックのみ受信**
    

この要件を満たす典型構成は次の通り。

```text
Client
   │
   ▼
Azure Front Door
   │
   ├ VM (Region 1)
   └ VM (Region 2)
```

さらにセキュリティとして

**Network Security Group + Service Tag**

を利用し、Front Door以外からの通信を拒否する。

---

# 2 背景

クラウドアプリケーションでは次の課題がある。

- 世界中のユーザーからアクセス
    
- リージョン障害対策
    
- 高可用性
    
- セキュリティ
    

従来の単一リージョン構成

```text
Users
  │
  ▼
Load Balancer
  │
  ▼
VM
```

問題

- リージョン障害
    
- 高レイテンシ
    
- 可用性不足
    

Azureではこれを解決するために

**グローバルロードバランサ**

が提供されている。

# 3 Azure Front Door の仕組み

Azure Front Doorは **Layer7グローバルロードバランサ**である。

特徴

- HTTP/HTTPSトラフィック
    
- Anycastネットワーク
    
- CDN統合
    
- WAF
    

基本構造

```text
Users
   │
   ▼
Azure Edge Network
   │
   ▼
Azure Front Door
   │
   ├ Backend (Region1)
   └ Backend (Region2)
```

処理フロー

1 ユーザーがFront Doorにアクセス  
2 最も近いエッジロケーションへ接続  
3 最適なバックエンドへルーティング

---

# 4 Service Tag + NSG の仕組み

今回の重要ポイントは

**VMがFront Doorからの通信のみ受信**

という要件である。

これを実現する方法

**NSG Service Tag**

Service TagはAzureサービスのIP範囲を抽象化したもの。

例

|Service Tag|意味|
|---|---|
|AzureFrontDoor.Backend|Front Door|
|Storage|Storage|
[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Azure Load Balancer]]
|AzureLoadBalancer|Load Balancer|

NSGで次のルールを作成する。

```text
Source: AzureFrontDoor.Backend
Destination: VM
Port: 80 / 443
Allow
```

それ以外は拒否する。

# 6 サービス比較

Azureロードバランサの違い

|サービス|層|スコープ|
|---|---|---|
|Front Door|L7|グローバル|
|Traffic Manager|DNS|グローバル|
|Application Gateway|L7|リージョン|
|Load Balancer|L4|リージョン|

今回の要件

- HTTPアプリ
    
- 複数リージョン
    

→ **Front Door**

# 7 アーキテクチャ

## 基本構成

```text
Users
   │
   ▼
Azure Front Door
   │
   ├ VM (East US)
   └ VM (West Europe)
```

---

## セキュア構成

```text
Users
   │
   ▼
Azure Front Door
   │
   ▼
NSG (Service Tag)
   │
   ├ VM1
   └ VM2
```

NSGルール

```text
Allow
Source: AzureFrontDoor.Backend

Deny
Source: Internet
```

この問題の要点

- **Azure Front Door**  
    → グローバルロードバランサ
    
- **Network Security Group + Service Tag**  
    → Front Doorからの通信のみ許可
    

構成

```text
User
  │
  ▼
Azure Front Door
  │
  ▼
NSG (AzureFrontDoor.Backend)
  │
  ├ VM1
  └ VM2
```

これにより

- マルチリージョン負荷分散
    
- セキュアバックエンド
    

を実現できる。
