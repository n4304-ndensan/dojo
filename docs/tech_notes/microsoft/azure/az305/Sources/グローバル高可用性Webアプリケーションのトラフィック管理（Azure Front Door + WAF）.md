# グローバル高可用性Webアプリケーションのトラフィック管理（Azure Front Door + WAF）

## 1. 背景（シナリオ）

ある組織では、Azure 上で **複数リージョンに展開された Web アプリケーション**を運用しています。このアプリケーションは世界中のユーザーが利用するため、高い可用性とパフォーマンスが求められています。

そのため、単一リージョンに依存しない **グローバル負荷分散アーキテクチャ**を設計する必要があります。さらに、トラフィックの急増や悪意のあるアクセスからアプリケーションを保護しながら、リージョン障害が発生してもサービスが継続する仕組みが必要です。

このシナリオでは、Azure のどのサービスを利用すれば、グローバル負荷分散、レート制限、そして自動フェールオーバーを実現できるかが問われています。

## 2. 要件整理

問題文から読み取れる要件を整理すると、次のような機能が必要であることが分かります。

まず、ユーザーの位置やネットワーク遅延に基づいてトラフィックを最適なリージョンにルーティングする **グローバルロードバランシング**が必要です。

次に、トラフィックの急増や攻撃を防ぐための **レート制限（Rate Limiting）** 機能が求められています。

さらに、リージョン障害が発生した場合には、ユーザーのトラフィックを別リージョンへ自動的に切り替える **自動フェールオーバー**が必要です。

これらの要件をまとめると、必要な機能は次の通りです。

- グローバル HTTP/HTTPS 負荷分散  
- 遅延・地理的位置ベースのルーティング  
- レート制限などのセキュリティ機能  
- 自動フェールオーバー  
- 高可用性のマルチリージョン対応  

## 3. 技術の基本概念

Azure Front Door は、Azure の **グローバルアプリケーションロードバランサー**です。Microsoft のグローバルエッジネットワークを利用し、ユーザーのリクエストを最適なバックエンドへルーティングします。

Azure Front Door の主な特徴には次のようなものがあります。

- グローバル HTTP/HTTPS ロードバランシング  
- 遅延ベースルーティング  
- 地理的位置ルーティング  
- 自動フェールオーバー  
- エッジキャッシュ  
- Web Application Firewall（WAF）との統合  

特に **WAF（Web Application Firewall）** を組み合わせることで、セキュリティルールやレート制限などの機能を利用できます。

## 4. アーキテクチャまたは設計のポイント

グローバルアプリケーションを設計する場合、ユーザーのリクエストはまず Azure Front Door に到達します。Front Door はエッジネットワークでリクエストを受け取り、最適なバックエンドリージョンへ転送します。

このアーキテクチャでは次のような流れになります。

- ユーザーは最寄りの Azure エッジロケーションに接続する  
- Azure Front Door がリクエストを受信する  
- WAF がトラフィックを検査し、レート制限などを適用する  
- 健全なリージョンのバックエンドへリクエストを転送する  
- リージョン障害が発生した場合、自動的に別リージョンへフェールオーバーする  

このような構成により、グローバルな高可用性とセキュリティを同時に実現できます。

## 5. 設計判断（なぜこの構成になるか）

Azure Front Door は、グローバル Web アプリケーションの入口として設計されています。そのため、遅延や地理的位置に基づいたインテリジェントなルーティングが可能です。

また、Front Door はバックエンドのヘルスチェックを継続的に実行します。もし特定リージョンのサービスが利用できなくなった場合、トラフィックは自動的に別リージョンへ転送されます。これにより高可用性が確保されます。

さらに、WAF を統合することで、レート制限やセキュリティルールを適用できます。これによりトラフィックの急増や悪用からアプリケーションを保護できます。

これらの機能により、Azure Front Door + WAF はグローバル Web アプリケーションに最適なソリューションとなります。

## 6. 他の選択肢が誤りな理由

Azure のロードバランシングサービスにはそれぞれ異なる役割があります。

Azure Traffic Manager は DNS ベースのロードバランサーであり、リージョン間のルーティングは可能ですが、レート制限や WAF 機能を提供しません。

Azure Load Balancer はレイヤー4のロードバランサーであり、主に同一リージョン内の負荷分散に使用されます。グローバルルーティングや WAF 機能は提供されません。

Azure Application Gateway はレイヤー7ロードバランサーであり、WAF をサポートしていますが、基本的にはリージョンレベルのトラフィック管理に使用されます。グローバルエッジネットワークを利用した負荷分散は提供されません。

## 7. 最終回答

C. Web Application Firewall（WAF）付き Azure Front Door

## 8. まとめ

Azure Front Door は、グローバル Web アプリケーションのためのアプリケーションレベルロードバランサーです。遅延ベースルーティング、地理的位置ルーティング、自動フェールオーバーなどの機能により、高可用性とパフォーマンスを実現します。

さらに、WAF を組み合わせることでレート制限やセキュリティルールを適用でき、トラフィックの急増や攻撃からアプリケーションを保護できます。そのため、マルチリージョンの高可用性 Web アプリケーションを構築する場合には、Azure Front Door + WAF が最適なソリューションとなります。
