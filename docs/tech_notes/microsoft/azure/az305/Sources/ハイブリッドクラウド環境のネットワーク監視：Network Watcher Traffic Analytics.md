# ハイブリッドクラウド環境のネットワーク監視：Network Watcher Traffic Analytics

## 1. 背景（シナリオ）

企業では、Azureクラウドとオンプレミス環境の両方に仮想マシンを配置したハイブリッドクラウドインフラストラクチャを運用しています。この環境では、クラウドとオンプレミス間で大量のネットワークトラフィックが発生します。

このような環境では、ネットワークのセキュリティとパフォーマンスを維持することが重要になります。特に、不正アクセスの試みを検出したり、通信の異常を特定したりするためには、ネットワークトラフィックを継続的に監視する必要があります。

また、トラフィックのパターンを分析することで、ネットワークの利用状況や潜在的なボトルネックを把握し、パフォーマンス改善にも役立てることができます。

そのため、このシナリオではネットワークトラフィックの監視と分析を包括的に行うことができるAzureの機能を選択する必要があります。

---

## 2. 要件整理

この問題のシナリオから読み取れる要件を整理すると次のようになります。

まず、Azureとオンプレミスを含むハイブリッドクラウド環境のネットワークトラフィックを監視する必要があります。

次に、不正アクセスの試みを検出する必要があります。これはセキュリティログや通信ログの分析が必要であることを意味します。

さらに、ネットワークトラフィックのパターンを分析する必要があります。これにより、通信量や通信経路の可視化が可能になります。

最後に、これらの機能を統合した監視および分析ソリューションが必要です。

要件をまとめると次の通りです。

- ハイブリッドクラウドのネットワーク監視  
- 不正アクセスの検出  
- トラフィックパターンの分析  
- ネットワークログの収集と分析  

---

## 3. 技術の基本概念

Azure Network Watcherは、Azureネットワークの監視および診断を行うためのサービスです。

このサービスは、仮想ネットワークやネットワーク接続の状態を監視し、ネットワークトラフィックの詳細な分析を提供します。

Network Watcherには複数の機能がありますが、特に重要なのが「Traffic Analytics」です。

Traffic Analyticsは、Network Security Group（NSG）のフローログを分析して、ネットワーク通信の状況を可視化する機能です。

NSGフローログには次のような情報が記録されます。

- 通信元IPアドレス  
- 通信先IPアドレス  
- ポート番号  
- プロトコル  
- 許可または拒否されたトラフィック  

これらのログを分析することで、ネットワーク通信の全体像を把握できます。

---

## 4. アーキテクチャまたは設計のポイント

Network Watcher Traffic Analyticsは、NSGフローログを基にネットワークトラフィックを分析します。

この構成では、まずNSGフローログを有効にします。これにより、NSGを通過するすべてのトラフィックがログとして記録されます。

次に、Traffic Analyticsがこれらのログを解析します。

解析結果はAzure Portal上で可視化され、次のような情報を確認できます。

- 通信しているリソース  
- 通信量  
- トラフィックの方向  
- 拒否された通信  

このような情報を基に、ネットワークの異常や不正アクセスを検出することができます。

また、トラフィックの利用状況を分析することで、ネットワーク設計の改善にも役立ちます。

---

## 5. 設計判断（なぜこの構成になるか）

この問題では「ネットワークトラフィックの監視と分析」が重要なポイントです。

Azure Network WatcherのTraffic Analyticsは、NSGフローログを利用してネットワーク通信を詳細に分析する機能を提供します。

この組み合わせにより、次のことが可能になります。

- ネットワークトラフィックの可視化  
- 不正アクセスの検出  
- トラフィックパターン分析  
- セキュリティ監視  

さらに、Azureとオンプレミスを含むハイブリッド環境でも利用できるため、統合的なネットワーク監視が可能になります。

そのため、このシナリオではNetwork Watcher Traffic AnalyticsとNSGフローログの組み合わせが最適です。

---

## 6. 他の選択肢が誤りな理由

Azure MonitorとApplication Insightsは、主にアプリケーションのパフォーマンス監視を目的としたサービスです。ネットワークトラフィックの詳細な分析には適していません。

Azure SentinelはSIEM（セキュリティ情報イベント管理）サービスであり、セキュリティログ分析には強力ですが、ネットワークトラフィックの詳細なフロー分析を目的としたサービスではありません。

Azure Security CenterとAzure Policyは、セキュリティ管理とコンプライアンスのためのサービスであり、ネットワークトラフィックの分析機能は提供していません。

---

## 7. 最終回答

A.  
Azure Network Watcher Traffic Analytics と  
Azure Network Security Group（NSG）フローログ

---

## 8. まとめ

ハイブリッドクラウド環境では、ネットワークトラフィックの可視化と分析が重要になります。

Azure Network Watcher Traffic Analyticsは、NSGフローログを分析することで次の機能を提供します。

- ネットワーク通信の可視化  
- 不正アクセス検出  
- トラフィックパターン分析  
- ネットワークパフォーマンス監視  

そのため、ハイブリッドクラウド環境のネットワーク監視には **Network Watcher Traffic Analytics + NSG Flow Logs** が最適な組み合わせです。
