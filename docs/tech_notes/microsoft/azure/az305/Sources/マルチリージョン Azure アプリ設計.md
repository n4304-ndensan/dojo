# マルチリージョン Azure アプリ設計

[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Azure Virtual Machines]]
[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Azure Virtual Network (VNet)]]
## CPU クレジットを利用してコスト最適化する VM と VNET 構成

---

# 1 背景（シナリオ）

企業は可用性と災害対策（DR）を目的として、クラウドアプリケーションを **複数リージョンに展開する構成**を採用することが多い。

今回のシナリオでは、アプリケーションは **2つの Azure リージョンに展開**されている。

このアプリには次の利用パターンがある。

### 利用パターン

|時間帯|負荷|
|---|---|
|8:00〜9:00|ピーク|
|9:00〜16:00|低負荷|
|16:00〜17:00|ピーク|
|17:00〜翌朝|低負荷|

つまり

```text
短時間のピーク
+
長時間の低負荷
```

という **バースト型ワークロード**である。

---

# 2 要件

このシステムでは次の条件を満たす必要がある。

### 要件

1. **マルチリージョン構成**
    
2. **CPU クレジットを利用してコスト削減**
    
3. **ピーク時間の処理能力を確保**
    

この要件から

- **仮想ネットワーク数**
    
- **仮想マシンシリーズ**
    

を適切に選択する必要がある。

---

# 3 Azure VNET の基本特性

Azure Virtual Network（VNET）は

```text
リージョン単位
```

で作成される。

つまり

```text
1 VNET
↓
1 Azure Region
```

となる。

そのため

```text
2リージョン
↓
2 VNET
```

が必要になる。

---

# 4 マルチリージョンネットワーク構成

今回の構成は次のようになる。

```text
Region1
 └ VNET1
      └ VM

Region2
 └ VNET2
      └ VM
```

2つの VNET は通常

```text
VNET Peering
```

によって接続される。

---

# 5 VNET Peering

VNET Peering を使用すると

```text
VNET1
  │
  │ Azure backbone
  │
VNET2
```

という形で **低遅延通信**が可能になる。

特徴

|特徴|説明|
|---|---|
|低遅延|Azure バックボーン|
|高帯域|高スループット|
|セキュア|パブリックインターネット不要|

---

# 6 CPU クレジットとは

CPU クレジットは

```text
VMがアイドル時
↓
CPUクレジット蓄積
```

し

```text
ピーク時
↓
CPUクレジット消費
```

する仕組みである。

これは **バースト可能 VM**で利用される。

---

# 7 バースト可能 VM（Bシリーズ）

Azure Bシリーズ VM は

```text
Burstable VM
```

と呼ばれる。

特徴

|特徴|説明|
|---|---|
|低コスト|ベースCPU低い|
|バースト性能|CPUクレジット使用|
|低負荷ワークロード|最適|

---

# 8 CPU クレジットの動作

Bシリーズ VM は次のように動作する。

```text
低負荷時間
↓
CPUクレジット蓄積
↓
ピーク時間
↓
CPUクレジット使用
```

今回のワークロード

|時間|状態|
|---|---|
|大部分|低負荷|
|短時間|ピーク|

このパターンは

```text
Bシリーズに最適
```

である。

---

# 9 Dシリーズ VM

Dシリーズ VM は

```text
汎用VM
```

である。

特徴

|特徴|説明|
|---|---|
|CPU固定|クレジットなし|
|安定性能|高い|
|コスト|Bシリーズより高い|

つまり

```text
ピークが少ない
```

環境では **コスト効率が悪い**。

---

# 10 Aシリーズ VM

Aシリーズ VM は

```text
旧世代VM
```

である。

問題

- CPU クレジットなし
    
- パフォーマンス低い
    
- コスト効率悪い
    

現在はほとんど推奨されない。

---

# 11 VNET 数の検討

選択肢

|選択肢|問題|
|---|---|
|1 VNET|リージョン跨ぎ不可|
|2 VNET|正しい|
|4 VNET|過剰設計|

Azure VNET は

```text
リージョン固定
```

なので

```text
2リージョン
↓
2 VNET
```

となる。

---

# 12 最適アーキテクチャ

```text
Region A
 └ VNET-A
      └ B-series VM

Region B
 └ VNET-B
      └ B-series VM

VNET Peering
```

特徴

- リージョン分散
    
- 低遅延通信
    
- CPU クレジット活用
    

---

# 13 他の選択肢が不適切な理由

### B

1 VNET + Dシリーズ

問題

```text
1 VNET
↓
2リージョン不可
```

さらに

```text
Dシリーズ
↓
CPUクレジットなし
```

---

### C

2 VNET + Aシリーズ

ネットワークは正しいが

```text
Aシリーズ
↓
バースト不可
```

---

### D

4 VNET + Bシリーズ

VM は適切だが

```text
VNET過剰
↓
管理複雑化
```

---

# 14 最終回答

正解

```text
A

VNET: 2
VMシリーズ: Bシリーズ
```

---

# 15 まとめ

今回の問題の重要ポイントは

```text
ワークロード特性
```

を理解することである。

ワークロード

```text
低負荷時間 多い
+
短時間ピーク
```

この場合

```text
Bシリーズ
```

が最適である。

ネットワーク構成は

```text
リージョン数
↓
VNET数
```

となるため

```text
2リージョン
↓
2 VNET
```

が必要になる。
