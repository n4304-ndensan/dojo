# ミッションクリティカルなSQL Serverワークロードのための高可用性データ基盤

## 1. 背景（シナリオ）

ある組織では、SQL Server データベースに大きく依存するミッションクリティカルなアプリケーションを運用しています。現在、そのデータ基盤をクラウドへ移行し、より現代的で可用性の高いアーキテクチャへと刷新する計画が進められています。

このアプリケーションはビジネスにとって非常に重要であるため、システム停止は極力避ける必要があります。また、将来的な負荷増加にも対応できるように、スケーラブルなデータベース基盤が求められています。

さらに、リージョン障害などの大規模障害に備えて、複数のAzureリージョンにまたがる災害復旧（Disaster Recovery）機能も必要です。そのうえで、運用負荷を減らすため、可能な限り自動化された管理サービスを利用することが求められています。

## 2. 要件整理

このシナリオから読み取れる要件を整理すると、次のようになります。

- SQL Server ベースのワークロードをサポートする  
- ミッションクリティカル用途の高可用性が必要  
- メンテナンス時のダウンタイムを最小化する  
- 自動スケーリングまたは柔軟なスケーリングが可能  
- 複数リージョンにまたがる災害復旧機能を提供する  
- 運用管理の手動作業を最小化する  

これらの要件から、Azureのフルマネージド型データベースサービスの中でも、SQL Serverとの互換性が高く、高可用性と災害復旧機能を備えたサービスを選択する必要があります。

## 3. 技術の基本概念

AzureにはSQL Serverワークロードを実行するための複数のデータベースサービスが提供されています。これらのサービスは用途や互換性レベル、スケーラビリティなどによって使い分ける必要があります。

主な選択肢として次のようなサービスがあります。

- Azure SQL Database  
- Azure SQL Managed Instance  
- Azure SQL Database Hyperscale  
- Azure SQL Database Elastic Pool  

これらの中で、Azure SQL Managed Instance は特にオンプレミスSQL Serverとの互換性を重視したサービスとして設計されています。

Azure SQL Managed Instanceの主な特徴は次の通りです。

- SQL Serverと高い互換性を持つ  
- 自動パッチ適用および自動バックアップ  
- 高可用性（99.99% SLA）  
- 自動フェイルオーバー  
- ジオレプリケーションによる災害復旧  
- マネージドサービスによる運用負荷の軽減  

このような特徴により、オンプレミスSQL Server環境をクラウドへ移行する際の最も一般的なターゲットサービスの一つとなっています。

## 4. アーキテクチャまたは設計のポイント

ミッションクリティカルなデータベース基盤をAzureで設計する場合には、いくつかの重要な設計ポイントを考慮する必要があります。

まず重要になるのが高可用性です。Azure SQL Managed Instanceでは、サービス内部で冗長構成が自動的に構築され、障害が発生した場合には自動フェイルオーバーが行われます。

次に重要なのが災害復旧です。リージョンレベルの障害に備えるために、ジオレプリケーションやフェイルオーバーグループを利用して別リージョンにデータを複製することが可能です。

また、マネージドサービスであるため、以下のような管理作業はAzureが自動的に処理します。

- OSパッチ適用  
- データベースパッチ適用  
- 自動バックアップ  
- 自動障害回復  

このような仕組みにより、運用チームはインフラ管理ではなくアプリケーション改善に集中することができます。

## 5. 設計判断（なぜこの構成になるか）

この問題では「SQL Serverベースのミッションクリティカルアプリケーション」という条件が重要です。

Azure SQL Managed Instanceは、SQL Serverの機能互換性が高く、既存のSQL Serverアプリケーションをほぼ変更せずに移行できるという大きな利点があります。

さらに次の要件を満たしています。

- 高可用性（自動フェイルオーバー）
- 自動メンテナンス
- 高いスケーラビリティ
- ジオレプリケーションによる災害復旧
- フルマネージドサービス

これらの要件を総合的に満たすサービスとして、Azure SQL Managed Instanceが最も適切な選択になります。

## 6. 他の選択肢が誤りな理由

この問題では他にもAzure SQL関連のサービスが選択肢として提示されていますが、それぞれ用途が異なります。

B. アクティブジオレプリケーション  

これはAzure SQL Databaseで利用される機能であり、単体のデータベースサービスではありません。災害復旧機能の一部ではありますが、データベースプラットフォーム自体を提供するものではないため、この問題の回答には適しません。

C. Azure SQL Database Hyperscale  

Hyperscaleは非常に大規模なデータベース（数十TB規模）を扱うためのサービスです。しかし、SQL Serverとの完全な互換性という観点ではManaged Instanceほど適していない場合があります。

D. Azure SQL Database Elastic Pool  

Elastic Poolは、複数の小規模データベースがリソースを共有するシナリオに適しています。ミッションクリティカルな単一アプリケーションのための高可用性データベースとしては適切ではありません。

## 7. 最終回答

A. Azure SQL Managed Instance

## 8. まとめ

SQL ServerベースのミッションクリティカルアプリケーションをAzureへ移行する場合、高可用性、スケーラビリティ、災害復旧、そして運用管理の自動化が重要な要件となります。

Azure SQL Managed Instanceは、SQL Serverとの高い互換性を維持しながら、フルマネージド型の高可用性データベース環境を提供します。そのため、既存のSQL Serverワークロードをクラウドへ移行する際の最適な選択肢となります。
