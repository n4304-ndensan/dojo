# 外部ユーザーアクセスの定期監査（Azure AD Access Reviews）

## 1. 背景（シナリオ）

多くの企業では、Azure Entra ID（旧 Azure AD）を使用して外部ユーザーとコラボレーションを行っています。例えば、パートナー企業、ベンダー、コンサルタント、委託開発者などが Azure テナントにゲストユーザーとして招待されることがあります。

このような外部ユーザーは、SharePoint、Teams、SaaS アプリ、Azure リソースなどにアクセスできる場合があります。しかし、外部ユーザーのアクセスは長期間放置されるとセキュリティリスクになります。

例えば次のようなケースがあります。

- プロジェクト終了後もアカウントが残る  
- ベンダー契約終了後もアクセスが残る  
- 監査時に不要なアカウントが発見される  

そのため、多くの組織では **定期的に外部ユーザーアクセスを確認するガバナンスポリシー**を導入しています。

今回のシナリオでは、企業が次のようなポリシーを導入しています。

外部ユーザーアクセスは **60日ごとにレビューされる必要があります。**

また、レビューの結果、アクセスが不要と判断されたユーザーやレビューに応答しなかったユーザーのアクセスは **自動的に削除される必要があります。**

この要件を満たす Azure Entra ID の機能を選択する必要があります。

---

## 2. 要件整理

この問題では、アクセスガバナンス機能が必要になります。

まず、外部ユーザーアクセスを **定期的にレビューする仕組み**が必要です。

次に、レビューは **60日ごとに自動実行**される必要があります。

さらに、不要なアクセスは **自動的に削除**できる必要があります。

[[docs/tech_notes/microsoft/azure/az305/Azure用語集.md#Identity Governance]]
このようなアクセスライフサイクル管理は、一般的に **Identity Governance** 機能によって実現されます。

今回の要件を整理すると次の通りです。

- 外部ユーザーのアクセスを対象  
- 定期レビュー（60日）  
- 自動化されたレビューサイクル  
- 不要なアクセスの自動削除  
- 監査可能なガバナンス  

このような要件を満たす Azure Entra ID の機能が **Access Reviews** です。

---

## 3. Access Reviews の概要

Azure Entra ID の Access Reviews は、ユーザーアクセスの定期レビューを実施する機能です。

この機能を使用すると、管理者は次のようなレビューキャンペーンを作成できます。

- グループメンバーシップ  
- アプリケーションアクセス  
- Azure AD ロール  
- 外部ユーザーアクセス  

Access Reviews は **定期的なレビューサイクル**を設定できます。

例えば次のような設定が可能です。

- 毎月  
- 60日ごと  
- 四半期  
- 年次  

レビューはリソース所有者や管理者が実行できます。

---

## 4. 自動アクセス削除機能

Access Reviews の重要な機能の一つが **自動アクセス削除**です。

レビュー終了後に、次のような自動処理を設定できます。

例えば、レビューで拒否されたユーザーは自動的にアクセスが削除されます。

また、レビューに応答しなかったユーザーについても、ポリシーに基づいてアクセスを削除できます。

さらに、ゲストユーザーの場合は **テナントから完全に削除する設定**も可能です。

この機能により、外部ユーザーのアクセスライフサイクルを自動管理できます。

---

## 5. 外部ユーザー管理（B2B）

Azure Entra ID の B2B コラボレーションでは、外部ユーザーはゲストユーザーとして管理されます。

これらのユーザーは次のリソースへアクセスできます。

- Microsoft Teams  
- SharePoint Online  
- SaaS アプリ  
- Azure リソース  

Access Reviews を使用すると、これらの外部ユーザーのアクセス権を定期的に確認できます。

これにより、組織は次のようなメリットを得られます。

- 不要なアカウントの削除  
- セキュリティリスクの削減  
- コンプライアンス対応  
- 監査証跡の記録  

---

## 6. なぜ Access Reviews が最適なのか

この問題では **60日ごとのアクセスレビュー**が必要です。

さらに、不要なユーザーのアクセスは **自動削除**する必要があります。

Azure Entra ID Access Reviews は、この両方の要件を直接サポートしています。

まず、定期レビューを自動的に実行できます。

次に、レビュー結果に基づいてアクセスを自動削除できます。

そのため、このシナリオでは Access Reviews が最適な機能になります。

---

## 7. 他の選択肢が適さない理由

### Privileged Identity Management（PIM）

PIM は特権ロールの管理機能です。

主に管理者ロールの Just-In-Time アクセスを管理するための機能であり、一般ユーザーや外部ユーザーの定期アクセスレビューには使用されません。

---

### Identity Protection

Identity Protection は、ユーザーサインインのリスクを検出するセキュリティ機能です。

例えば、異常なサインインや侵害されたアカウントを検出します。

アクセスレビュー機能ではありません。

---

### Dynamic Groups

Dynamic Groups はユーザー属性に基づいてグループメンバーシップを自動管理する機能です。

しかし、ユーザーアクセスを定期的にレビューする機能はありません。

---

## 8. 最終回答

A. Azure AD Access Reviews
