# 複数リージョンの Web アプリケーションにおけるグローバル負荷分散（Azure Front Door）

## 1. 背景（シナリオ）

ある企業では、Web アプリケーションを Azure 上で運用しています。ユーザーは世界中からアクセスするため、アプリケーションを複数リージョンに展開し、可用性とパフォーマンスを向上させる必要があります。特に、リージョン障害が発生した場合でもサービスが停止しないよう、高可用性を確保することが重要です。

また、ユーザーが地理的に離れた場所からアクセスする場合、ネットワーク遅延が発生する可能性があります。そのため、ユーザーを最も近いリージョンのアプリケーションへ誘導し、応答時間を短縮する仕組みも必要です。

このようなシナリオでは、複数リージョン間でトラフィックを分散させる「グローバルロードバランシング」が重要になります。Azure には複数のロードバランシングサービスが存在するため、それぞれの役割を理解し、最適なサービスを選択する必要があります。

## 2. 要件整理

問題文から読み取れる要件を整理すると、このシナリオでは次の条件を満たすサービスが必要になります。

まず、Web アプリケーションは 2 つのリージョンに展開される必要があります。つまり、リージョン単位でアプリケーションの冗長化が行われます。

次に、ユーザーの可用性を確保するために、リージョン障害が発生してもトラフィックを別リージョンへ自動的にルーティングできる仕組みが必要です。

さらに、ユーザー体験を向上させるため、トラフィックを最適なリージョンへ誘導し、ネットワーク遅延を最小化する必要があります。

これらの要件をまとめると、以下のようになります。

- Web アプリケーションは複数リージョンに展開される  
- グローバルレベルでトラフィックを分散する必要がある  
- 高可用性を確保する必要がある  
- ユーザーの遅延を最小化する必要がある  
- Web アプリケーション向けのロードバランシングが必要  

この条件を満たす Azure サービスを選択することが重要です。

## 3. 技術の基本概念

Azure では、ロードバランシングサービスは大きく「リージョナル負荷分散」と「グローバル負荷分散」に分かれます。

リージョナルロードバランサーは、単一リージョン内でトラフィックを分散するサービスです。例えば Azure Load Balancer や Azure Application Gateway がこれに該当します。

一方で、グローバルロードバランサーは、複数リージョン間でトラフィックを分散するサービスです。代表的なものとして Azure Traffic Manager と Azure Front Door があります。

Azure Front Door は、Web アプリケーション向けのグローバルエントリーポイントを提供するサービスです。Microsoft のグローバルエッジネットワークを利用してユーザーを最適なバックエンドへルーティングします。また、ヘルスチェック機能により、障害が発生したリージョンを自動的に除外し、別リージョンへトラフィックを転送することができます。

## 4. アーキテクチャまたは設計のポイント

このシナリオでは、複数リージョンの Web アプリケーションへトラフィックを分散するため、グローバルロードバランサーを利用する必要があります。特に Web アプリケーションであることから、HTTP / HTTPS レベルでのトラフィック管理が重要になります。

Azure Front Door を使用すると、次のような設計が可能になります。

まず、Azure Front Door をグローバルエントリーポイントとして配置します。ユーザーは世界中の Microsoft エッジネットワークに接続し、そこから最適なリージョンのアプリケーションへルーティングされます。

次に、複数リージョンにデプロイされた Web アプリケーションをバックエンドとして登録します。Front Door はこれらのエンドポイントを継続的に監視し、正常なエンドポイントのみにトラフィックを送ります。

さらに、ユーザーの地理的位置やパフォーマンスに基づいて最適なリージョンへトラフィックを誘導することで、ユーザーの遅延を最小化できます。

## 5. 設計判断（なぜこの構成になるか）

この問題の正解は **B. Azure Front Door** です。

Azure Front Door は、Web アプリケーション向けに設計されたグローバルロードバランシングサービスであり、複数リージョン間のトラフィック分散と高可用性を同時に実現できます。

Front Door は Microsoft のグローバルエッジネットワークを利用してユーザーリクエストを最適なバックエンドへルーティングします。これにより、ユーザーの位置に最も近いリージョンへ接続されるため、遅延を大幅に削減できます。

さらに、Front Door はバックエンドのヘルスチェックを継続的に実行します。もし一つのリージョンが利用できなくなった場合、トラフィックは自動的に別リージョンへ転送されるため、アプリケーションの可用性が維持されます。

また、SSL 終端、キャッシュ、Web Application Firewall（WAF）などの機能も統合されているため、Web アプリケーションのセキュリティとパフォーマンスを同時に向上させることができます。

## 6. 他の選択肢が誤りな理由

他の選択肢にも Azure のロードバランシングサービスが含まれていますが、それぞれ用途が異なります。

まず、Azure Traffic Manager は DNS ベースのグローバルロードバランサーです。リージョン間のトラフィック分散は可能ですが、HTTP レベルの最適化やキャッシュ、アプリケーションアクセラレーションなどの機能は提供されません。そのため、Web アプリケーションのパフォーマンス最適化という観点では Azure Front Door の方が適しています。

次に、Azure Load Balancer は L4（TCP / UDP）レベルのロードバランサーであり、主に単一リージョン内で仮想マシン間のトラフィックを分散する用途で使用されます。グローバルレベルのトラフィック分散には対応していません。

最後に、Azure Application Gateway は L7（HTTP）ロードバランサーであり、URL ベースルーティングや WAF を提供します。しかし、これはリージョン内のトラフィック管理を目的としたサービスであり、複数リージョン間のトラフィック分散には使用されません。

## 7. 最終回答

B. Azure Front Door

## 8. まとめ

この問題は、Azure のロードバランシングサービスの役割を理解しているかを確認する問題です。Azure では、ロードバランサーは主に次の 2 つのレイヤーに分類されます。

リージョナルロードバランサーには Azure Load Balancer や Azure Application Gateway があり、単一リージョン内のトラフィックを分散します。一方、グローバルロードバランサーには Azure Traffic Manager や Azure Front Door があり、複数リージョン間のトラフィック分散を行います。

その中でも Azure Front Door は Web アプリケーション向けに設計されたサービスであり、グローバルロードバランシング、アプリケーションアクセラレーション、ヘルスチェック、WAF などの機能を提供します。

したがって、複数リージョンに展開された Web アプリケーションの可用性を確保し、ユーザーの遅延を最小化するための最適なサービスは Azure Front Door です。
