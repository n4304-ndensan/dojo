# 高可用性マルチリージョンWebアプリケーション向けAzureサービス選択

## 1. 背景（シナリオ）
Webアプリケーション向けに、高可用性かつマルチリージョンのアーキテクチャを設計する必要があります。ユーザーの地理的な近接性やリアルタイムのパフォーマンス指標に基づいてトラフィックをインテリジェントに分散させ、地域的な停電が発生した場合でもサービスを継続できる自動フェイルオーバー機能を提供する必要があります。

## 2. 要件整理
- **リアルタイムパフォーマンス指標に基づくトラフィック分散**
- **ユーザーの地域的近接性に応じたルーティング**
- **自動フェイルオーバーによる高可用性**
- **グローバル負荷分散と低遅延**
- **セキュリティ強化 (WAF, DDoS 保護など)**

## 3. 技術の基本概念
**Azure Front Door (AFD)** は、グローバルに分散されたWebアプリケーション向けのエントリーポイントサービスです。主な機能は以下の通りです。

- グローバルロードバランシング: 複数リージョン間でのトラフィック分散
- インテリジェントルーティング: リアルタイムパフォーマンス指標とユーザー位置情報に基づき最適なバックエンドへルーティング
- 自動フェイルオーバー: ヘルスモニタリングにより障害発生時にトラフィックを自動切り替え
- セキュリティ機能: Web Application Firewall (WAF)、DDoS保護、SSLオフロード
- 他Azureサービスとの統合: App Service、Functions、CDNなど

## 4. アーキテクチャまたは設計のポイント
- フロントエンドとしてAFDを配置し、全ユーザーリクエストをグローバルに最適化
- 複数リージョンにデプロイされたアプリケーションバックエンドに対して、レイテンシー・ヘルスベースのルーティングを行う
- AFDのフェイルオーバー機能により、地域障害発生時もサービス継続可能
- WAFやSSLオフロードにより、アプリケーションセキュリティを確保

## 5. 設計判断（なぜこの構成になるか）
- パフォーマンス指標に基づくインテリジェントルーティングを提供
- 自動フェイルオーバーにより高可用性を確保
- グローバルユーザーに対して低遅延アクセスを提供
- セキュリティ機能とAzureサービス統合により包括的ソリューションを提供

## 6. 他の選択肢が誤りな理由
- **Azure Traffic Manager**: DNSベースでトラフィックルーティングするため、リアルタイムパフォーマンスに基づくルーティングやHTTP(S)レベルの制御は不十分
- **Azure Application Gateway**: 単一リージョン向けでグローバル負荷分散機能がない
- **Azure Load Balancer**: L4ロードバランサーでマルチリージョン対応不可。HTTP(S)ルーティングやインテリジェントルーティングは提供されない

## 7. 最終回答
**A. Azure フロントドア**

## 8. まとめ
Azure Front Door は、マルチリージョンWebアプリケーション向けに、リアルタイムパフォーマンス指標に基づくインテリジェントなトラフィック分散、自動フェイルオーバー、グローバルロードバランシング、セキュリティ機能を統合的に提供します。これにより、Webアプリケーションに求められる高可用性と信頼性を確保できます。
